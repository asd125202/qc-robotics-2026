# 第二批路演网站方向

这一批不重复第一批的“硬件核心 / 云训练 / 技能商店 / 行业套件”，而是从竞品和市场研究中继续分化出更具体的商业包装。

## 方向 A：DragonWorks

定位：Qualcomm-first 机器人开发者工作台。

核心画面：一个中文控制台把设备、数据集、训练 job、模型版本、边缘性能和技能发布放在同一屏。

主叙事：

- 竞品有 Jetson + Isaac ROS；Qualcomm 需要一个同样清晰的开发者体验入口。
- DragonWorks 不取代 ROS / LeRobot / AI Hub，而是把它们组织成产品旅程。
- 评委会看到 Qualcomm 生态如何从样例项目走向可复制开发流程。

## 方向 B：EdgeFleet

定位：机器人队列运行与远程运维平台。

核心画面：工厂/校园/实验室中的多台机器人，仪表盘显示运行状态、技能版本、故障片段和训练回流。

主叙事：

- 企业买机器人后最怕维护不可控。
- EdgeFleet 把 Qualcomm 机器人变成可管理设备：在线状态、日志、模型版本、性能指标、故障回放。
- 安全原则：监控和更新可以上云，安全控制仍然本地闭环。

## 方向 C：EduForge

定位：高校与开发者教育套件。

核心画面：一套可开箱上课的机器人学习实验台，学生从数据采集、训练到边缘部署完成完整闭环。

主叙事：

- 中国具身智能人才需求上升，比赛本身也需要培养开发者。
- EduForge 让高校不再从散件和环境配置开始教机器人学习。
- Qualcomm 可以在教育入口建立 Dragonwing 开发者心智。

## 方向 D：Certified Skills

定位：Qualcomm 认证机器人技能计划。

核心画面：技能包通过硬件兼容性、延迟、安全边界、数据来源和评估报告检查，形成认证标签。

主叙事：

- 技能市场要可信，必须有运行认证。
- Qualcomm 可以成为机器人技能从云端训练到边缘运行的认证层。
- 这比单次比赛 demo 更具生态意义。

## 第二批建议

优先做三个网站：

1. `DragonWorks`：最适合说服 Qualcomm。
2. `EdgeFleet`：最适合说服企业客户。
3. `EduForge`：最适合比赛和开发者生态。

`Certified Skills` 可以和 `SkillDock` 合并，作为第二轮扩展章节或后续第三批独立网站。
