# RevenueStack Pitch

更新时间：2026-07-05。

## Core Thesis

评委要看到商业价值，不只是“这个机器人有市场”。RevenueStack 把 RobotMac 拆成客户能理解、销售能报价、企业能试点的收入结构：

- 一次性硬件 SKU。
- 云训练额度。
- 行业技能包。
- 队列运维订阅。
- 安全治理和审计包。
- 系统集成和试点合同。

这让项目从“做一个机器人 demo”变成“卖一个机器人产品平台”。

## Buyer Problem

企业客户通常不想先买一堆开发板、线材、电源、相机、GPU 云账号和开源 repo，然后自己集成。

他们想买：

- 一个能跑起来的硬件套件。
- 一个能训练自己任务的流程。
- 一个能上线、回滚、运维的控制台。
- 一个清楚的试点合同。
- 一个扩展到多台设备的价格路径。

RevenueStack 的作用是把这些打包成可采购产品，而不是让客户自己拼装。

## Pricing Ladder

### 1. Starter Kit

对象：高校、开发者、比赛团队。

内容：

- RobotMac Core Lite。
- EduForge / LabForge 基础套件。
- 少量 CloudTwin 训练额度。
- 基础 SkillDock 技能包。

目的：

- 扩大开发者入口。
- 让 Qualcomm edge target 成为默认学习路径。

### 2. Pilot Kit

对象：实验室、工厂、系统集成商。

内容：

- RobotMac Core Pro。
- Teleop Studio。
- CloudTwin 训练额度。
- DataFlywheel 数据治理。
- SafetyOps 发布门禁。

目的：

- 用 4-8 周证明一个具体任务。
- 收集客户数据并形成首个行业技能包。

### 3. Fleet Kit

对象：多台机器人部署客户。

内容：

- 多台 RobotMac Core。
- EdgeFleet 运维订阅。
- SkillDock 企业权限。
- SafetyOps 审计日志。
- 年度模型更新和支持。

目的：

- 从单点试点走向队列部署。
- 让收入从硬件一次性转向订阅和服务。

### 4. Industry Pack

对象：行业集成商和大客户。

内容：

- FactoryPilot、LogisticsMove、LabForge 等行业模板。
- 私有数据集、专属技能包和交付文档。
- 企业培训、现场部署和 SLA 支持。

目的：

- 把垂直场景沉淀成可复用行业包。
- 提高单客户合同额。

## Revenue Lines

### Hardware

RobotMac Core Lite / Pro / Industrial。硬件收入建立入口，但不应该是唯一利润来源。

### Training

CloudTwin 训练额度、双云 adapter、训练报告和模型评估。

### Data

DataFlywheel 数据账本、失败样本挖掘、行业数据模板和私有数据治理。

### Skills

SkillDock 技能包、行业模板、认证分发和版本更新。

### Fleet

EdgeFleet 设备状态、日志、OTA、模型灰度发布、维护工单。

### Governance

SafetyOps 权限、发布门禁、审计证据和回滚包。

## Competition Story

初赛项目书需要回答三个商业问题：

1. 谁买？
2. 买什么？
3. 以后为什么继续付钱？

RevenueStack 的回答：

- 高校和开发者买 Starter Kit。
- 实验室和工厂买 Pilot Kit。
- 试点成功后升级 Fleet Kit。
- 行业集成商购买 Industry Pack。
- 后续持续支付训练、技能、运维和治理订阅。

这比“卖一个机器人”更接近 Mac/Windows 式平台生意。

## Why Qualcomm Should Care

RevenueStack 把 Qualcomm 的价值变成多条收入线，而不是一次开发板曝光：

- 每个 Starter Kit 都培养一个 Qualcomm edge 开发者。
- 每个 Pilot Kit 都把客户数据和模型部署到 Qualcomm target。
- 每个 Fleet Kit 都带来长期 edge AI 运维场景。
- 每个 Industry Pack 都有机会形成 Qualcomm 参考设计和生态案例。
