# 产品主张：把机器人变成下一个个人计算平台

## 核心判断

个人电脑真正普及，不是因为 CPU 更快，而是因为用户不再需要从电源、主板、驱动、操作系统和应用生态开始拼机器。智能手机真正普及，也不是因为每个人都懂基带和传感器，而是因为硬件、系统、网络、应用分发和开发工具被压成了一个可购买、可升级、可迭代的平台。

机器人行业现在仍然停在更早的阶段：开发者通常要同时处理开发板选型、电源安全、传感器同步、运动控制、数据采集、模型训练、边缘部署、远程诊断和量产交付。这个项目的机会，是把这些复杂性压进一个商业化的一体化产品，让用户像使用电脑或手机一样使用机器人平台。

## 产品一句话

面向开发者、教育团队、工厂集成商和服务机器人创业团队的一体化机器人计算与训练平台：本地用 Qualcomm Dragonwing 边缘 AI 模块完成低延迟感知与控制，云端用 LeRobot 工作流完成数据管理、GPU 训练和技能迭代，最终把可复用的机器人技能部署回硬件。

## 为什么是 LeRobot + 云训练 + Qualcomm Edge

LeRobot 已经把机器人学习的关键对象抽象出来：硬件接口、数据集格式、模仿学习/VLA 策略、训练脚本、可视化与真实机器人教程。它降低的是“机器人学习软件栈”的门槛。

云 GPU 解决的是本地开发者没有训练算力、训练环境难复现、团队协作成本高的问题。中国版本可以优先适配阿里云 PAI、腾讯云 GPU、华为 ModelArts、AutoDL；海外版本可以优先适配 Runpod、Lambda、Modal、Paperspace 等。不同市场用不同云供应商，但统一数据格式、训练任务模板和部署包。

Qualcomm Dragonwing 解决的是机器人必须在本体上稳定运行的问题：低功耗、异构计算、摄像头/多媒体、连接能力、AI 加速、安全和长期供货。云端可以训练，展示可以上云，但控制闭环不能依赖公网。

## 要卖的不是开发板，而是“机器人产品底座”

### 硬件层

- 一体化边缘计算核心：Qualcomm Dragonwing SoC、散热、存储、网络、安全启动。
- 电源与执行层：电池/外部电源输入、稳压、电机供电隔离、急停、保险丝、状态灯。
- 机器人 IO：摄像头、USB、以太网、CAN、UART、PWM、I2C、SPI、音频、IMU、可选雷达/深度相机接口。
- 形态扩展：桌面机械臂、移动底盘、巡检车、服务机器人、行业夹具。

### 软件层

- 机器人运行时：传感器采集、时间同步、模型推理、策略执行、安全边界、日志。
- LeRobot 适配层：记录数据、上传/同步数据集、训练 ACT/SmolVLA/X-VLA 等策略、导出部署包。
- Qualcomm 部署层：模型转换、性能测量、NPU/GPU/CPU 分配、离线回放。
- 云端控制台：数据集版本、训练任务、评估结果、设备状态、技能发布。

### 商业层

- 硬件套装收入：一体机、行业套件、传感器/执行器扩展。
- 云训练订阅：按项目、按训练小时、按团队席位、按私有部署收费。
- 技能市场抽成：抓取、巡检、分拣、跟随、质检、教学等技能包。
- 企业交付：行业模板、私有数据、现场调试、长期维护。

## 第一批网站概念

1. `RobotMac Core`：一体化机器人计算与电源核心，主打“把开发板变成可交付产品”。
2. `LeRobot CloudTwin`：云端训练闭环，主打“采集数据、云上训练、本体部署”的产品飞轮。
3. `SkillDock`：机器人技能商店，主打“开发者生态与技能复用”。
4. `Embodied Kit`：行业交付套件，主打“工厂、实验室、服务公司能采购的完整套装”。
