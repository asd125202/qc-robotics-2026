Pitch 50 · Eldercare Exception Closure

把远方子女的担心，变成可执行的照护。

护元 CareOS 面向居家、社区、机构、长护险服务商和护理团队，把日常节律、温和提醒、非急救异常、机器人递送、人机接管、家属摘要和服务证据，变成可派单、可验证、可复盘、可训练的照护异常闭环。

看照护闭环 看商业模型

01 · Problem

养老的痛点不是缺一个聊天机器人，而是没人知道“现在该谁做什么”。

父母不想被摄像头监控，也不想过早离开熟悉的家；子女每天在猜：没接电话是在睡觉、忘了提醒，还是需要人去看一眼？护理员进门前也常常不知道过去 12 小时发生了什么。

家庭在猜

电话、手表、按钮、摄像头、微信群和家政记录碎片化，不能形成连续照护上下文。

护理员在跑

巡房、提醒、送水、取物、记录、家属沟通切碎时间，真正需要人的时刻反而被淹没。

机构难举证

夜间巡视、离床/无回应、用药提醒、清洁和家属沟通需要可审计证据，而不是事后补表。

支付难闭环

长护险、居家床位和社区养老服务需要服务过程、异常处置、确认人和结算材料。

02 · Why Now

老龄化、护理缺口、政策支付和 edge AI 在同一窗口叠加。

如果只有老龄化，这只是宏观趋势；真正的创业窗口来自家庭照护能力变薄、机构人力不足、长护险制度化、智能养老机器人试点和端侧机器人能力同时成熟。

3.2338亿 2025 年末中国 60 岁及以上人口，占总人口 23.0%。
40.6万 2024 年末中国养老机构和设施数量，已形成服务部署入口。
1100万 WHO 估计到 2030 年全球卫生工作者短缺规模。
765,800 BLS 预计美国居家护理和个人照护岗位每年平均空缺。

03 · Core Insight

家庭不会为“机器人”付费，他们为“我知道下一步该谁做”付费。

最好的养老机器人不是替代护工，而是把人的注意力留给真正需要人的时刻。CareOS 的核心不是诊断或防跌倒承诺，而是发现照护异常、找到责任人、辅助确认、关闭任务、留下证据。

不是跌倒报警器

报警只是开始；关键是确认、分派、升级、复核、记录和家属/机构交接。

不是陪聊玩具

陪伴和提醒有价值，但服务商和机构预算看的是任务闭环、服务记录和人力放大。

不是机器人护士

先做低风险、可接管、可审计的提醒、递送、巡护和非急救异常闭环。

04 · Solution

护元 CareOS 是照护异常闭环系统。

它在老人同意的边界内学习日常基线，先用温和提醒处理低风险任务，再按风险和权限升级给家人、护理员、机构或服务商。机器人只在安全边界内执行递送、提醒、巡护和低风险辅助。

  • 发现异常：未回应、药盒未确认、夜间多次起身、长时间静止、离床/离房、通道障碍、服务逾期。
  • 判断优先级：按风险等级、护理计划、时间窗口、家属偏好、机构 SOP 和服务商 SLA 排序。
  • 自动派单：分配给老人本人、家属、邻居、护理员、机构值班人员、服务商或远程接管员。
  • 验证结案：用确认按钮、语音回应、位置摘要、机器人状态、护理员记录和签名证据关闭任务。
  • 训练迭代：把低置信度、人工接管、误报/漏报复盘和机器人动作失败导出为 LeRobot episode。

05 · Product Workflow

从 Care Circle 到 Check-in、Escalation 和 Evidence。

CareOS 的产品体验不是“看监控”，而是让家庭、护理员和机构在权限清晰的照护圈里知道发生了什么、谁处理了、下一步是什么。

Care Circle

老人、子女、护理员、邻居、机构和医生联系人各自拥有不同权限和通知规则。

Baseline

端侧学习起居、活动、门磁、药盒、睡眠和厨房节律，只保存必要元数据。

Assist

温和提醒吃饭、喝水、预约、药盒确认、起夜柔光和轻物品递送。

Escalate

疑似跌倒、无回应、长时间静止或异常开门，先询问本人，再通知照护圈。

Close

确认、接管、上门、复核和下一步安排进入护理日志与训练数据。

06 · Market Wedge

先从高焦虑、高支付意愿、高闭环需求的照护入口切入。

CareOS 不从“所有老人买一台机器人”开始。先做远程子女 + 独居/半独居父母，或 assisted living / memory care / 社区长护服务中的高频照护异常闭环。

远程子女家庭

术后恢复、行动不便、轻度记忆下降、夜间风险高和子女异地，是付费焦虑最明确的入口。

Home-care Agencies

班次之间缺少可见性；CareOS 给出 no-contact、迟到、漏记、未确认和家属沟通闭环。

Senior Living

wellness check、夜间离床、漏服药提醒、家属询问和不必要人工巡查需要排序。

中国社区养老

街道养老站、家庭养老床位、居家上门服务和补贴核销需要统一服务证据。

长护险服务商

失能评估、护理项目目录、服务证据、结算材料和质量监管需要闭环。

医养/康复机构

出院后康复、慢病随访、康复提醒、护理机器人接入和家属摘要。

07 · Business Model

按可闭环照护异常收费，不按机器人硬件一次性收费。

先用软件编排 + 轻传感器 + 可选机器人降低部署阻力；机器人作为放大器进入高频递送、巡护和接管场景。价格按 resident、active client、episode、家庭订阅和 OEM 授权分层。

机构 / Senior Living

中国 ¥20-60/人/月 core，¥60-150 pro；海外 US$8-20 core，US$20-45 pro。

高护理强度

护理院/高 acuity：中国 ¥60-150/人/月，¥150-280 高阶；海外 US$15-95/人/月。

居家服务机构

中国 ¥10-30/活跃客户/月，¥30-80 带家属门户；海外 US$5-30/客户/月。

出院 / 康复 Episode

中国 ¥80-300/30天 episode，¥300-800 带临床/RPM 伙伴；海外 US$25-175。

家庭订阅 / OEM

家庭 app、监测、concierge 月费；长期以 Qualcomm/ODM/OEM reference design + CareOS 授权扩张。

08 · Go-To-Market

先证明“照护异常更快被承认和关闭”，再谈医疗结果。

8-12 周试点只承诺工作流指标：确认时间、关闭时间、SLA 内关闭率、未分配异常、重复误报、照护记录完整率、家属咨询量和人工接管。跌倒、急诊、再入院只跟踪，不作为早期效果承诺。

90 天家庭试点

10-25 个家庭验证老人接受度、家庭日活、无回应升级时间、误报复盘和隐私满意度。

机构试点

3-10 台设备，夜间巡护、提醒、轻递送和异常上报；目标 75%-85% 异常 SLA 内关闭。

服务商试点

home-care agency 验证迟到/漏访/无联系异常、护理记录 24 小时内完整率和家属咨询下降。

中国政策入口

智能养老服务机器人试点、街道养老站、家庭养老床位、长护险定点服务商和 OEM 伙伴。

09 · Competition

现有产品证明需求，但多数停在单点事件。

CareOS 的差异不是再做一个跌倒检测、药盒提醒或陪伴机器人，而是把传感器、机器人、家属、护理员、机构系统和服务证据连成照护异常闭环。

跌倒 / 远程监测

SafelyYou、Vayyar、Nobi、CarePredict、Apple Watch 等发现事件强，但闭环和分派弱。

药物提醒

Hero、MedMinder、Spencer 等提醒明确，但缺少和起居、活动、护理任务的统一优先级。

陪伴机器人

ElliQ、PARO、Lovot 等情感交互强，但物理照护任务和机构 workflow 弱。

机构软件 / EHR

PointClickCare、MatrixCare、WellSky、AlayaCare 是 record system，不是实时执行层。

配送 / 辅助机器人

Moxi、Aethon、Relay、Labrador、Pudu 可减轻跑腿，但缺少 eldercare exception OS。

CareOS

把发现、确认、分派、机器人辅助、家属摘要、服务证据和 HIL 训练串起来。

10 · Moat

养老的护城河是信任、流程和隐私边界。

单点传感器会被替换，闭环流程、家庭权限、机构 SOP、HIL 恢复数据、机器人安全边界和 Qualcomm edge profile 会长期沉淀。

信任界面

老人愿意长期放在家里，子女能理解摘要，护理员不觉得被监控。

隐私边缘计算

默认输出事件、轮廓、时间线和置信度，不把生活空间变成云端摄像头。

照护图谱

谁该知道什么、何时升级、谁确认、如何交接，随家庭和机构长期使用变准。

多模态基线

视觉、语音、门磁、药盒、活动、环境和机器人状态融合，比单设备更稳。

服务证据

任务、异常、确认、接管、上门、家属查看和服务商结算材料可审计。

Edge Profiles

QNN/AI Hub profile、签名 OTA、回滚、模型版本、延迟和安全拦截记录。

11 · Architecture

本地感知，本地安全，云端只接收最小必要证据。

CareOS 用 Qualcomm edge 运行感知、导航、提醒和安全门控；云端管理照护圈、证据包、设备策略、模型注册和区域租户。云训练不能绕过本体侧安全监督。

Robot Edge

RB5/RB3 Gen 2/IQ10 路线，连接 RGB-D/ToF、LiDAR、IMU、麦克风、托盘、低速底盘和急停。

CareOps

Care Circle、提醒、异常、服务记录、角色权限、家属摘要、机构 dashboard 和审计日志。

Safety Layer

低速、禁区、隐私区、力/速度限制、接管状态、local stop、Nav2 collision monitor 和 HIL。

Learning Cloud

LeRobot episode、去标识化 HIL correction、中国/全球租户、AI Hub/QNN profile、签名部署和回滚。

12 · Competition Demo

3 分钟演示照护闭环，不碰医疗红线。

用模拟客厅、厨房 waypoint、二维码药盒、封闭水瓶/餐卡、低速机器人、游戏手柄接管和 caregiver dashboard 展示 reminder、handoff、non-emergency anomaly、evidence packet 和 LeRobot HIL。

Schedule

护理员设置药盒、水和餐食提醒；药物为 placebo prop，只做计划提醒。

Assist

机器人本地提醒并低速导航到厨房 waypoint，递送封闭水瓶或餐卡。

Check-in

演员未回应并在椅边暂停，系统标记“需要确认/风险信号”，不说“已跌倒”。

Escalate

机器人退后询问是否需要帮助；无回应则把最小必要证据发给 caregiver。

Learn

低置信度 handoff 由人接管，生成 HIL episode 和 AI Hub/QNN profile 卡片。

养老场景长期运行、弱网、隐私敏感、需要低功耗端侧 AI、多传感、机器人连接和签名 OTA。它比炫技人形更能展示 Qualcomm 在 home physical AI、health-adjacent edge 和机器人生态里的战略位置。

Always-on Edge AI

姿态、活动、物体、条码、语音、异常节律和隐私过滤优先在本地完成。

Robotics Runtime

ROS 2、Nav2、Open-RMF、低速导航、HIL 接管和安全门控需要稳定 edge。

AI Hub / QNN

比赛阶段目标不是证明机器人能替代护理员，而是证明照护异常可以被发现、确认、分派、辅助、关闭、留证并转成下一轮 HIL 数据。

Edge Kit

RB3 Gen 2、RB5、QCS6490/QCS8550 或 IQ10 路线，用于端侧感知、导航和 handoff demo。

Scenario Data

Partners

养老机构、社区养老站、home-care agency、长护险服务商、机器人 OEM 和适老化渠道。

Trial Metrics
