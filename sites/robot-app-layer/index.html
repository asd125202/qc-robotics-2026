Pitch 18 · App Store + MDM + CI/CD For Robots

RobotAppLayer

把机器人技能从“能跑的 demo”变成可签名、可验证、可上架、可收费、可灰度、可回滚的 app release。RobotAppLayer 是 ROS 2 + LeRobot + Qualcomm edge 之上的可信应用发布层。

看平台断层 看发布演示

01 · Problem

机器人开发现在仍像项目制系统集成。

客户买的不是 checkpoint 或 ROS package，而是“这个技能能否在这台机器人、这个现场、这个 Qualcomm edge target 上被允许安装、运行、计费、审计和回滚”。今天这些证据散落在脚本、日志、表格和工程师脑子里。

Fragmented

发布对象不完整

ROS package、launch、topic/action、驱动、模型、现场参数、权限、日志、OTA、回滚和收费各自分散。

Developer

开发者先做集成

想写巡检、配送或样品转移，却先被相机、夹爪、ROS QoS、driver bring-up、模型导出和现场调参拖住。

Physical

权限会影响真实世界

机器人 app 不只是访问相机/位置，它会移动、抓取、开门、靠近人、上传现场数据和影响真实资产。

Trust

买方要责任边界

企业需要 SBOM/VEX、签名来源、数据归属、事故日志、支持期、回滚和谁为风险负责。

Market

生态无法复用收费

没有可信兼容性、认证、分发、授权、用量计费和分成，技能只能停留在一次性项目。

02 · Current Alternatives Fail

现有层都必要，但没有谁负责 trusted robot app release。

缺口不是“再造一个 robot OS”。缺口是把技能包、物理权限、LeRobot policy、Qualcomm edge profile、商业分发、安装门禁和回滚治理放进同一条 release workflow。

ROS 2

提供 topic、service、action、lifecycle、QoS、DDS security；不是 app ABI、权限商店、计费系统或发布治理层。

LeRobot

正在闭合 robot learning loop；不负责签名技能包、企业上架、entitlement、billing、fleet rollback。

AI Hub / QNN

能产生模型级 compile/profile evidence，但这些证据还没有变成“技能能否上架和安装”的发行门禁。

OEM Stores

UR+、MiR Go、Unitree、OpenMind 证明方向成立，但多数锁在单一品牌或早期行为包。

Viam / Intrinsic

解决开发、registry、工作站或工业技能问题；中立 release workflow 和物理权限仍有空位。

Isaac / Open-RMF

强化仿真、基础模型或调度协议；不是跨硬件应用分发、收费和回滚系统。

03 · Solution

RobotAppLayer = 机器人应用 ABI + trusted release workflow。

开发者提交 `.rap` signed skill bundle；平台检查签名、权限、ODD、ROS graph、LeRobot lineage、Qualcomm edge profile、billing SKU、support boundary 和 rollback target，才允许安装到真实机器人。

.rap Bundle

app id、signature、SBOM/VEX、OCI artifact、container/Wasm、ROS map、policy binding、billing SKU、rollback target。

RobotKit API

observe、move、grasp、speak、record、train、profile、install、monitor、rollback。

Safety Broker

app 提交 intent，不直接发布 raw motor、cmd_vel 或 joint trajectory；broker 检查 geofence、E-stop、人距和 ODD。

SkillDock

公开市场、企业私有商店、OEM 商店和 SI 套件共享签名、兼容、授权、计费和分成对象。

EdgeFleet

canary、cohort targeting、health gate、incident capsule、kill switch、A/B rollback 和 audit export。

04 · Why Now

2026 年，机器人 app store 从概念变成市场信号。

工业市场已经为认证兼容性付费，Unitree/OpenMind 把 humanoid/quadruped 行为包带到 app-store 叙事，LeRobot 正在闭合学习循环，Qualcomm 正在铺 prototype-to-production robotics path。现在缺的是 release loop。

500+ UR+ 已有 over 500 certified kits、components、software 和 accessories。
160+ MiR Go 已有 160+ applications / top modules，验证 AMR 生态分发。
2026 Unitree UniStore、OpenMind robot app store 让机器人行为包进入大众叙事。
v0.6 LeRobot v0.6.0 聚焦 deployment CLI、HIL corrections、HF Jobs 和 benchmark/eval。
OCI 1.1 OCI artifact、Sigstore、TUF/Uptane 让签名、attestation 和安全 OTA 有成熟基座。
Dragonwing Robotics Hub、RB3、QNN/QAIRT、AI Hub 和 IQ / Dragonwing path 让 edge target 更清晰。

05 · Product

先做受控子集：ROS 2 + LeRobot + Qualcomm edge 的可信技能发布。

MVP 从低风险高复用场景切入：LabForge 样品转移、warehouse aisle scan、indoor inspection、education kit、teleop data collection、failure replay 和低力矩操作技能。

  • RobotApp SDK：Python / TypeScript API，面向 observe、move、grasp、record、train、profile、deploy、monitor。
  • Permission Manifest：camera、mic、map-read、navigate-zone、drive-low-speed、manipulate-object、remote-control、near-human-operation、cloud upload。
  • Compatibility Registry：robot profile、sensor、effector、ROS distro、LeRobot version、ONNX/QAIRT/QNN runtime、Qualcomm target、compute budget、ODD。
  • Certification CI：static manifest、ROS graph contract、MCAP replay、sim smoke、HIL、latency budget、collision/intervention/success gates、SBOM/VEX、rollback test。
  • Qualcomm EdgeProfile Builder：AI Hub / QNN / ONNX Runtime QNN profile，并另测 end-to-end camera-to-action p95。
  • SkillDock + EdgeFleet：listing、trial、subscription、metering、settlement、canary、incident capsule、kill switch、rollback 和 site audit。

06 · Product API/Evidence

评委要看到 app release API 和 evidence pack，不是平台口号。

每个技能都必须解释自己是谁、能做什么、在哪些机器人上可运行、用哪些数据、请求哪些物理权限、跑在哪个 Qualcomm runtime、怎么收费、怎么回滚。

SkillManifest

PermissionPolicy

LeRobotBinding

dataset lineage、checkpoint、train_config、normalization stats、observation/action schema、eval metrics。

QualcommEdgeProfile

device、runtime、QNN/QAIRT version、ONNX opset、p95 latency、memory、thermal drift、fallback ops。

AppRelease

install checks、canary rollout、active/degraded state、incident capsule、fleet metrics、entitlement、rollback。

Evidence pack：`skill-passport.json`、`edge-profile.qualcomm.json`、`qnn_profile.csv`、`mcap_replay_report.json`、`lerobot-lineage.json`、`permission-policy.yaml`、`sbom.cdx.json`、`vex.json`、`signature.bundle`、`rollout-plan.yaml`、`rollback-plan.yaml`。

07 · Market & Business Model

用户买的不是“app”，而是降低集成风险、复用行为和持续发布能力。

RobotAppLayer 服务 robot OEM、SI、RaaS/fleet operator、企业机器人团队、教育/研究机构和技能开发者。它把 marketplace、认证、runtime、billing、payout 和 support boundary 连接起来。

中国版

企业/教育 skill marketplace，不做纯消费 app store

支持 OEM bundle、SI 私有部署、教育/比赛套件、本地云/BYOC、WeChat/Alipay 分账、e-fapiao、ICP/App filing metadata、China-hosted telemetry/model endpoints。重点类别：motion/action packs、酒店/餐饮配送、清洁/巡检、teleop/data collection、education lesson、enterprise connectors。

海外版

enterprise control plane + private marketplace

支持 Stripe Connect / Merchant of Record、Apple/Google-style commission、GDPR/COPPA/EU AI Act controls、regional hosting、SSO/RBAC、audit/SLA、certified partner app、ROS/Open-RMF/Isaac/LeRobot integrations。

$0 免费 SDK、simulator adapter、样例应用和公开 listing。
$49-149 Team / Pro 每开发者每月：私有应用、CI 验证、日志、权限和仿真任务。
$50-250 Production runtime 每活跃机器人每月：部署、遥测、权限、回滚、审计。
$1k+ Certification / review fee；高风险或高合规应用 $10k+。
10-30% Marketplace take rate；工业场景可降低分成并增加认证费 / 支持费。

08 · Competition & Moat

竞争定位不是“另一个 robot OS”，而是 trusted app release layer。

相邻玩家各有强项：ROS Index、LeRobot、Viam、Intrinsic、Foxglove、Open-RMF、NVIDIA Isaac、UR+、MiR Go、PUDU、Unitree、OpenMind。RobotAppLayer 的壁垒是把这些能力汇入可复用、可审计、可收费的发布流程。

App ABI

比 ROS topic 更稳定的 RobotKit 能力接口，降低每个技能对底层实现细节的依赖。

Physical Permission Graph

数据、传感器、地图、动作、远程控制、云训练和 near-human operation 的可审计权限图。

Compatibility Graph

`skill x robot profile x sensor x ROS distro x LeRobot policy x QNN runtime x site policy`。

Certification Corpus

sim、HIL、MCAP replay、incident replay、safety broker rejection 和 rollback recovery 的失败模式资产。

Marketplace Gravity

开发者、SI、OEM、企业客户和教育用户共享安装、试用、授权、计费和分成流程。

Qualcomm Evidence

AI Hub/QNN/QAIRT profile、低功耗边缘运行、连接、安全启动和 device attestation 成为上架门槛。

09 · Why Qualcomm

Qualcomm 的战略价值不是“被写进硬件清单”，而是成为每个机器人技能 release 的默认 edge target。

对抗 NVIDIA 的叙事不是只比 TOPS，而是 edge-efficient robot app release pipeline：cloud trains, Qualcomm edge ships, marketplace distributes, EdgeFleet rolls back, RevenueStack bills。

  • Dragonwing Robotics Hub sample project 可以变成 signed skill package。
  • RB3 / QCS6490 做近期验证，QCS8550 / IQ / Dragonwing 做 production-forward profile。
  • AI Hub/QNN/QAIRT evidence 变成 buyer-facing release receipt：latency、memory、backend、NPU placement、fallback、profile date、target board。
  • 每个 app listing 展示 `Dragonwing-profiled candidate`：target、runtime、model inference p95、end-to-end p95、memory、thermal、permissions、rollback、support boundary。

10 · Demo & Ask

7 分钟 demo：把 LabForge 样品转移从脚本变成可发布 app。

评委看到的不是一次性机械臂脚本，而是一条真实应用发布链路：签名、权限、数据、训练、edge profile、skill card、canary 安装、回滚、计费和审计。

Init

`robotapp init lab-transfer --target rb3-gen2` 生成 `.rap`、RobotProfile、PermissionPolicy、TaskContract。

Run

代码调用 observe / grasp / record，不直接暴露 raw motor topic、cmd_vel 或临时 launch file。

Train

运行样品转移，生成 MCAP + LeRobotDataset v3 episode；失败和人工接管回流训练。

Profile

策略导出后走 AI Hub / QNN / ONNX Runtime QNN，生成 `edge-profile.qualcomm.json`。

Publish

进入 SkillDock card，EdgeFleet canary 安装；异常触发 rollback，并生成 meter event 和 audit export。

Ask

向高通要什么

6-8 周 validation sprint；RB3 Gen 2/QCS6490 target；QCS8550 / IQ / Dragonwing roadmap feedback；AI Hub/QNN/QAIRT office hours；2-3 个 reference skills；Robotics Hub 发布指导；5-10 个 OEM/SI/developer intro。

Outcome

比赛交付物

一个 `Dragonwing-profiled candidate` 应用卡、一个可运行 LabForge `.rap` 技能包、一个可复现数据/训练/边缘部署链路、一个开发者 marketplace 原型和一套商业定价模型。

RobotMac 解决“机器人怎么稳定运行”，RobotAppLayer 解决“开发者怎么在机器人上发布可信、可收费、可回滚的应用”。
