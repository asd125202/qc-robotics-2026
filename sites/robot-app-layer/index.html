Pitch 18 · Robotics Application Layer

RobotAppLayer

机器人缺的不是又一个 ROS 发行版，而是一个能让开发者像发布手机应用一样发布机器人技能的应用层：签名包、权限、兼容性、仿真评测、LeRobot 策略、Qualcomm edge profile、上架、计费和回滚。

看平台断层 看应用演示

01 · Problem

今天写一个机器人应用，仍然像重新做一遍系统集成。

OEM、系统集成商和企业开发者不是缺少单个 ROS package，而是缺少一个稳定的应用合约。每个项目都要重新处理设备适配、权限、安全边界、模型部署、数据回放、现场配置、日志、OTA 和售后责任。

Integration

应用从底层开始

开发者想写巡检、配送、实验室流程，却先被相机、夹爪、ROS topic、launch file、QoS 和 driver bring-up 拦住。

Trust

没有物理权限模型

手机应用会声明相机和位置权限；机器人应用需要声明导航、抓取、语音、云上传、人体接近和高力矩动作。

Deploy

上线不可复制

一个技能在 demo 里能跑，不代表可以被签名、安装、灰度、回滚、审计，并在不同机器人 profile 上稳定运行。

Data

学习闭环断裂

LeRobot 数据、MCAP 日志、现场失败、人工接管、仿真评测和边缘 profile 经常散落在不同工具里。

Market

技能无法形成生意

开发者没有可信兼容性、认证、分发、试用、授权、用量计费和收入分成，生态就很难越过一次性项目。

02 · Current Alternatives Fail

底层、中间件、云平台和 OEM 商店各有价值，但都没有定义通用机器人应用层。

ROS 2 是必要基座，LeRobot 是学习和策略基座，Foxglove 是观测和数据基座，Open-RMF / VDA 5050 是调度和协议基座，Viam / Intrinsic / Isaac 是强平台信号。缺口是跨机器人、跨技能、跨商业模式的应用运行时和分发层。

ROS 2

提供 topic、service、action、lifecycle、QoS 和 DDS 安全能力，但不是 app store、权限系统、计费系统或认证市场。

Open-RMF / VDA 5050

适合多机调度、AMR/AGV 协议和设施协调；它们不负责技能包、模型、权限、上架和收益分成。

Foxglove / MCAP

让物理 AI 数据更可观察，但观测平台本身不会决定一个机器人应用能否安装、运行、回滚和收费。

Viam / Intrinsic

最接近机器人开发平台方向，但更偏自有 runtime、企业工作站或工业自动化；机会在中立应用合约。

NVIDIA Isaac

强化 GPU-first 仿真、ROS 加速和 foundation model 路线；不是 Qualcomm-native 的技能分发和边缘证据层。

OEM App Store

UR+、KUKA、FANUC、ABB、PUDU、Unitree 都证明生态方向成立，但多数锁在单一硬件品牌。

03 · Solution

RobotAppLayer 是 ROS 2 和 LeRobot 之上的机器人应用 ABI。

它不替代 ROS，而是把机器人能力暴露成稳定应用接口：开发者提交 `.rap` 技能包，声明权限、硬件 profile、LeRobot policy、仿真测试、Qualcomm edge target 和计费方式；平台负责映射到底层 ROS、数据、边缘运行和 fleet 发布。

Package

`.rap` bundle 包含 app id、签名、SBOM、容器、launch graph、ROS interface map、模型引用和兼容矩阵。

Permission

声明 camera、lidar、map、navigate、grasp、human proximity、cloud upload、train on customer data。

RobotKit

稳定 API 覆盖 observe、move、grasp、speak、record、train、profile、install、monitor 和 rollback。

Cert

用 MCAP replay、Gazebo/Isaac 仿真、HIL、ODD 门禁和 QNN profile 作为安装前证据。

Market

公开 marketplace、私有企业商店、OEM 商店和 SI 套件都使用同一套签名、授权和计费对象。

04 · Why Now

机器人正在进入“应用生态”阶段，但还没有通用运行时。

ROS 2 基础设施成熟，LeRobot 把低成本数据和策略训练带给更大开发者群体，Foxglove / MCAP 把物理 AI 数据标准化，VDA 5050 v3.0.0 推动 AMR 协议，Unitree、Hugging Face、OpenMind 和 OEM 商店让“机器人应用”成为用户能理解的形态。

ROS 2 topic、service、action、lifecycle、QoS、DDS security 已经足够承接应用层映射。
LeRobot 开源数据、策略、HIL 和低成本机器人把 skill training 变成开发者工作流。
OEM Stores UR+、KUKA、FANUC、ABB、PUDU、Unitree 等生态说明买家愿意为可信组件降低风险。
Physical AI Intrinsic、NVIDIA Isaac、GR00T 和 Foxglove 把机器人软件竞争推向平台层。
Dragonwing RB3 Gen 2 可做近期验证，IQ10 RRD 可成为 2026 年后的生产 profile 路线。

05 · Product

开发者写任务，平台处理机器人世界的复杂性。

RobotAppLayer 的第一版本不追求“支持所有机器人”。它先服务 ROS 2 + LeRobot + Qualcomm edge 的可控子集，从低风险数据采集、巡检、实验室样品转移、远程运维和教学应用开始，再扩展到导航和操作技能。

  • RobotApp SDK：Python / TypeScript API，面向 observe、move、grasp、record、train、profile、deploy、monitor。
  • Permission Manifest：物理世界权限、ODD 限制、ROS/DDS 发布订阅边界、网络出站和云训练授权。
  • Compatibility Registry：robot profile、sensor、effector、ROS distro、LeRobot version、Qualcomm target、runtime 和 compute budget。
  • Certification CI：MCAP replay、仿真、HIL、碰撞/接管/成功率门槛、QNN profile、SBOM 和回滚测试。
  • SkillDock Store：公开市场、企业私有商店、OEM 商店、试用、订阅、用量计费和开发者分成。
  • EdgeFleet Runtime：签名安装、灰度、遥测、事故包、数据回流、版本回滚和客户现场审计。

06 · Product API

应用层对象必须比 ROS topic 更稳定，比普通移动应用权限更严格。

RobotAppLayer 的核心是结构化合约：每个技能知道自己依赖什么机器人能力、能做什么动作、使用什么数据、部署到哪种 Qualcomm edge target、发生失败时如何记录和回滚。

RobotProfile

ROS domain、namespace、lifecycle、传感器、夹爪、底盘、机械臂、E-stop、RB3/QCS6490/QCS8550/IQ10 target。

TaskContract

任务阶段、前置条件、成功标准、人工接管、失败标签、MCAP 记录和 LeRobot episode 生成策略。

SkillManifest

app id、publisher、签名、版本、容器、ROS interface map、LeRobot policy、rollback target 和 billing SKU。

PermissionPolicy

允许 camera/map/navigate/grasp；禁止 raw motor、E-stop reset、人体接触和未经授权的 cloud upload。

EdgeProfile

QNN context binary、ONNX Runtime QNN、输入 shape、p95 latency、memory、compute unit 和 AI Hub profile evidence。

AppRelease

install checks、canary rollout、active/degraded 状态、incident capsule、fleet metrics、entitlement 和 rollback。

07 · Market & Business Model

卖给机器人公司和系统集成商的，是应用生态的基础设施。

机器人软件市场已经是数百亿美元级别赛道，OEM 生态商店和 developer portal 正在验证方向。RobotAppLayer 的商业机会不是取代 OEM，而是让 OEM、SI、企业开发者和应用开发者共享一套 app packaging、验证、部署、计费和分成标准。

中国版

OEM 套件 + 私有云 + 集成商分发

中国市场硬件迭代快、低价机器人多、服务机器人和人形开发者活跃。商业化先走 OEM bundle、SI 私有部署、教育/比赛套件、低 take-rate 市场和云训练用量。

海外版

企业控制平面 + 认证市场 + 合规 SLA

美国、欧洲和日本采购更慢，但愿意为安全、责任边界、审计、私有商店、SSO、合规证据、认证 partner app 和支持 SLA 付费。

$0 免费 SDK、模拟器适配、样例应用和公开 marketplace listing，用来获得开发者。
$49-149 Team / Pro 每开发者每月：私有应用、CI 验证、日志、角色权限和仿真任务。
$50-250 每台生产机器人每月：部署、权限、遥测、回滚、审计和 fleet runtime。
15-30% 付费技能市场 take rate；工业场景可用较低分成加认证费。
$25k+ 企业站点许可：私有商店、SSO、合规、on-prem、WMS/MES 集成和 SLA。

08 · Competition & Moat

壁垒不是“我也封装了 ROS”，而是可信应用生态的复利。

中立应用层一旦形成，会积累 app ABI、权限 taxonomy、兼容性矩阵、仿真认证 corpus、真实事故/失败数据、开发者分发、OEM/SI 合作和 Qualcomm edge profile。每一个新技能都会让下一个技能更容易上线。

App ABI

比 topic 更稳定的 RobotKit 能力接口，避免每个应用依赖底层实现细节。

Permission Graph

物理动作、数据、网络、云训练和人工接管形成可审计权限图。

Compatibility Matrix

机器人型号、传感器、执行器、ROS distro、LeRobot policy 和 Qualcomm target 的兼容证据。

Certification Corpus

仿真、HIL、MCAP replay、事故回放和安全门禁让平台越用越懂机器人失败模式。

Marketplace Gravity

开发者、SI、OEM、企业客户和教育用户被同一套安装、试用、授权和分成流程连接。

Qualcomm Evidence

AI Hub/QNN profile、低功耗边缘运行、连接、安全启动和设备 attestation 变成上架门槛。

09 · Why Qualcomm

高通不只需要开发板曝光，更需要成为机器人应用默认 target。

NVIDIA 正在用 Jetson / Isaac / GR00T 占据 physical AI 心智。Qualcomm 的优势在边缘 AI、功耗、相机、连接、安全、Linux/Android 生态和量产硬件。RobotAppLayer 可以把这些优势放进每个机器人应用的发布流程。

  • 开发者 funnel：Robotics Hub sample project -> RobotAppLayer SDK template -> AI Hub/QNN profile -> signed skill package -> marketplace rollout。
  • 近期目标：RB3 Gen 2 / QCS6490 做比赛和早期验证，不把 demo 绑定在尚未大规模可得的未来硬件上。
  • 路线目标：QCS8550 / IQ10 RRD 作为 forward profile，强调 production robotics 的多相机、传感器、连接、安全和 fleet lifecycle。
  • 上架门槛：每个技能卡都展示 Qualcomm edge profile、latency、memory、compute unit、permissions、rollback 和 fleet evidence。

10 · Demo & Ask

7 分钟证明：机器人应用可以像软件一样安装、验证、运行、失败回流和再发布。

比赛 demo 不只展示机械臂完成样品转移，而是展示一个真正可复用的应用发布流程。评委看到的是商业化平台，不是一次性脚本。

Init

`robotapp init lab-transfer --target rb3-gen2` 生成 `.rap`、RobotProfile、权限和任务合约。

Run

代码调用 observe / grasp / record，而不是直接暴露 raw motor topic 和临时 launch file。

Train

失败片段、人工接管和成功样本生成 LeRobot Dataset v3 episode，进入双云训练。

Profile

策略导出后通过 AI Hub / QNN / ONNX Runtime QNN 生成 Qualcomm edge evidence。

Publish

SkillCertKit 通过后进入 SkillDock，EdgeFleet 灰度安装；异常时生成 incident capsule 并回滚。

Ask

向高通要什么

6-8 周联合验证 sprint：RB3 Gen 2/QCS6490 target、AI Hub/QNN/QAIRT office hours、Robotics Hub 发布指导、IQ10 RRD roadmap 指导，以及 5-10 个 OEM/SI/开发者访谈。

Outcome

比赛交付物

一个 `Dragonwing-profiled candidate` 应用卡、一个可运行 LabForge 技能包、一个可复现数据/训练/边缘部署链路、一个开发者 marketplace 原型和一套商业定价模型。

RobotMac 解决“机器人怎么稳定运行”，RobotAppLayer 解决“开发者怎么在机器人上发布可收费应用”。
