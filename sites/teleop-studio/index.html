Pitch 24 · Demonstration Data Factory

TeleopStudio

云训练之前，先要有可信数据。TeleopStudio 把遥操作、相机同步、episode 记录、数据质检、LeRobot dataset adapter 和云端交接做成可购买的数据采集工位。

查看采集工位 连接 DataFlywheel

01 · Capture Station

机器人平台的第一台机器，应该是数据生产机器。

TeleopStudio 不是简单录屏工具，而是标准化采集工位：操作员输入、机器人执行、双相机同步、边缘缓存、episode 时间轴和云端上传都在同一张工作台上完成。

Operator

手柄、leader arm、键鼠、手机或 3D mouse，优先选择比赛阶段最稳输入。

Capture Cell

机械臂、托盘、固定光照、相机支架、标定板和 reset tray。

Edge Recorder

Qualcomm edge box 记录相机、动作、状态、时间戳和本地缓存。

Dataset

自动生成 LeRobot-compatible episode、任务说明和 robot profile。

Cloud Handoff

中国云或海外云 adapter 接收数据，启动 CloudTwin 训练队列。

02 · Data Quality Gates

不是每条示教都应该进入训练集。

商业客户不会接受“多录一点再说”。TeleopStudio 在采集时就检查画面、动作、结果、重置、接管和权限，让数据在进入 GPU 训练前先具备最低可信度。

Sync

相机帧、关节状态、夹爪状态和 action stream 对齐。

Visibility

检查样品、末端执行器、目标位和关键动作是否被遮挡。

Outcome

每个 episode 标注成功、失败、接管、异常和重置原因。

Motion

识别过度抖动、动作跳变、速度越界和非任务动作。

Lineage

保留操作员、设备、任务、场景、版本和客户数据边界。

Privacy

本地脱敏、私有数据隔离和企业项目权限管理。

03 · LeRobot Contract

把人类示教转成云训练能直接消费的数据合约。

TeleopStudio 的输出不是一堆视频文件，而是带 metadata、robot profile、任务描述、成功标签、失败样本和版本号的 LeRobot-compatible dataset。

  • 每个 episode 绑定 robot profile、相机配置和 action schema。
  • 每个任务绑定自然语言描述、起点、目标、重置协议和验收标准。
  • 每个 dataset 绑定客户权限、版本记录、训练 job 和回滚路径。
  • 失败片段自动回流，成为下一轮 DataFlywheel 的最高价值样本。

04 · Qualcomm Edge Value

数据采集工位也应该强化 Qualcomm 的必要性。

TeleopStudio 让 Qualcomm 参与训练闭环的第一步：低延迟预览、相机同步、本地缓存、安全状态、边缘预处理和断网可运行，都发生在机器人本体侧。

Low Latency 操作员需要稳定预览和即时反馈，不能把控制闭环交给不稳定公网。
Sensor Sync 多相机、关节状态、夹爪状态和动作流在边缘侧统一记录。
Private Data 企业客户可以先本地缓存、脱敏、筛选，再决定是否上云训练。
Skill Path 数据进入 CloudTwin、EdgeRuntimeBench、SkillCertKit 和 SkillDock。

TeleopStudio 把“我们录了一些示教”变成“客户拥有一台持续生产机器人技能数据的机器”。
