Pitch 24 · Robot-Learning Episode Factory

TeleopStudio

录一段机器人视频不是数据资产。TeleopStudio 把遥操作、相机同步、动作状态记录、失败接管、质量门禁和 LeRobot dataset export 做成可购买的示教数据工位，为 CloudTwin、TrainRouter 和 Qualcomm edge skill release 提供干净输入。

从数据问题开始 连接 TrainRouter

01 · Problem

机器人团队不缺 demo，缺的是可训练、可复现、可验收的 episode。

相机不同步、reset 不一致、失败没标注、动作流丢帧、操作者差异过大，最后 CloudTwin 拿到的不是数据资产，而是一堆不可比较的文件。后面的云训练再强，也救不回来坏数据。

痛点 1

录制流程靠人记

任务说明、起点、目标、reset、变化维度和成功标准没有写进 session，换一个操作员就变成另一个数据分布。

痛点 2

视频和动作不同步

多相机、关节状态、夹爪状态、base motion、teleop action 和安全事件没有统一时间轴，训练时无法判断动作原因。

痛点 3

失败样本被浪费

人工接管、碰撞前停止、遮挡、动作抖动和 reset 失败常常被删掉，错过了最有价值的 HIL recovery 数据。

痛点 4

训练前没有验收

数据没有 frame count、timestamp skew、metadata、visibility、action shape 和 privacy gate，就会把 GPU 预算浪费在坏数据上。

02 · Why Now

机器人学习的竞争正在从模型 demo，转向真实数据供应链。

LeRobot 已经把真实机器人 teleoperate、record、replay、train 和 evaluate 变成开发者可用工作流；ALOHA、Mobile ALOHA、DROID 和 Open X-Embodiment 证明了高质量示教数据的价值；Qualcomm Dragonwing 和 AI Hub 则给端侧采集、预处理和部署证据提供了硬件落点。

开源时机

LeRobot 给了统一出口

TeleopStudio 不替代 LeRobot，而是把现场 session、operator workflow、质量门禁和 provenance 接到 LeRobotDataset v3。

数据时机

真实 episode 成为稀缺供给

DROID、Open X-Embodiment 等项目说明行业正在向大规模、多机器人、可复用的真实轨迹数据靠拢。

边缘时机

采集端也需要边缘 AI

低延迟预览、多相机同步、本地缓存、隐私过滤和质量检查都发生在机器人旁边，天然适合 Qualcomm edge box。

03 · Solution

把遥操作 session 变成一条可审计的数据生产线。

TeleopStudio 的核心洞察是：真正稀缺的不是又一个遥操作 UI，而是合格的人机交互 episode 供应。每条 episode 都要有任务、相机、动作、状态、操作员、reset、成功/失败、接管、hash、region 和可训练性评分。

Plan

选择任务、robot profile、variation plan、操作员、相机、reset 协议和数据地域。

Capture

同步记录多相机、state/action、teleop input、时间戳、安全事件和本地缓存。

Gate

自动拒绝遮挡、丢帧、metadata 缺失、动作跳变、过短 episode 和隐私违规。

Export

输出 LeRobotDataset、MCAP/ROS bag、dataset card、manifest 和训练 job contract。

Loop

进入 TrainRouter 训练，失败和接管片段回流为下一轮高价值示教样本。

04 · Product

第一版不追求万能，先把 50 条干净 episode 做成产品。

从桌面机械臂样品转移、扫码、pick-place 这类窄任务开始，像早期 marketplace 一样先手工把供给标准化。目标不是录最多数据，而是提高每小时 valid episode yield。

  • 操作员工位：手柄、leader arm、SpaceMouse、键鼠或移动端输入。
  • 采集单元：固定光照、相机支架、标定板、reset tray 和任务道具。
  • Qualcomm Edge Recorder：低延迟预览、时间戳、本地缓存、断网保护和边缘质量检查。
  • LeRobot Adapter：输出数据集、dataset card、训练命令和 CloudTwin handoff。

05 · Market & Business Model

客户买的不是遥控器，而是更快得到可训练数据。

首批客户不是所有机器人公司，而是正在把 demo 变成交付的团队：机器人初创、AI manipulation lab、OEM、系统集成商、高校课程和企业 4-8 周试点。IFR 2025 的工业机器人和服务机器人数据说明市场已有机器人存量，缺的是把机器人变成持续学习资产的工作流。

Developer / BYO

软件订阅

面向已有机器人团队，提供 teleop UI、recorder、dataset export、ROS 2/Python adapter 和基础 QA。

Research Station

单臂/双臂工位

面向实验室和初创团队，按工作站销售硬件、相机、标定、采集软件、支持和数据质量报告。

SI Pilot

集成商试点

4-8 周固定交付包，把客户现场任务采成数据集，并接入 CloudTwin / TrainRouter 训练。

Enterprise Lane

企业私有部署

私有存储、权限、审计、数据不出域、操作员管理、质量门禁和 China / Overseas 双版本。

06 · Competition & Moat

我们不做第 N 个遥操作工具，而做机器人学习的数据运营层。

LeRobot 是开源格式和训练底座；Trossen/ALOHA、GELLO、Quest/SpaceMouse 是硬件和输入方式；Formant、Viam 是更宽的 fleet ops；Foxglove、Rerun 是数据可视化和日志；NVIDIA Isaac/GR00T 强在仿真与合成数据。TeleopStudio 的位置是：真实世界 human-in-the-loop session 到 model-ready dataset 的质量控制层。

短期壁垒

任务模板和 QC gate

每个任务都有 variation plan、reset protocol、相机配置、成功标准、拒绝规则和 dataset card 模板。

中期壁垒

operator yield 和 failure taxonomy

记录每小时有效 episode、失败原因、接管类型、reset 摩擦和操作员质量，持续提高数据产出效率。

长期壁垒

skill release graph

追踪哪些任务、相机、硬件、数据质量和 policy 组合能稳定部署到哪些 Qualcomm edge target。

07 · Why Qualcomm

Qualcomm 不只应该是推理目标，也应该成为数据原点。

TeleopStudio 让 Dragonwing / RB3 / QCS edge board 站在机器人学习闭环第一步：相机接入、时间戳、低延迟预览、边缘预处理、隐私过滤、断网缓存和数据 lineage 都从端侧开始。这样后续 AI Hub、QNN/QAIRT 和 SkillDock 才有可信输入。

Record 多相机、state/action、teleop input 和安全事件在端侧形成 episode manifest。
Check 端侧检测模糊、遮挡、丢帧、动作抖动、隐私风险和本地存储状态。
Train 合格数据进入 CloudTwin 和 TrainRouter，按 China / Overseas lane 训练。
Deploy 训练结果回到 AI Hub / QNN / QAIRT / ONNX Runtime QNN 证据链。
需要高通支持

多相机与时间戳验证

确认 RB3 / Dragonwing 采集链路、编码、预览、同步和断网缓存的推荐实现。

需要高通支持

AI Hub / QNN 导出路径

把由 TeleopStudio 采集的数据训练出的一个策略组件，跑通 compile/profile/deploy 证据。

需要高通支持

Robotics Hub 样例

把“LeRobot capture to Qualcomm edge deployment”做成可复用开发者教程和比赛展示资产。

08 · Demo & Evidence

3 分钟演示证明：这不是机械臂表演，而是数据工厂。

评委需要看到端到端证据：一次真实遥操作如何变成可训练 episode，坏数据如何被拒绝，接管如何变成高价值样本，最后如何交给 TrainRouter 和 Qualcomm edge release。

0:00-0:45

连接工位

选择 robot profile、任务、三路相机、operator、variation plan、reset protocol 和数据地域。

0:45-1:30

采集 episode

同屏展示相机预览、action/state timeline、timestamp skew、FPS、dropped frame 和安全状态。

1:30-2:15

拒绝坏数据

故意制造遮挡或丢帧，QC gate 拒绝该 episode；人工接管片段变成 HIL recovery 样本。

2:15-3:00

导出训练合约

生成 LeRobot dataset card、manifest、TrainRouter job 和 Qualcomm target profile。

TeleopStudio 的一句话：把“我们录了一些示教”变成“客户拥有一台持续生产机器人技能数据的机器”。
