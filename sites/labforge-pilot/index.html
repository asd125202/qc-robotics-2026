LabForgePilot

01 · Problem

实验室最贵的浪费，不是一次拿放，而是样本记录和物理动作断裂。

中小型研发、QC、诊断和转化实验室里，大量样本转移仍靠人手、条码枪、Excel、LIMS/ELN 补录和事后复核。真正的成本是慢、不一致、错样、漏扫、夜间等待、失败原因丢失，以及无法证明谁在何时把哪个样本转到了哪里。

Manual

人手转移不可规模化

拿、扫、放、拍照、核对和等待占据实验员时间，重复性和夜间运行能力都弱。

Trace

样本链路靠补录

sample ID、source、destination、operator、timestamp、失败原因和 plate map 如果分散记录，审计就不可信。

Learning

失败没有变成数据

夹取失败、孔位偏移、遮挡、错样和人工接管通常被当场修掉，没有进入训练集。

Edge

云端控制不适合实验台

实验台需要本地视觉、本地策略、本地安全停机和断网继续运行，不应把关键动作交给不稳定公网。

Product

低成本机器人仍像玩具

裸机械臂越来越便宜，但缺少夹具、样品模板、数据采集、应用、日志、培训和维护体系。

02 · Current Alternatives Fail

传统实验室自动化很强，但不是这个楔子。

LabForgePilot 不和 Hamilton、Tecan、Beckman、Opentrons 正面争“液体处理精度”。它从更窄的样本物流、扫码、视觉核对、失败学习和边缘证据切入，补上台面 sample movement / staging / verification。

人工 + Excel

便宜但不可复制；GxP 或客户审计下，很难证明样本链路、操作者、失败和修正动作。

Opentrons Flex

开放、低价、适合自动移液；视觉、失败学习、LeRobot 数据、LIMS 证据链仍需要用户自己拼。

Automata / Biosero / Cellario

跨设备编排和整实验室调度很强，但部署、销售和集成较重，更适合成熟自动化团队。

Hamilton / Tecan / Beckman

高可靠液体处理和条码/LIMS 能力成熟；不是围绕低成本机器人、失败回流和 edge AI 数据闭环设计。

myCobot / Dobot / xArm

硬件商品化证明成本下降，但客户买到的是手臂，不是可购买、可培训、可维护的实验台工作站。

大型 SI 项目

能做非标交付，但每个项目重新设计夹具、软件、记录、培训和维护，难以沉淀成商品。

03 · Solution

LabForgePilot 是 RobotMac 平台的第一个商业楔子。

一个桌面样品转移工作站，把 Qualcomm 边缘 AI、LeRobot 数据/策略训练、RobotAppLayer 可安装应用、OpsConnector 写回和 SafetyOps 证据链，压缩进实验台上能购买的产品。

Scan

识别样本管、孔板、目标位、条码、颜色标记、AprilTag 和工作台状态。

Record

用 teleop / leader arm 采集 LeRobot v3 episode，绑定 sample ID、rack slot、target slot 和 operator。

Train

用 ACT 做可靠 baseline，再用 HIL / DAgger 把失败接管片段变成下一轮训练数据。

Deploy

把视觉模型和策略包 profile 到 RB3 / QCS6490 路线，保留 CPU fallback，不把 NPU 路径当唯一成功条件。

Writeback

完成、异常、样本 lineage、图像证据、模型版本、延迟和 operator 写回 LIMS/Benchling 或审计包。

实验室自动化市场已经存在，但很多小批量、多变化流程仍没有合适的自动化入口。与此同时，LeRobot/ACT/SO-101 把机器人采集和模仿学习门槛拉低，Qualcomm AI Hub/QNN/RB3 给本地边缘推理提供可展示证据。

ACT ACT 是 LeRobot 推荐的新手 baseline，适合用 50 demos 起步做可靠操作策略。
12 TOPS QCS6490/RB3 Gen 2 可作为比赛 MVP edge target，展示本地视觉和策略证据。

05 · Product

第一版只做三件高频事：拿、扫、放。

不要承诺临床诊断、GMP、CLIA、IVD 或高精度湿实验认证。第一版定位 research / education / QC workflow automation，用空管、彩色水、假样本和可验证记录证明平台闭环。

  • Tube-to-plate：从管架取样本，放到目标孔位，拍照确认并生成 plate map。
  • Plate-to-plate：在两个托盘之间搬运样品盒、管架或 mock labware，验证位姿和任务模板。
  • Plate-to-tube：从 staging 位取出样本，放回目标管架，完成扫码、复核和记录。
  • BarcodeVerify：条码与 worklist 不匹配时拒绝执行，生成审计事件。
  • QCPhoto：每个关键动作后抓取图像证据，绑定样本、孔位、策略版本和时间戳。
  • FailureMining：夹取失败、错位、遮挡、drop、timeout 自动进入失败片段库和 HIL 修正队列。

06 · Product API

每次转移，都是一条可追溯、可训练、可回放的数据。

LabForgePilot 的数据对象直接兼容 LeRobot v3，又加上实验室语义：样本、孔位、操作者、任务、失败、策略版本和边缘 profile。这样它不是一次机械臂脚本，而是能越跑越好的 workcell。

Episode

一次完整尝试，含 reset、teleop、policy、HIL、success/failure、front/top video 和 joint/action parquet。

Task

`transfer_sample_tube_from_rack_to_slot`，带自然语言、sample ID、rack slot、target slot 和 worklist。

Variation

rack 位置、目标槽、光照、管子朝向、背景扰动、遮挡和随机布局，证明不是单脚本。

Failure Clip

从 rollout ring buffer 保存失败前后 10-30 秒，标记 miss grasp、wrong slot、drop、collision、timeout。

Policy Card

ACT/SmolVLA checkpoint、dataset hash、camera config、训练步数、success@20、interventions/min 和 latency。

Edge Profile

p50/p95 延迟、control FPS、QNN/CPU/GPU path、memory、model hash、device id 和 runtime version。

08 · Competition & Moat

壁垒不是机械臂，而是实验室样本转移数据资产。

硬件会商品化，传统自动化会继续服务高通量液体处理。LabForgePilot 的壁垒在条码、视觉、轨迹、失败、HIL、LIMS 语义、边缘 profile 和审计证据之间的绑定。

Labware Templates

管架、孔板、样品盒、检测槽、夹具和灯箱模板，让新部署从配置开始而不是从机械设计开始。

Failure Dataset

每个错夹、掉落、遮挡、错位和人工接管都能训练下一版策略。

Audit Semantics

sample ID、operator、timestamp、source/destination、worklist、image evidence 和 policy version 绑定。

Robot Apps

TubeRackTransfer、PlateStaging、BarcodeVerify、QCPhoto、LoadUnload 可进入 RobotAppLayer / SkillDock。

Edge Evidence

QNN profile、latency、memory、model hash、device ID 和 local runtime logs 成为 Qualcomm 差异化证据。

Channel Loop

学校、SI、OEM、科研平台和 QC 实验室共同沉淀课程、模板、应用和维护手册。

09 · Why Qualcomm

比赛必须证明：这不是云端遥控，而是可量化的实验台边缘部署。

LabForgePilot 用 Qualcomm Dragonwing 边缘平台在本地完成视觉识别、策略执行、证据记录和运行指标。云端只负责模型准备、AI Hub 编译/profile、可选 OTA 和双云训练。

  • RB3 Gen 2 / QCS6490：比赛 MVP 平台，展示 Linux/Ubuntu、相机、USB/以太网/GPIO、AI vision 和 12 TOPS NPU 路线。
  • QCS8550：高性能扩展 profile，适合多摄像头、多模型并行和更强 on-prem robot operations box。
  • AI Hub / QNN / ONNX Runtime QNN：生成 compile/profile/inference evidence，记录 runtime、latency、memory、compute unit 和 fallback 状态。
  • Qualcomm Linux / Ubuntu：承接 camera/video/sensor overlays、ROS2、GStreamer、ONNX、QNN、容器和本地服务。
  • IQ10 RRD：只写 roadmap / early access ask；截至 2026-07-06 不写成已量产或已集成硬件。
