01 · Runtime Stack

开发板能跑起来，不等于机器人可以交付。

RobotCoreOS 把板级支持、机器人运行时、模型执行、安全服务、数据回流和远程运维压进一个 golden image。客户买到的是可启动、可验证、可更新的机器人底座。

BSP

启动、内核、驱动、相机、网络、IO 和加速器 profile。

Runtime

ROS 2 bridge、设备发现、时间同步、动作接口和日志。

Policy

运行 CloudTwin 导出的策略包，记录延迟和失败事件。

Manifest

记录 SoC、OS、QNN / QAIRT、模型、传感器和性能 profile。

Fleet

设备身份、OTA、回滚、安全服务、客户归属和运维状态。

02 · Golden Image

像卖电脑一样卖机器人，系统镜像必须成为产品。

RobotCoreOS 不是某个单独脚本，而是从工厂烧录、首次开机、设备绑定、驱动验证、模型安装到远程运维的完整产品流程。

Factory

烧录 golden image，绑定硬件 profile 和出厂验证记录。

Provision

首次开机完成设备身份、客户项目、网络和权限绑定。

Validate

自动检查相机、IO、机械臂、传感器、NPU/GPU 和日志。

Install

安装通过 SkillCertKit 的技能包和 runtime dependency。

Operate

本地运行策略、安全状态机、episode 日志和失败回流。

Maintain

通过 EdgeFleet 做更新、回滚、诊断和现场恢复。

04 · OTA And Rollback

机器人更新不能赌运气。

模型、技能、驱动和系统镜像都可能更新。RobotCoreOS 把 signed OTA、A/B rollback、recovery mode、staged rollout、SBOM 和审计日志做成默认机制，避免一次更新毁掉客户现场。

  • 比赛 demo lane：Ubuntu 24.04 + ROS 2 Jazzy + Qualcomm board support。
  • 生产 lane：Ubuntu Core 或 Yocto + Mender / RAUC / SWUpdate / OSTree。
  • 系统镜像、机器人应用、技能包和客户数据分层管理。
  • 更新失败或性能退化时自动回到上一稳定版本。
