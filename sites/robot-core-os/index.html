Pitch 26 · Qualcomm-first Robot Runtime OS

RobotCoreOS

面向 Qualcomm Dragonwing/ARM 机器人的生产运行层：把 ROS 2 原型、LeRobot 数据闭环、AI Hub/QNN 端侧推理、签名 OTA、回滚、设备身份和 release evidence，打包成可继承的商业化底座。

进入路演 连接 RobotMac Core

01 · Problem

机器人行业不缺 Demo，缺的是从 Demo 到客户现场队列的生产底座。

每个团队都在重复做 OS image、BSP、驱动、ROS 2 lifecycle、相机/NPU 配置、日志、远程诊断、OTA、回滚、证书、SBOM、模型发布门禁和企业审计。买方担心的不是算法不够酷，而是 20 台、100 台、1000 台机器能不能被长期运营。

版本不可追溯

现场到底跑哪个 kernel、driver、ROS graph、模型、技能包和参数，常常没有一份可信记录。

更新像赌博

模型、驱动、镜像和配置都可能破坏任务。没有灰度、健康门禁和 A/B 回滚，OTA 会变成现场风险。

边缘 AI 难交付

云上训练出的 policy 要落到本体，还要处理 QNN/QAIRT、输入规格、fallback、温度和功耗。

数据闭环断裂

失败片段、人工接管、低置信度和训练 checkpoint 没有统一 lineage，模型迭代无法解释。

买方怕运维

企业客户要 uptime、审计、恢复、责任边界、安全发布和 CVE 响应，不只是一段演示视频。

集成商被拖慢

系统集成商每个项目都从镜像、驱动和远程支持重新开始，交付成本吞掉毛利。

02 · Current Alternatives Fail

现有工具都重要，但没有单独解决 Qualcomm 机器人从原型到量产的问题。

RobotCoreOS 不应该声称“发明机器人 OS”。更准确的定位是：把通用 OS、ROS 2、OTA、安全、边缘 AI、数据闭环和 fleet 运维，组装成 Qualcomm 机器人可复用的生产层。

ROS 2

事实开发生态和中间件，但不是 secure product OS、fleet updater、rollback layer 或发布证据系统。

Linux / Yocto

提供 OS/BSP/长期维护基础，但不定义 robot skill、LeRobot lineage、QNN artifact 和现场 incident bundle。

Mender / Balena

证明签名 OTA、A/B 和 rollback 是刚需，但默认不理解 ROS graph、AI model、MCU firmware 和校准 schema。

Viam / Formant

RobOps、telemetry 和 remote control 强，但不是 bootable Qualcomm runtime image，也不负责 board profile。

Foxglove

证明机器人数据平台有付费意愿，但不解决系统镜像、驱动矩阵、QNN release 和 OTA 回滚。

NVIDIA / 中国 OS

Isaac/Jetson 和 M-Robots/openEuler/AGIROS 都说明平台化正在发生，Qualcomm 需要自己的 Dragonwing-first 路径。

03 · Solution

把 Qualcomm 开发板变成可出货机器人，需要一套生产运行层。

RobotCoreOS = Qualcomm board profile + golden image + runtime agent + ROS 2 lifecycle supervisor + LeRobot hooks + QNN/QAIRT policy runner + signed rollout + A/B rollback + evidence pack + fleet identity。

  • 它不替代 ROS 2，而是把 ROS 2 产品化、托管化、安全化。
  • 它不替代 Qualcomm Linux/Ubuntu/Yocto，而是在其上定义机器人 runtime contract。
  • 它不替代 Viam、Formant、Foxglove，而是给这些云层提供可信的本体 endpoint。
  • 它把模型、系统、传感器、QNN profile、OTA 和事故证据绑定到同一份 manifest。

04 · Why Now

机器人、ROS 2、LeRobot、Qualcomm 和合规采购同时进入生产化阶段。

过去做 RobotCoreOS 太早，因为模型、数据、边缘 AI 和 fleet 需求还没有收敛。现在 ROS 2 Jazzy 对齐 Ubuntu 24.04，LeRobot 建立数据和 policy 工作流，Qualcomm 正在构建 Dragonwing 开发者生态，企业采购也开始要求 SBOM、provenance、签名发布和可审计运维。

部署量变大

IFR 报告 2024 年工业机器人新装约 54.2 万台，运行存量约 466.4 万台；软件基础设施价值开始显性化。

ROS 2 成为主线

2025 ROS Metrics 显示下载接近 9.84 亿次，ROS 2 占 91.2%；Jazzy LTS 对齐 Ubuntu 24.04 和 ARM64。

LeRobot 可产品化

Dataset v3、HF Jobs、checkpoint lineage、rollout、DAgger/HIL 和失败片段采集，让数据飞轮可以进入产品层。

Qualcomm 补齐工具链

Dragonwing、AI Hub、QNN/QAIRT、Device Cloud、Profiler、Qualcomm Linux 2.0 和 Robotics Hub 形成端侧 AI 路径。

05 · Product

第一版 wedge：从 Qualcomm 裸板到可注册、可更新、可回滚的机器人 runtime。

RobotCoreOS 的产品价值不是一个镜像文件，而是一条从 hardware bring-up 到 fleet ops 的交付流程。客户拿到的是可启动、可验证、可观察、可恢复的机器人底座。

Flash

烧录 golden image，绑定 SoC、BSP、kernel、driver、传感器和 recovery profile。

Enroll

首次开机生成 device identity，完成证书、客户归属、网络和 fleet enrollment。

Validate

检查 camera、IO、IMU、CAN/UART、急停、NPU placement、ROS lifecycle 和 thermal headroom。

Install

安装通过 SkillCertKit 的 policy package，记录模型 hash、输入输出、权限和回滚包。

Operate

本地执行感知、策略、控制、安全边界、telemetry、episode capture 和日志。

Profile

输出 latency、FPS、memory、accelerator utilization、温度、功耗和 runtime manifest。

Update

对 OS、ROS app、AI model、MCU firmware、校准 schema 和 config 做签名灰度发布。

Rollback

失败启动、性能退化或安全门禁失败时，回到上一稳定版本或安全模式。

Capture

把失败片段、低置信度、人工接管和 safety hit 回流到 LeRobot 数据闭环。

Audit

导出 SBOM、签名、provenance、发布记录、硬件 profile 和模型 lineage。

06 · Product API/Evidence

真正的产品 API 是一份让工程、运维、采购、安全和评委都能读懂的 runtime contract。

每个机器人 release 都必须产出 evidence bundle：board profile、image digest、ROS graph、policy hash、QNN profile、SBOM、provenance、CVE triage、health gate、rollback drill 和 incident replay。

  • Board：SoC、base OS、ROS distro、camera、IO、accelerator、thermal、power、recovery path。
  • Policy：LeRobot/ONNX/QNN_DLC artifact、model hash、inputs、outputs、max rate、safety boundary。
  • Release：SPDX/CycloneDX、SLSA-style provenance、canary rollout、rollback slot、health gate。
  • Telemetry：latency、FPS、memory、power、thermal、accelerator placement、safety hits、rollback cause。
  • Data：MCAP/ROS bag/OTel export、失败 episode、人工接管、checkpoint lineage 和再训练入口。

07 · Market & Business Model

第一批客户不是“所有机器人公司”，而是最怕现场失控的人。

RobotCoreOS 赚的不是整个机器人硬件市场的钱，而是机器人软件基础设施、设备管理、release evidence、fleet ops 和板级 productization 的钱。

机器人 OEM

10-1000 台规模的团队需要可复现 BSP、Linux、ROS、driver、OTA baseline，避免长期养发行版团队。

系统集成商

需要 installable image、known-good stack、远程支持、rollback 和项目交付模板，保护项目毛利。

RaaS 运营商

按月对 uptime 负责，需要 identity、telemetry、proof-of-work、staged rollout 和 incident recovery。

企业 IT / 安全

关心设备身份、补丁 SLA、SBOM/CVE、访问控制、审计日志和 fleet inventory。

教育与开发套件

需要学生和开发者能直接刷机、恢复、运行 ROS/LeRobot 示例，而不是先修嵌入式 Linux。

商业模式

Community 免费入口，Runtime 按设备授权，Fleet 按机器人/月订阅，OEM/SI 收 bring-up，企业版做私有化和 SLA。

08 · Competition & Moat

护城河不是 dashboard，而是 Qualcomm 运行证据和兼容图谱。

每一次 bring-up、失败 OTA、传感器漂移、温度降频、人工接管、技能安装和客户恢复，都会沉淀成下一版 runtime rule、compatibility profile 和发布门禁。

Board Profile QCS6490、QCS8550、IQ 系列、driver、camera、IO、accelerator、thermal、power 和 recovery 矩阵。
Hardware CI SDK、kernel、driver、QNN、ROS 包、policy runtime 每次变化都留下可比较 profile。
Evidence Graph SBOM、provenance、签名、OTA、回滚、health gate、watchdog、audit trail 和 recovery report。
Skill Graph 技能包与硬件、ROS graph、模型 runtime、权限、安全边界和 rollback package 的兼容关系。

RobotCoreOS 让 Dragonwing 从“被使用的一块板”变成机器人技能的默认边缘目标。

Qualcomm 的机会不是只报 TOPS，而是把低功耗、多摄像头、连接、端侧 AI、Device Cloud、AI Hub/QNN、Profiler、Qualcomm Linux 和 Robotics Hub 组合成开发者可以继承的 runtime product。

对 Qualcomm 的价值

每个 demo 都输出 Qualcomm edge profile：latency、memory、FPS/W、thermal headroom、accelerator placement 和 rollback evidence。

生态位置

RobotCoreOS 可以成为 Dragonwing Robotics Hub 的 reference runtime project，而不是只有单点 sample code。

双版本叙事

中国和海外都能讲端侧推理、云端训练、数据驻留、企业私有化和跨云模型 lineage。

10 · Demo & Ask

三分钟证明：这不是 Demo 机器，而是可交付机器。

初赛先用模拟 dashboard、本地样例流程和现有资产证明闭环；复赛拿到开发板后补真实 Qualcomm profile、视频、功耗、温度、延迟和稳定性数据。

Boot

开发板启动 RobotCoreOS console，展示 board profile、image digest、ROS 2 Jazzy baseline、device identity。

Enroll

设备进入 fleet dashboard，自动生成 runtime manifest 和初始 health check。

Run

安装 LeRobot/ACT policy，走 AI Hub/QNN profile，显示 camera、ROS topics、latency、NPU placement。

Fail

推送 crash 或延迟超标的更新，canary health gate 失败，系统停止 rollout 并回滚。

Export

导出 SBOM、provenance、runtime manifest、CVE impact view、failure episode 和评委版 evidence bundle。

Claim Boundaries

边界讲清楚，可信度才会提高。

RobotCoreOS 不是 Qualcomm 官方认证 OS，不替代 ROS 2，不替代安全认证，不保证所有板卡 secure boot/FBE/attestation 开箱可用，也不声称所有 LeRobot/VLA 模型都能直接跑 NPU。它的可 defend claim 是：一套 Qualcomm-first、可签名、可更新、可回滚、可观测、可审计的机器人运行镜像和发布证据流程。

RobotCoreOS 把“这块板能跑 Demo”升级成“这台机器人可以被交付、更新、回滚和长期维护”。
