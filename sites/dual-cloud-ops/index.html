Pitch 16 · LeRobot To Qualcomm Release Layer

DualCloudOps

一个 LeRobot 训练任务，自动选择中国云或海外云，最后生成可验签的 Qualcomm 边缘部署包。我们不卖另一朵 GPU 云；我们卖机器人训练从数据、算力、评估到上板的可审计执行层。

看训练断点 看 8 分钟演示

01 · Problem

机器人团队不是缺 GPU，而是缺一条能交付的训练流水线。

LeRobot、低成本机械臂和云 GPU 让原型更快，但商业客户要的是另一件事：工厂视频能不能留在本地？换云后结果能不能复现？训练花费能不能封顶？模型能不能变成 Qualcomm edge 上可验签、可回滚、可审计的部署包？

Data

真实机器人数据很敏感

视频、音频、地图、工艺流程、人员和动作轨迹可能涉及个人信息、商业秘密、重要数据和客户现场安全。

GPU

算力选择不可预测

大云可信但贵且有 quota；neocloud 快但采购和合规要解释；租卡平台便宜但不适合敏感生产数据。

Repro

训练脚本很难变成交付物

dataset commit、container、LeRobot version、seed、policy config、checkpoint、日志和评估指标经常散在不同系统。

Eval

仿真、真实回放和客户验收断裂

模型 loss 下降不等于机器人任务成功；客户要 success rate、intervention rate、latency、failure replay 和 rollback。

Edge

云端 checkpoint 不是上板部署包

Qualcomm 目标需要 ONNX/QNN/Qualcomm AI Runtime profile、延迟/内存报告、签名、SBOM 和设备配置。

02 · Current Alternatives Fail

GPU 云、MLOps 和机器人 fleet 工具各自强，但没有把机器人策略变成可发布 artifact。

DualCloudOps 不和 AWS、阿里云、RunPod、Lambda、CoreWeave、Run:ai、Weights & Biases、Foxglove 或 Formant 争同一个位置。它站在这些工具之上，定义机器人训练任务、数据边界、评估门禁和 Qualcomm edge release。

Hyperscaler

AWS、GCP、Azure、阿里云、腾讯云可信，但 quota、成本、区域、销售流程和机器人上板链路都要团队自己拼。

Neocloud

Lambda、RunPod、CoreWeave、Modal 很快，但客户仍要决定数据能不能去、如何复现、如何审计和如何交付。

GPU Marketplace

Vast、AutoDL、潞晨云等适合非敏感实验，但敏感 robot video、客户工厂数据和长期 SLA 需要更强分类和隔离。

MLOps

W&B、MLflow、SageMaker、Vertex 追踪实验很好，但不天然理解 LeRobot episode、robot body、动作成功率和 QNN profile。

Fleet Ops

Foxglove、Formant、Viam、InOrbit 收集和观察机器人数据，但不是以 train/eval/export release 为核心。

Self Build

自建脚本最快启动，最慢商业化；一旦进入中国/海外双市场、客户审计和边缘量产，维护成本迅速上升。

03 · Solution

DualCloudOps 用一份 Robot Training Contract 连接双云训练和 Qualcomm edge release。

同一个 LeRobot job 可以路由到中国云、海外云、企业专有云或低成本实验云；但数据合约、训练配置、评估门禁、artifact 签名和 Qualcomm 设备 profile 保持一致。

Contract

数据集、policy、预算、区域、隐私等级、robot body、Qualcomm target 和验收指标。

Route

按数据敏感度、区域、成本、GPU 可用性和企业采购策略选择中国或海外 provider。

Train

生成 provider runner，固定 container、LeRobot version、seed、dataset commit 和 checkpoint cadence。

Gate

用 sim、real rollout、failure replay、latency、intervention rate 和 rollback 规则做 release decision。

Ship

输出 ONNX/QNN candidate、runtime config、profile report、SBOM、provenance、签名和回滚包。

04 · Why Now

LeRobot 开源动量、GPU 供应碎片化和边缘 AI 上板需求同时成熟。

2026 年的机会不是“训练一个更大模型”，而是把机器人策略训练变成可重复、可采购、可审计、可上板的工业流程。云 GPU 访问已经不是稀缺能力；被治理的 GPU 路由才是商业化能力。

LeRobot 数据集、teleoperation、ACT/SmolVLA、rollout、sim eval 和 Hub workflow 形成可用开源底座。
双市场 中国客户要数据驻留、ICP/等保式材料和国内云采购；海外客户要 SOC2-style evidence 和已有 MLOps 集成。
碎片化 AWS/GCP/Azure 可信但重；Lambda/RunPod/Modal 快；CoreWeave 适合大规模；Vast/AutoDL 适合低敏实验。
边缘化 机器人实时控制不能绕过本体；云端训练必须回到 Qualcomm edge 的低延迟、低功耗和离线执行。
证据化 客户、评委、保险和质量团队需要 dataset hash、model hash、SBOM、签名、profile 和可回滚 release。

05 · Product

把 LeRobot 原型变成跨区域、跨 provider、可上板的训练执行系统。

第一版不追求接入所有 GPU 云，而是把一个 bounded robot task 跑完整：采集 episode，生成 LeRobotDataset，选择中国/海外 lane，训练 ACT/SmolVLA，跑 sim/real eval，再导出 Qualcomm edge release package。

  • Training Contract Builder：dataset、policy、robot body、camera schema、预算、区域、隐私等级、目标芯片和验收指标。
  • Provider Router：阿里云/腾讯云/华为云/百度、AWS/GCP/Azure、Lambda/RunPod/CoreWeave/Modal、Hugging Face Jobs。
  • Cost & Risk Guard：训练前预算、训练中 spend cap、spot/preempt retry、data egress rule、敏感数据 provider blocklist。
  • Eval Harness：固定 split、sim benchmark、real rollout、intervention labels、failure replay、latency 和 rollback gate。
  • Edge Release Exporter：ONNX/QNN candidate、Qualcomm AI Hub / QNN profile、SBOM、SLSA/in-toto provenance、cosign signature。

06 · Product API

卖点不是接了几朵云，而是每次训练都有合同、账本和 release evidence。

DualCloudOps 的数据结构把 GPU 供应商变成可替换资源，把 robot policy release 变成可审计资产。它可以接入 W&B、Hugging Face Hub、企业对象存储和 Qualcomm AI Hub，但核心记录留在我们自己的 release ledger。

TrainingContract

dataset_repo、policy_type、robot_body、sensors、privacy_tier、region_lane、budget_cap、target_device。

DatasetLedger

LeRobotDataset v3 schema、episode ids、rights、redaction、calibration hash、firmware hash、train/val/test split。

ProviderRun

cloud、region、GPU SKU、container digest、LeRobot version、git SHA、seed、checkpoint cadence、spend。

EvalGate

sim success、real rollout、intervention rate、latency p95、failure class、rollback target、release decision。

EdgeProfile

Qualcomm target、ONNX/QNN artifact、runtime、CPU/NPU fallback、memory、thermal envelope、offline mode。

SignedRelease

model hash、dataset hash、SBOM、provenance、signature、attestation note、install manifest、rollback receipt。

07 · Market & Business Model

先服务从 LeRobot demo 走向客户 pilot 的机器人团队。

第一批买家不是大模型实验室，而是机器人 OEM、系统集成商、具身智能实验室、制造业自动化团队、教育套件和 Qualcomm 生态开发者。他们需要的是“capture, train, evaluate, export”的 4-8 周交付闭环。

中国版

数据驻留 + 国内云训练 + 上板证据

销售给机器人 OEM、实验室、SI 和制造客户。开发者版：训练模板 + GPU 代付/抽成。团队版：人民币 3k-20k / month + compute pass-through。企业/私有化：人民币 100k-800k / year，包含国内云 adapter、SSO、审计、对象存储、内网 runner 和本地发票/支持。

海外版

LeRobot-To-Edge Release SaaS

开发者/教育版：starter credits、dataset conversion、basic eval report。团队版：$499-$5k / month + compute pass-through。SI pilot package：$20k-$80k / 4-8 weeks。Enterprise lane：$100k-$500k / year for SSO/RBAC、private registry、audit logs、provider policies and support SLA。

08 · Competition & Moat

Provider adapter 很容易复制；机器人 release evidence graph 不容易复制。

DualCloudOps 的壁垒不是“我也能开一台 H100”，而是每次训练和部署留下的机器人上下文：哪个 robot body、哪个 sensor schema、哪个任务、哪个数据区域、哪个失败模式、哪个 Qualcomm target、什么延迟和功耗。

Episode Ledger

真实机器人 episode、失败、接管、权利、区域、传感器和标定形成可复用的数据资产。

Eval Harness

固定 split、sim/real 指标、回放标签、regression gates 和 release decision 形成行业默认评测。

Edge Graph

policy type + robot body + sensor config + Qualcomm target + latency/power/runtime evidence 形成兼容性图谱。

Dual-Region Playbook

中国和海外 provider 的数据边界、预算上限、artifact 移动规则、发票和支持语言沉淀成 playbook。

Distribution

LeRobot 社区、Qualcomm 机器人生态、教育套件、SI pilot 模板和云市场形成低成本获客入口。

Procurement Evidence

每个 policy release 都带 lineage、cost、eval、safety gate、rollback、SBOM 和 edge profile。

09 · Why Qualcomm

DualCloudOps 把全球 GPU 云的不确定性，转化成 Qualcomm edge 的确定性需求。

如果开发者只在云端训练，Qualcomm 只是最后一个可选目标。DualCloudOps 让每个 LeRobot job 从一开始就声明 Qualcomm target，并以 AI Hub/QNN profile、延迟/内存报告、签名包和回滚包结束。

Pull-Through

中国云、海外云、neocloud 和 HF Jobs 的训练结果都导向 Qualcomm edge package。

AI Hub

模型编译、量化、profile 和 hosted-device validation 成为训练 job 的标准最后一步。

QNN Runtime

把 PyTorch/ONNX checkpoint 转成 Qualcomm AI Runtime/QNN 候选 artifact，并记录 fallback 策略。

FAE Demo

同一份 LeRobot job，切换中国/海外云，最后上同一块 Qualcomm board，适合销售和比赛演示。

Ecosystem

机器人 OEM、SI、教育硬件、云服务和开发者围绕 Qualcomm target 形成默认交付路径。

10 · Demo & Ask

8 分钟演示：同一份 LeRobot job，两条云路径，一份可验签边缘包。

Demo 不需要真的训练大模型。它要让评委看到一个真实商业闭环：数据边界、云路由、训练日志、评估门禁、Qualcomm profile、签名 release 和本体部署验证。

0-2m 选择 SO-101/桌面臂任务、LeRobotDataset、ACT/SmolVLA、预算、数据等级和 Qualcomm RB3/IQ10 target。
2-4m 切换中国云/海外云；系统展示数据驻留、provider runner、GPU、dataset hash、日志和成本守卫。
4-6m 训练完成后跑 sim/real eval，选择通过 gate 的 checkpoint，导出 ONNX/QNN candidate 和 profile report。
6-8m 生成 signed edge package：SBOM、provenance、cosign signature、install manifest、rollback receipt，并在边缘端验证。

对 Qualcomm 的请求：提供开发板、AI Hub/QNN profile hook、云训练额度和一个 OEM/SI 试点，让 LeRobot 社区第一次把“云端训练”自然落到 Qualcomm 机器人边缘。

DualCloudOps 的核心不是替客户省几块钱 GPU，而是让机器人策略训练从实验脚本变成可采购、可审计、可上板、可复用的商业交付流程。
