00 · Five-Thread Research

施工机器人已经商业化，但最强机会不是“自动盖楼”，而是现场证据和学习闭环。

Dusty、HP SitePrint、Rugged Robotics、Built Robotics、Canvas/JLG、Hilti Jaibot、TyBOT、Spot/DroneDeploy 说明：成熟市场集中在窄任务、可验收 KPI 和重复场景。SiteLedger 把这些任务连接到 BIM、进度、质量、安全、付款和训练数据。

Layout

放线机器人已经证明客户愿意为 BIM/CAD 到现场的准确转换付费。

Reality Capture

每日 360、点云、路线、照片和模型对比正在成为企业级施工数据流。

China Wedge

中国版先做实测实量、巡检和放线辅助，避开民营商品房周期压力。

Payment Proof

业主、总包和分包需要 work-in-place 证据，而不是更多未结构化照片。

Learning Loop

人工复核、低置信度和误报都变成 LeRobot episode，现场越跑越懂。

01 · Project Ledger

交付物不是视频流，而是项目可用的证据账本。

工地每天都在变化。SiteLedger 把机器人巡检路线、位置、图纸版本、照片、检测结果、人工复核和责任分包连成可以写回 BIM/CDE/工单系统的结构化记录。

Capture

多相机、IMU、里程计、RTK、LiDAR/雷达采集路线、照片、点云和位置。

Compare

把现场观测与 IFC、CAD、PDF、点云、计划工区和控制点对齐。

Flag

标出通道阻塞、错位、缺失检查点、护栏/洞口、材料堆放和进度差异。

Review

工程师确认、驳回或修正，生成 BCF-style issue 和每日实录。

Learn

复核片段进入 LeRobot dataset，再部署到 Qualcomm edge。

02 · Edge + BIM + LeRobot Stack

用 Qualcomm 做边缘大脑，用 BIM/CDE 做工地事实源，用 LeRobot 做学习闭环。

SiteLedger 不把云端放进安全关键闭环。机器人本地完成感知、定位、安全降级和任务缓存；云端负责训练、评估、项目报告和非实时同步。

  • Safety MCU：急停、制动、bumper、watchdog、motor inhibit 和硬 geofence。
  • ROS 2 / Nav2：路线、costmap、keepout、定位融合、teleop 和任务状态。
  • BIM/CDE：IFC、openCDE、BCF、图纸版本、点云和责任工区。
  • LeRobot：episode、HIL、训练、评估、模型卡、灰度发布和回滚。

03 · First Workflows

第一批任务选择低操纵风险、高证据价值的工作。

SiteLedger 不从重型无人施工机械开始，而从工地每天都要做、且容易用数据验收的任务开始。后续可以连接放线机器人、喷涂机器人、钻孔机器人和巡检机器人。

放线复核

验证控制点、墙线、门洞、设备基础、机电点位和图纸版本。

实测实量

记录偏差、照片、位置、责任分包、闭环状态和验收证据。

进度实录

按工区/构件记录 work-in-place，支持支付、争议和业主报告。

安全观察

通道阻塞、临边洞口、材料堆放、受限区进入和复核提醒。

材料定位

识别材料区、设备、托盘、二维码和错放位置，减少现场寻找。

机器人数据

把路线失败、误报、低置信度和人工接管变成训练样本。

04 · China / Overseas Versions

中国版讲智能建造示范，海外版讲 layout、progress、QA 和 payment evidence。

同一套底层技术可以服务两种市场语言。中国版强调政策、央国企、公共项目、本地云和数字档案；海外版强调工期、返工、劳动力短缺、业主证据和 SaaS/RaaS 计费。

中国智能建造版

面向试点城市、央国企、公共建筑、轨交机场、医院学校、城市更新和产业园。交付实测实量、巡检、放线辅助、数字档案、本地云、中文工单和数据不跨境策略。

海外项目证据版

面向 GC、subcontractor、owner、layout crew、safety team 和 rental channel。用 6-8 周试点验证 sq ft/day、issues closed、payment support、crew-hours 和 rework risk。

05 · Competition Demo

用小车和模拟工地，三分钟讲清现场数据闭环。

演示不需要重型机械。用小型机器人巡检一个 mock 工地：楼层图、锥桶、梯子、材料堆、错位标记和受限区。机器人发现低置信度事件，人工复核，生成 issue，保存 LeRobot episode，再展示 Qualcomm edge 部署证据。

IFC 模型、图纸、控制点和现场坐标必须版本化。
BCF 每个问题都链接位置、照片、构件、责任人和复核状态。
HIL 人工修正误报和漏报，形成可训练 episode。
Edge 断网、RTK 丢失和图纸过期时，本体安全降级。

06 · Why Qualcomm

施工现场是边缘 AI、连接和低功耗机器人平台的天然压力测试。

工地有粉尘、弱网、临时照明、临时墙体、频繁变化的路线和安全边界。Qualcomm 的价值不是单次推理，而是多传感器、实时视频、低功耗、连接、安全身份、AI Hub profile 和长期量产路线。

Vision

多相机、GStreamer、视觉预处理、本地检测和场景理解。

Fusion

IMU、轮速、RTK、VIO、LiDAR/雷达和定位协方差融合。

Runtime

ROS 2 / Nav2 / AI Hub / QNN / TFLite / ONNX Runtime 路线。

Network

5G / Wi-Fi / private network 支持远程复核和 store-and-forward。

Trust

模型、地图、图纸、日志、issue 和训练数据都有版本证据。

SiteLedger 的商业承诺不是“无人施工”，而是把每一次巡检、复核、进度、偏差、误报和接管变成项目可用的证据与下一轮训练数据。它让施工机器人从单点工具进入项目级平台。
