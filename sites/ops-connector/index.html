Pitch 27 · Edge-Verified Robot Workflow Connector

OpsConnector

把 WMS/MES/ERP/LIMS/SCADA/CMMS 里的业务请求变成可执行、可审计、可复用、可写回的机器人工作流包。机器人已经有 fleet manager；企业缺的是 workflow connector。

看集成断点 看写回演示

01 · Problem

企业机器人部署失败，通常不是因为机器人不会动，而是因为机器人动作没有变成企业记录。

WMS、MES、LIMS、SCADA、ERP 需要任务来源、审批、异常、验收、写回和责任边界；机器人系统通常只返回 `mission done`。这就是为什么很多试点能跑，规模化却变成现场脚本、人工补录和项目制接口。

Warehouse

WMS 不会直接控制机器人

WMS 管订单、库存、波次、库位和报表；机器人需要秒级位置、路径、拥堵、充电、工位容量和异常恢复。

Lab

机器人动作没有变成 lineage

扫码、交接、称重和样本转移如果不写回 LIMS/ELN，就会破坏 sample lineage、chain of custody 和审计。

Factory

工单和设备状态断裂

MES/eBR 需要材料、人员、设备、批次、质量和偏差；机器人平台通常不会自动生成 eBR 或 CMMS work order。

Exception

异常靠人工补救

缺货、条码不符、温度越界、工位满、门禁未放行、机器人阻塞和人工接管经常掉进聊天、电话和 Excel。

Scale

每个现场重做集成

换一个 WMS、机器人品牌、流程、样本规则或安全区，项目就重新写接口，无法复制到下一站点。

02 · Current Alternatives Fail

现有方案各有价值，但没有拥有“企业任务 -> 机器人执行 -> 异常处理 -> 审计写回”的闭环。

OpsConnector 不替代 WMS、MES、LIMS、SCADA、robot fleet manager 或 SI。它把最难复用的业务语义、状态、异常、权限和审计写成可复制的工作流合约。

SI 项目制

能让一个现场上线，但接口、异常和写回逻辑沉没在项目代码里，下一站点继续卖人天。

Robot OEM Fleet Software

Locus、GreyOrange、MiR、Geek+、Hikrobot 等能调度自家场景，但通常优先服务自家 WES/WCS/RCS。

InOrbit / Formant / Orbit

验证了 robot ops、workflow AI、inspection-to-work-order 需求；OpsConnector 应集成它们，而不是替代。

WMS / MES 扩展

SAP、Manhattan、Blue Yonder、Siemens、国产 WMS/MES 能发任务，但不应承担机器人实时执行和 OT 安全。

iPaaS

MuleSoft、Boomi、Workato 懂 SaaS API；不懂 robot state、mission、safety、latency、on-prem edge、PLC。

Standards

VDA 5050、MassRobotics、Open-RMF、OPC UA 是基础，但不覆盖订单语义、样本 lineage、审批和商业 SLA。

03 · Solution

OpsConnector 是企业系统和机器人执行层之间的 workflow contract compiler。

核心原则：workflow-first, adapter-second. 先定义业务意图、验收、审批、异常和写回，再选择 RobotAppLayer、EdgeFleet、Open-RMF、VDA 5050、OPC UA 或厂商 API。

Intent

WMS 补货单、cycle count、MES 工单、LIMS 样本请求、SCADA interlock、CMMS work order 或人工审批。

Contract

生成 TaskContract、ObjectMap、ApprovalGate、AdapterProfile、idempotency key、acceptance rule 和 trace context。

Execute

Dragonwing edge gateway 本地执行已授权任务；断网期间只执行有效 lease 和 local-authority-safe command。

Evidence

采集 barcode/RFID、timestamp、robot run id、sensor reading、photo、QNN inference、operator approval 和 signature。

Writeback

把库存移动、样本 lineage、CMMS work order、QMS deviation、eBR record 和 audit pack 写回系统。

05 · Product

先卖行业工作流包，再沉淀成企业机器人 iPaaS。

第一批产品不是空泛 connector marketplace，而是带 SOP、UI、权限、事件、验收指标、异常处理和写回路径的 workflow pack。每个 paid pilot 都要沉淀一个可复用包。

  • Warehouse first：WMS-driven cycle count、replenishment、pick-exception sweeps。ROI 清晰、重复性高、低操作风险。
  • Factory second：inspection-to-work-order loop。视觉/热/声学异常 -> MES/CMMS ticket，最适合 Qualcomm edge AI。
  • Lab / hospital third：secure specimen / supply courier with chain-of-custody into LIMS/LIS/EHR。
  • SCADA / OPC Pack：OPC UA、MQTT/Sparkplug、PLC interlock、设备状态、门禁放行、alarm、historian event。
  • Exception Compensation Pack：缺货、工位满、条码不符、温度越界、机器人阻塞、超时、人工接管、retry/skip/abort。
  • SI Sandbox & Certification Pack：模拟 WMS/LIMS/MES、robot task simulator、dead-letter queue、回归测试、partner certification。

06 · Product API/Evidence

核心对象不是 endpoint，而是可恢复、可审计、可写回的任务合约。

企业客户不只问“机器人有没有完成”，还会问谁批准、用哪个版本、是否符合前置条件、异常如何处理、系统记录如何更新、证据是否可以导出。

TaskContract

source、idempotency key、business object、robot capability、location、priority、due time、acceptance rule、rollback。

ApprovalGate

requested_by、required_role、approver、decision、reason_code、e-signature、quality hold、SCADA interlock、expires_at。

CommandRequest

lease、TTL、safety policy、local authority、approval status、max offline window、revalidation rule。

ExceptionRaised

category、severity、retryable、safe_to_retry、source_system、source_code、human_action_required、next_allowed_actions。

EvidencePacket

barcode/RFID、timestamp、run id、command log、sensor reading、photo、QNN inference、operator approval、signature。

AuditWriteback

inventory update、sample lineage、FHIR Task/Specimen、CMMS work order、QMS deviation、eBR record、export pack。

Event fabric：CloudEvents envelope + `tenant_id`、`site_id`、`edge_gateway_id`、`workflow_instance_id`、`correlation_id`、`causation_id`、`idempotency_key`、`actor`、`approval_id`、`offline_window`。Edge pattern: read many, write narrowly.

目标客户是已经买机器人、已有企业系统、但每个现场仍靠 SI 胶水的仓库、实验室、工厂和园区。商业模型是 90 天 paid pilot、年度 site subscription、edge gateway fee、connector pack fee、workflow pack fee 和 enterprise/on-prem/compliance package。

壁垒不是某一个连接器，而是不断扩大的机器人工作流地图。

OpsConnector 的白区：标准移动 robot orders 和 status；OT/lab 平台管理人类 workflow 和 records；robot vendors 暴露 API。OpsConnector 拥有 cross-system choreography、approval gates、exception handling 和 audit-grade writeback。

Connector Library

Edge Runtime

Certified SI Network

10 · Demo & Ask

7 分钟 demo：一个机器人，两个企业工作流配置。

比赛 demo 重点不是机器人动作炫技，而是完整任务合约、异常处理、离线边缘执行、Qualcomm edge evidence 和系统写回。

Warehouse

WMS cycle-count / replenishment task enters OpsConnector with bin、SKU、barcode、acceptance rule。

Contract

生成 TaskContract、ObjectMap、ApprovalGate、AdapterProfile、idempotency key 和 CloudEvents trace。

Edge

Dragonwing gateway via RobotAppLayer / VDA 5050 / Open-RMF adapter；network loss 不打断已授权本地任务。

Exception

机器人用 on-device vision 发现 bin/location mismatch；不写 false success，触发人工审批。

Writeback

写回 inventory exception / CMMS work order / sample lineage，附 QualcommEdgeProfile 和 EvidencePacket。
