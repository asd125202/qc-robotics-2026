Pitch 27 · Enterprise Robot Workflow Contract

OpsConnector

机器人已经会看、会走、会抓。企业真正缺的是：谁能让它安全接单、按流程执行、断网不停、异常可处理、结果可审计。OpsConnector 把机器人动作变成 WMS、MES、LIMS、SCADA 和 ERP 里的可信企业记录。

01 · Problem

企业机器人部署失败，不是因为机器人不会动，而是因为它进不了企业流程。

仓库有 WMS，工厂有 MES/SCADA，实验室有 LIMS/ELN，医院有 LIS/EHR，财务和供应链有 ERP。机器人任务如果不能接收业务意图、遵守审批和安全边界、处理异常、把结果写回系统记录，最终就会变成现场脚本、人工补录和一次性集成项目。

Warehouse

WMS 不会直接控制机器人

WMS 管订单、库存、波次、库位和报表；机器人需要秒级位置、路径、拥堵、充电、工位容量和异常恢复。

Lab

机器人动作没有变成记录

移液、扫码、交接、称重和样本转移如果不写回 LIMS/ELN，就会破坏 sample lineage、chain of custody 和审计。

Factory

工单和设备状态断裂

MES/eBR 需要材料、人员、设备、批次、质量和偏差记录；机器人平台通常只返回 mission done。

Exception

异常靠人工补救

缺货、条码不符、温度越界、工位满、门禁未放行、机器人阻塞和人工接管经常掉进微信群、电话和 Excel。

Scale

每个现场重做集成

换一个 WMS、机器人品牌、仓库流程、样本规则或安全区，项目就重新写接口，无法复制到下一站点。

02 · Current Alternatives Fail

现在的方案都解决了一部分，但没有拥有“企业任务到机器人执行再到系统写回”的闭环。

OpsConnector 不替代 WMS、MES、LIMS、SCADA、机器人厂商 fleet manager 或系统集成商。它把这些系统之间最难复用的语义、状态、异常、权限和审计写成可复制的工作流合约。

SI 项目制

系统集成商能让一个现场上线，但接口、异常处理和回写逻辑常常沉没在项目代码里，下一站点还要再卖人天。

机器人 OEM 软件

Locus、Geek+、Hai、Exotec、OTTO、MiR 等证明 fleet 软件有价值，但通常优先服务自家车队和任务模型。

WMS / MES 扩展

SAP EWM、Manhattan、Blue Yonder、Körber 或定制 WMS 能发业务任务，但不应直接承担机器人实时控制和 OT 安全。

iPaaS

MuleSoft、Boomi、Workato 懂 SaaS API 和数据流，但不懂机器人地图、任务状态、低延迟、离线执行、PLC interlock 和安全停机。

标准协议

VDA 5050、MassRobotics、Open-RMF、OPC UA Robotics 是好基础，但不覆盖订单语义、样本 lineage、库存写回和商业 SLA。

Ops Dashboard

Formant、InOrbit、Foxglove 等验证了可观测性和运维需求；企业仍需要任务合约、审批、异常补偿和系统记录写回。

03 · Solution

OpsConnector 是企业系统和机器人执行层之间的工作流合约层。

它把 WMS/MES/ERP/LIMS/SCADA/人工审批里的业务请求，编译成可验证、可追踪、可审计的 Robot Task Contract，再分发给 RobotAppLayer、EdgeFleet、Open-RMF、VDA 5050、MassRobotics、OPC UA 或厂商 fleet API。

Intent

订单、工单、样本请求、FHIR Task、SCADA interlock、QMS 偏差或人工审批进入统一入口。

Contract

任务 ID、对象、位置、设备需求、审批、验收规则、超时、回滚、审计目标和 trace context。

Dispatch

按能力注册表选择 RobotAppLayer、VDA 5050、Open-RMF、OPC UA、厂商 API 或人工队列。

Evidence

采集条码、时间戳、robot run id、传感器、图片、模型版本、异常、接管和签名事件。

Writeback

把完成、失败、库存移动、样本状态、批记录、QMS 偏差和审计包写回企业系统。

05 · Product

先卖行业工作流包，再沉淀成企业机器人 iPaaS。

OpsConnector 的第一批产品不是空泛 connector marketplace，而是带 SOP、UI、权限、事件、验收指标、异常处理和写回路径的 workflow pack。每一个付费试点都要产出一个可复用包。

  • WMS Bridge Pack：订单、波次、拣选、补货、移库、入库、出库、盘点、异常和库存写回。
  • Lab Sample Transfer Pack：LIMS/Benchling 样本、plate map、协议、审批、条码核验、结果和 audit package。
  • MES / eBR Pack：工单、批次、设备、物料、操作员、quality hold、deviation 和 electronic batch record 写回。
  • SCADA / OPC Pack：OPC UA、Sparkplug、PLC interlock、设备状态、门禁放行、alarm 和 historian event。
  • Exception Pack：缺货、工位满、条码不符、温度越界、机器人阻塞、超时、人工接管、retry/skip/abort。
  • SI Sandbox：模拟 WMS/LIMS/MES、测试 robot task、dead-letter queue、回归测试和认证培训。

06 · Product API

核心对象不是 API endpoint，而是可恢复、可审计、可写回的任务合约。

企业客户不只问“机器人有没有完成”，还会问谁批准、用哪个版本、是否符合前置条件、异常怎么处理、系统记录如何更新、证据是否可以导出。

TaskContract

source、idempotency key、business object、robot capability、location、priority、due time、acceptance rule、rollback。

ObjectMap

样本、容器、货架、托盘、库位、设备、人员、工单、批次、条码、plate/well 和企业系统 ID 映射。

ApprovalGate

operator、reviewer、role、e-signature、quality hold、SCADA interlock、override reason 和 release token。

AdapterProfile

VDA 5050 order/state、Open-RMF task、MassRobotics telemetry、OPC UA job、Sparkplug、REST/gRPC。

EvidenceLedger

barcode/RFID、timestamp、run id、command log、sensor reading、photo、QNN inference、exception 和 signature。

AuditWriteback

Job Response、inventory update、sample lineage、FHIR Task/Specimen、QMS deviation、eBR record 和 export pack。

07 · Market & Business Model

把一次性 SI 人天，产品化成年度订阅、连接器包和工作流包。

机器人集成服务池是巨大的，但项目制收入不可复制。OpsConnector 的商业模型是：用 SI/OEM 做交付渠道，同时把工作流合约、edge gateway、连接器、异常处理和审计写回留在软件层。

中国版

快接入、低改造、多品牌、私有化

中国机器人厂商多、项目交付快、WMS/ERP 定制重。包装成样板间 PoC、OEM/SI white-label、私有云、国产系统适配、固定范围交付和 25%-40% 渠道毛利。

海外版

标准合规、on-prem、审计和年度合同

海外企业更看重 SAP/Manhattan/Blue Yonder 集成、IT/OT 网络隔离、网络安全、审计、SSO、数据驻留、可预测用量和采购安全的年度订阅。

$60k-180k 海外生产站点年费：control plane、workflow builder、RBAC、audit 和 observability。

08 · Competition & Moat

壁垒不是某一个连接器，而是不断扩大的机器人工作流地图。

InOrbit、Viam、Formant、Foxglove、Intrinsic、MoveIt Pro、Wandelbots、MuleSoft、Boomi、Workato 都证明需求存在。OpsConnector 的壁垒在于把 robot state、OT 安全、业务语义、异常处理、审计写回和 SI 交付资产放在同一个产品对象里。

Connector Library

WMS/MES/LIMS/ERP/SCADA/QMS/robot brand 的已验证 adapter 越多，下一次交付越快。

Workflow Templates

订单到机器人、样本到记录、工单到 eBR、异常到 QMS、库存到 ERP 的模板不断复用。

Edge Runtime

on-prem gateway 处理离线缓存、低延迟、幂等、dead-letter queue、OT/IT 网络分区和本地签名。

Audit Corpus

每个现场积累异常、审批、写回、失败原因和验收证据，形成可学习的部署知识。

SI Network

认证培训、sandbox、partner portal、私有 connector 发布和收入分成让集成商愿意沉淀资产。

Compliance Readiness

NIST SP 800-82、ISA/IEC 62443、Part 11、GxP、FHIR、ISA-95 让企业采购更容易通过审查。

10 · Demo & Ask

7 分钟证明：机器人任务可以从企业系统来，也能把结果写回企业系统。

比赛 demo 可以用两个场景讲清楚商业价值：仓库 WMS 派单和 LabForge 样本转移。重点不是机器人动作炫技，而是完整任务合约、异常处理、离线边缘执行和审计写回。

Create

模拟 WMS 补货单或 LIMS 样本转移请求，带货位/样本/条码/审批/验收条件。

Compile

OpsConnector 生成 TaskContract、ObjectMap、ApprovalGate、AdapterProfile 和 idempotency key。

Execute

Dragonwing edge gateway 断网仍能执行已授权任务，调用 RobotAppLayer 或 VDA 5050/Open-RMF adapter。

Exception

条码不符、工位满或温度越界触发人工审批，不写成功结果，只写 deviation 和 incident。

Writeback

任务完成后把库存移动、样本 lineage、QNN evidence、operator、版本和 audit pack 写回系统。

Ask

向高通要什么

RB3 Gen 2 Vision/Core Kit、QCS8550 合作渠道、IQ10 RRD early access 或 roadmap 指导、AI Hub/QNN/QAIRT office hours、Qualcomm Linux/Ubuntu/ROS2 指导，以及 secure boot/device identity/OTA 推荐路径。

Outcome

比赛交付物

一个可运行 Dragonwing edge workflow demo、两个 connector/workflow pack 原型、一个企业审计证据包、一个 SI sandbox，以及中国/海外两套商业包装和报价模型。

OpsConnector 的价值不是让机器人“完成任务”，而是让企业系统知道任务为什么发生、谁批准、如何执行、哪里异常、结果写回了什么。
