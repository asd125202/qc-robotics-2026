00 · Five-Thread Research

机器人商业化的瓶颈不是会不会动，而是能不能接进业务系统。

五条研究线指向同一个结论：标准、仓储、实验室、机器人平台公司和买家经济学都说明，连接器与工作流本身就是产品。OpsConnector 不做另一个 dashboard，而做 enterprise task 到 robot task 的可信翻译层。

Standards

ISA-95、OPC UA、MQTT/Sparkplug、VDA 5050、MassRobotics、Open-RMF。

Warehouse

WMS / WES / WCS / RCS 负责订单、库存、批次、搬运和异常确认。

Lab

LIMS / ELN 管理样品、协议、结果和 lineage，机器人执行物理流程。

Platforms

Viam、Formant、InOrbit、Foxglove 等验证 API、ops、incident 和日志价值。

Buyers

客户关心 TCO、停机、IT/OT 安全、项目周期和系统集成风险。

01 · Robot Task Contract

把企业任务翻译成机器人可以执行、可以回写、可以审计的合约。

OpsConnector 的核心不是接入某一个 API，而是定义一份跨工厂、仓库、实验室都能理解的任务合约，再通过 adapter 接入不同企业系统和机器人执行层。

Intent

订单、工单、实验请求、异常处理或人工审批。

Contract

任务类型、对象、目标、优先级、技能、权限和验收。

Dispatch

通过 RobotAppLayer、EdgeFleet、Open-RMF 或厂商 fleet 执行。

Events

状态、位置、健康、异常、接管、结果和证据持续回传。

Writeback

成功、失败、库存、样品、质量、日志和审计结果回写。

02 · Workflow Packs

连接器要卖成行业工作流，而不是卖成一堆 API 文档。

系统集成商和企业客户需要可复用模板：SOP、UI、权限、事件、验收指标、异常处理和回写路径。OpsConnector 把这些包装成 workflow pack。

Warehouse

WMS pick、putaway、transport、confirmation、exception 和 inventory update。

Factory

MES work order、OPC UA asset telemetry、PLC signal 和 quality record。

Lab

LIMS request、protocol compile、sample lineage、result artifact 和 audit。

Operator

人工审批、异常确认、retry / skip / abort 和 incident handoff。

Security

SSO、RBAC、TLS、certificate、broker ACL、schema validation 和 audit log。

Sandbox

模拟订单、模拟 LIMS、测试 robot task、dead-letter queue 和回归测试。

03 · Edge Gateway

企业集成不能依赖一条永远稳定的公网连接。

OpsConnector 的现场 edge gateway 运行在 RobotCoreOS / Qualcomm edge 环境里。它负责本地缓存、断网继续运行、低延迟任务下发、OT/IT 网络分区、idempotency、retry 和审计事件签名。

  • Northbound：WMS / MES / ERP / LIMS / ELN / SCADA。
  • Sideways：OPC UA / Sparkplug telemetry 和 enterprise observability。
  • Southbound：RobotMac skill runtime、VDA 5050、Open-RMF、MassRobotics。
  • Governance：device identity、schema validation、audit trail 和 permission policy。

04 · Qualcomm Business Value

让 Qualcomm edge 成为企业 IT/OT 流程里的可信执行节点。

OpsConnector 把 Qualcomm 的价值从“机器人本体推理”扩展到企业流程执行。任务来自业务系统，本体在 Qualcomm edge 执行，结果回写到客户记录，失败片段回到 CloudTwin / TrainRouter。

On-prem 现场 edge gateway 支持本地缓存、断网运行和 OT/IT 网络隔离。
Workflow 企业任务、机器人技能、人工审批和回写路径被绑定在一起。
Audit 状态、异常、接管、结果、版本和安全事件进入可导出证据链。
Channel 系统集成商可购买 connector pack、workflow pack 和认证培训。

OpsConnector 把“机器人能完成任务”变成“机器人能进入企业流程，并把结果写回系统记录”。
