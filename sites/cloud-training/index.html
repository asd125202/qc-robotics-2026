Pitch 02 · LeRobot-to-Qualcomm Training Control Plane

LeRobot CloudTwin

RobotMac Core 的云端训练工厂：把真实机器人 episode 变成 LeRobot 数据集，路由到中国或海外 GPU，统一评测，再打包成带 lineage、runtime profile、rollback 和证据表的 Qualcomm edge policy。

进入路演 查看 TrainRouter

01 · Problem

机器人团队能做 Demo，但很难把 Demo 变成可交付技能。

云端 notebook 不是机器人产品。商业机器人需要知道哪版数据训练了哪版模型、跑在哪块板、评测是否可比、失败能不能复现、坏版本能不能回滚、数据能不能留在正确区域。

数据格式散

视频、状态、动作、时间戳、操作者、任务标签、失败接管和安全事件常散在不同文件夹里。

训练云分裂

中国和海外面对不同账号、GPU、镜像、付款、网络、数据边界和支持渠道。

评测不可比

同一任务在不同云、不同 split、不同 seed 训练，输出结果很难被采购经理和工程负责人共同信任。

部署靠手工

模型权重不是产品交付物，还需要输入输出契约、runtime 依赖、硬件 profile、rollback 和运行证据。

失败没进入飞轮

部署失败、人工接管、低置信度和任务超时如果没有回流，就只是事故，不是下一轮训练资产。

边缘价值没沉淀

Qualcomm 的价值不能只停留在“最终跑一下”，而要进入每个 policy release 的 target profile。

02 · Why Now

机器人学习正在从论文和 notebook 变成产品交付流程。

LeRobot 已经把真实机器人数据、模仿学习、HIL 数据采集、数据集格式和策略训练推到开发者可用层；VLA、foundation policy、Physical AI data factory、robot fleet learning 都在快速推进。现在缺的是一套把这些能力送回 Qualcomm edge 的商业控制面。

LeRobot 标准化

LeRobotDataset v3 用 Parquet、video shards、metadata、tasks 和 episode 边界组织多模态时间序列。

HIL 变成数据资产

人工接管和恢复动作不只是救场，而是下一轮训练的高价值样本。

GPU 供给足够碎片化

RunPod、Lambda、Modal、阿里云 PAI、腾讯 GPU、华为 ModelArts、AutoDL 都能训练，但没有统一机器人 job contract。

Edge AI 进入生产

Qualcomm 正在强调 prototype-to-production edge AI；CloudTwin 把云训练自然导向本体侧低延迟执行。

03 · Insight

机器人技能不是“下载一个模型”，而是像 SaaS release 一样发布。

真正可卖的不是 GPU 小时，也不是模型权重，而是每个任务的可验证学习循环：episode lineage、locked split、training job record、eval gate、Qualcomm edge profile、runtime evidence、rollback 和 failure mining。

04 · Solution

CloudTwin 是 productized LeRobot-to-Qualcomm-edge control plane。

它把真实 robot episodes 变成版本化数据集，按中国或海外 GPU lane 路由训练，对 policy 做固定 gate 评测，再导出 signed Qualcomm-ready skill package，带 rollback、人类纠错反馈和运行证据模板。

Competition target

Dataset Ledger

记录 episode、robot profile、sensor manifest、task、region、rights passport、hash 和 dataset lineage。

Competition target

TrainRouter

同一个 LeRobot job contract 路由到 China、Overseas 或 Auto GPU lane，带预算和数据边界。

Competition target

Eval Harness

锁定 split、seed、指标和回放样例，让不同云训练结果可以比较。

Product target

Edge Packager

输出模型、输入输出契约、依赖、QNN/QAIRT/ONNX path、hardware profile、rollback 和 manifest。

Product target

Failure Mining

低置信、任务超时、安全事件、人工接管和客户拒收进入下一轮训练队列。

Future profile

VLA Upgrade

ACT 做比赛保底；SmolVLA、X-VLA、π0、GR00T-style policy 作为后续可插拔策略路线。

05 · Product Workflow

从真实 episode 到 Qualcomm edge policy，一条证据链。

CloudTwin 不把机器人安全交给云端。云负责训练和资产管理，Qualcomm edge 负责本地低延迟执行，失败再变成下一轮训练样本。

Capture

RobotMac Core / TeleopStudio 记录视频、状态、动作、时间戳、operator、success/failure 和 takeover。

Ledger

转成 LeRobot-compatible dataset，写入版本、hash、隐私级别、region、rights passport 和 task tags。

Route

TrainRouter 根据数据地域、预算、GPU class 和 provider availability 选择 China / Overseas lane。

Train

ACT 起步，扩展 Diffusion、SmolVLA、X-VLA；保存 config、seed、container digest、cost 和 logs。

Evaluate

输出 success rate、failure type、action smoothness、latency budget、replay examples 和 fixed split report。

Package

生成 Qualcomm edge package：policy hash、ONNX/QNN artifacts、runtime deps、rollback 和 manifest。

Profile

AI Hub / Device Cloud / Profiler 记录 latency、load time、memory、compute target、thermal 和 power。

Deploy

Skill package 进入 RobotCoreOS / SkillCertKit，在本体本地执行，不依赖云端实时控制。

Recover

坏模型、坏配置或安全 gate 失败触发 admission denial、safe mode 或 rollback。

Mine

失败片段、低置信度、接管和客户反馈进入下一轮训练队列和 regression test。

06 · Market

第一批客户买的是“可交付训练闭环”，不是便宜 GPU。

GPU 价格会变，但机器人团队真正缺的是 demo-to-skill 的确定性：数据能不能可信、训练能不能复现、评测能不能解释、部署能不能回滚、失败能不能继续学习。

LeRobot 开发者

需要低摩擦 ACT / Diffusion / SmolVLA 模板、dataset conversion、eval reports 和便宜 GPU routing。

机器人创业公司

有 prototype，但没有可靠 data/train/deploy loop；需要 dataset ledger、regression eval、model export 和 rollback。

系统集成商

卖 4-8 周试点，需要把“采 100 条示教、训练 policy、交付 edge package”写进固定范围合同。

企业创新团队

关心 uptime、ROI、数据地域、审计、预算 guardrail、approved rollout 和 compliance report。

RaaS 运营商

按活跃机器人购买 failure mining、fleet replay、canary model rollout、OTA 和 success dashboard。

中国本土团队

需要中文支持、国内账号、国内付款、数据本地训练和本地 GPU provider，而不是强制海外账号。

07 · Business Model

两段式收费：平台费 + GPU 成本 pass-through。

CloudTwin 不把“租 GPU”当核心收入，而是按 active robots、storage、teleop seats、captured demo-hours、training jobs、private data lane、eval reports、edge export targets 和 support SLA 计价。

Attach

Kit Bundle

随 RobotMac Core、TeleopStudio、教育套件或行业 demo kit 附带初始训练额度和示例任务。

SaaS

CloudTwin Platform

dataset ledger、training jobs、eval reports、edge export、private registry、policy lineage 和 rollout gates。

Metered

GPU Routing

GPU 成本透传，叠加 orchestration margin、budget guard、preemption handling、logs 和 artifact storage。

Enterprise

Private Lane

中国/海外私有 lane、数据边界、SSO/RBAC、审计报告、on-prem artifact registry 和 edge validation sprint。

08 · GTM

先用比赛证明一条小闭环，再卖课程、试点和 SI 模板。

CloudTwin 的早期销售不是大而全平台，而是固定范围学习闭环：一个任务、一个机器人形态、一份数据集、一条训练 job、一份 edge package、一组失败回流。

比赛闭环

桌面机械臂 10-20 条 episode，ACT 小训练，eval，Qualcomm edge package，runtime evidence。

教育课程

LeRobot-to-Qualcomm 课程包：第一天能跑，第一周能训练，第一月能部署。

SI 试点

pick/place、扫码、巡检、质检、lab transfer 做成 60-90 天 bounded-task pilot。

企业私有 lane

数据地域、预算、审计、approved rollout 和 failure mining 进入企业采购语言。

生态样例

争取 Dragonwing Robotics Hub tutorial、AI Hub workflow 和 board profile validation checklist。

09 · Competition

不是模型、不是 GPU 云、不是 fleet dashboard。

机器人学习平台类别正在成形，但大多数能力仍分散在开源框架、foundation model、仿真、GPU 云、fleet ops 和数据可视化里。CloudTwin 的位置是训练到 Qualcomm edge 交付的控制面。

LeRobot

开放机器人学习引擎；CloudTwin 是商业 job、数据边界、评测、edge packaging 和 release governance。

PI / Skild / Figure

证明 VLA 和 generalist robot policy 方向；CloudTwin 不做另一家 foundation model，而做可插拔生产闭环。

NVIDIA Isaac / GR00T

强 physical AI 全栈；CloudTwin 是 Qualcomm-first counter-position：双云训练、低功耗 edge、AI Hub/QNN packaging。

Viam / Formant

强 fleet、teleop、ops 和 incident；CloudTwin 聚焦 training/evaluation/deployment loop，并导出可运维 artifact。

Foxglove

强机器人数据观测和检索；CloudTwin 把数据观测推进到训练 gate、policy release 和 edge rollback。

GPU Clouds

RunPod、Lambda、Modal、PAI、腾讯、华为卖算力；CloudTwin 卖 robot-aware job spec、metrics、safety bounds 和 edge export。

10 · Moat

护城河是 episode ledger、eval harness 和 Qualcomm edge profile。

每一次成功、失败、接管、恢复、评测、发布和回滚都会增加下一次技能训练的确定性。

Ledger 真实 episode、dataset card、robot profile、operator role、rights passport、raw/clean/train/eval/release graph。
Eval 同一 split、同一指标、同一 replay harness，让不同云和不同模型结果可比较。
Profile Qualcomm board、camera、runtime、thermal、power、accelerator path、latency 和 rollback matrix。
Loop 失败片段、人工接管、低置信任务和客户拒收进入下一轮 regression tests。

11 · Release Evidence

每个 policy release 都要像软件发布一样可审计。

CloudTwin 不能声称“自动合规”或“官方认证”，但可以提供支持审查的证据包：数据、训练、评测、部署、回滚和失败回流全链路可追踪。

Competition target

Episode Schema

episode_id、dataset hash、task、robot profile、sensor manifest、timestamps、state、action、takeover、labels、safety events。

Competition target

Lineage

raw、clean、train、eval、release candidate 独立版本，记录 transforms、QC rejects、dedupe、license 和 retention。

Competition target

Job Record

provider、lane、region、GPU SKU、image digest、command、budget cap、logs hash、checkpoint hash 和 cost estimate。

Product target

Policy Manifest

policy_id、algorithm、LeRobot version、git commit、dataset split、seed、hyperparams、checkpoint、ONNX/QNN artifacts、rollback。

Product target

Data Passport

owner、license、allowed use、region、retention、commercial-training permission、deletion/freeze 和 shared-improvement policy。

Measured label

Metric Labels

所有数值分成 simulated、replay-eval、measured-on-device，不能混成一个成功率或延迟数字。

12 · Why Qualcomm

云负责训练，Qualcomm edge 负责执行。

机器人 policy 需要云 GPU 训练，但部署后的机器人需要本地推理、相机/传感器同步、低延迟、断网韧性、隐私和可测量 runtime behavior。CloudTwin 让 Qualcomm 拥有 post-training deployment standard，而不只是“机器人里的那块板”。

AI Hub Workbench

把 PyTorch/ONNX policy artifact 转成可 compile/profile/validate/deploy 的 Qualcomm runtime candidate。

QNN / QAIRT

定义 target runtime、input shape、backend、quantization、calibration、context/DLC/ONNX Runtime QNN 选择。

Device Cloud + Profiler

在 hosted Qualcomm targets 或本地板卡上记录 latency、memory、utilization、thermal、power 和 load time。

Robotics Hub

把“teleop to Qualcomm edge deployment”整理成 Dragonwing Robotics Hub tutorial 或 reference sample。

13 · Demo Story

三分钟讲清楚：Cloud-trained. Qualcomm edge-deployed. Evidence-backed.

比赛 demo 用桌面机械臂或小车做固定任务，展示数据、训练、评测、边缘部署和失败回流。重点不是一次动作，而是一次可复制的 policy release。

Record

Teleop 采集 10-20 条 episode，记录 camera、state、action、timestamps、success/failure 和 takeover。

Train

CloudTwin 生成 dataset card，提交 China/Overseas training job，展示 cost、logs、config 和 checkpoint。

Evaluate

Eval report 展示 split、success rate、failure type、replay clips 和是否通过 release gate。

Deploy

导出 Qualcomm edge package：policy hash、target hardware、AI Hub/QNN status、rollback package。

Improve

机器人本地执行，触发一次失败或接管，失败片段进入下一轮 training queue。

14 · Ask

请求 Qualcomm 支持一个 CloudTwin validation sprint。

6-8 周目标：跑通一个固定任务的 LeRobot episode、云训练、AI Hub/QNN workflow、Qualcomm edge package、runtime evidence 和 failure mining，并把它整理成可复用开发者样例。

  • 硬件路径：RB3 Gen 2 Vision Kit、QCS6490、QCS8550、IQ-8275 / IQ10 guidance。
  • AI Hub / QNN：ACT-class policy export、fixed input shapes、quantization、QNN DLC / context / ONNX Runtime QNN 选择。
  • Device Cloud：hosted Qualcomm targets 用于 policy artifact validation。
  • Profiler：定义官方 evidence table，覆盖 latency、load time、memory、utilization、thermal、power。
  • Robotics Hub：发布 “LeRobot CloudTwin: teleop to Qualcomm edge deployment” sample。

15 · Claim Boundaries

可信边界：CloudTwin 是训练与发布控制面，不是认证机构或实时安全控制器。

可以说 LeRobot-compatible、Qualcomm-first、dual-cloud training architecture、edge evidence workflow。不能说已获 Qualcomm/LeRobot 官方认证、自动合规、SOC 2/ISO 认证、功能安全认证、任意模型自动跑 QNN、性能优于 Jetson/x86。云端不进入 safety-critical control loop；所有延迟、功耗、温度、FPS、成功率都必须标注 simulated、replay-eval 或 measured-on-device。

CloudTwin 不卖“便宜 GPU”，它卖把真实机器人经验变成可部署、可回滚、可持续改进的 Qualcomm edge 技能。
