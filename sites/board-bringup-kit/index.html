Pitch 21 · Board-to-Robot Bring-up Kit

BoardBringupKit

开发板不是产品。BoardBringupKit 把 Rhino X1 / QCS8550 等比赛板卡接入电源、相机、IO、安全、运行时、部署 manifest 和 benchmark checklist，变成 RobotMac Core 的第一版工程样机。

01 · Board To Robot

收到开发板只是开始，关键是把它变成可复现机器人核心。

官方标准赛道强调系统级闭环、稳定运行和真实环境部署。BoardBringupKit 把裸板 bring-up 拆成电源、相机、IO、安全、runtime 和验证清单。

Primary

APLUX Rhino X1 / QCS8550：LabForgePilot 主 demo、RobotMac Core Pro 和 EdgeRuntimeBench profile。

Fallback

Radxa Dragon Q6A / QCS6490：Core Lite、教育版和简化感知控制任务。

Pioneer

Arduino VENTUNO Q / IQ-8275：工业控制、异构计算和未来 Industrial SKU。

02 · Bring-up Stack

每一块板都要经过同一套工程门禁。

这不是临时接线，而是可复制交付流程。未来企业试点、教育套件和系统集成商都能复用同一套 bring-up checklist。

Power

稳压、保险、状态灯、分路供电和故障重启。

Camera

固定视角、标定、时间戳和 LabForge 视觉输入。

Robot IO

机械臂、夹爪、急停、限位、CAN/UART/PWM/GPIO。

Safety

物理急停、watchdog、软边界和失败接管日志。

Runtime

系统镜像、ROS 2 bridge、LeRobot adapter 和 CloudTwin agent。

Validate

摄像头、机械臂、episode、部署包、profile 和 rollback。

03 · After Board Arrival

把 8 月第一周做成确定性工程节奏。

7 月底入围后开发板邮寄，最危险的是收到板子后才开始思考接线和运行时。BoardBringupKit 提前锁定硬件清单、bring-up 顺序和保底路径。

  • 到货前：准备相机、机械臂、样品托盘、急停、线束和普通 Linux fallback。
  • 第 1 周：启动、网络、存储、摄像头、runtime image。
  • 第 2 周：机械臂、急停、遥操作采集和固定任务回放。
  • 第 3-4 周：CloudTwin 输出、EdgeRuntimeBench profile、LabForgePilot 初版素材。

04 · Why Qualcomm

把开发板支持升级成机器人产品核心支持。

开发者真正需要的不是一块裸板，而是板卡进入机器人系统的路径。BoardBringupKit 让 Qualcomm 的价值落在电源、IO、相机、AI runtime、安全和可复现部署上。

Board

Rhino X1 / QCS8550 作为主目标。

Harness

电源、相机、IO 和安全接入。

Runtime

ROS 2、LeRobot、CloudTwin 和 EdgeRuntime。

Demo

LabForgePilot 快速形成真实闭环素材。

Product

沉淀为 RobotMac Core bring-up 标准。

BoardBringupKit 的目标，是让“收到开发板”不再是风险点，而是 RobotMac Core 工程样机启动的第一天。
