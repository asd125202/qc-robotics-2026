Pitch 21 · 48-hour Qualcomm Robot Bring-up Sprint

BoardBringupKit

开发板不是机器人产品。BoardBringupKit 把 RB3/QCS6490、Rhino Pi-X1/QCS8550、Dragon Q6A/QCS6490、IQ-9075 等板卡，从“桌上能开机”变成可复现、可验收、可交付的 RobotMac Core 候选核心。

看开发板断点 看 bring-up 演示

01 · Problem

机器人团队不缺开发板，缺的是 48 小时内变成可验收机器人核心的路径。

收到板卡后，真正吞掉时间的是电源、相机、I/O、安全、ROS 2、LeRobot 数据和 QNN runtime 的重复 bring-up。一个松动线束、错误电平、未标注 fallback 或静默热降频，都可能让 demo 和企业试点失败。

Power

供电不是插电

12/24V 输入、峰值电流、brownout、USB 供电、保险、TVS/surge、状态灯和 power-good gate 都要被验证。

Camera

相机不是画面

MIPI/USB/GMSL、V4L2、GStreamer、frame drop、标定和 timestamp 直接影响 LeRobot episode。

I/O

接口不是针脚

CAN、RS-485、GPIO、PWM、I2C、SPI 的电平、隔离、transceiver、terminator、pinmux 必须测试。

ROS

节点能跑不等于系统可靠

ROS 2、DDS、ros2_control lifecycle、QoS、topic rate、diagnostics 和启动顺序需要工程门禁。

Safety

安全不能赛后补

硬急停、watchdog、command timeout、soft boundary 和 actuator power-cut 必须独立于 Linux 应用层。

Evidence

没证据就不能复现

OS image、kernel/device tree、BSP、firmware、QNN/QAIRT、LeRobot、dataset hash 和 image digest 必须锁定。

02 · Current Alternatives Fail

市场不缺板卡，缺的是从 bench 到 robot 的可复用验收工作流。

Vendor docs、ROS tutorials、Jetson/Isaac、整机机器人、fleet tools 和一次性 SI 都重要，但它们没有把 first robot bring-up 做成 board recipe、golden image、HIL fixture 和 evidence pack。

Vendor Docs

讲接口、烧录和 SDK，不讲完整机器人线束、E-stop、sensor timing、ROS graph 和客户验收包。

ROS 2 / ros2_control

能连接硬件，但团队仍要写 hardware component、处理 lifecycle、实时调度、fallback 和诊断。

NVIDIA Jetson / Isaac

开发者信心强，是主要对标；但 developer kit 不是 production，carrier、thermal、power 和 validation 仍需工程化。

Pi / Orange Pi

可及性强，适合 prototype；进入企业机器人还需要 BSP、长供、安全、EMC 和 validation。

Advantech / AAEON / ADLINK

工业包装强，但通常是企业销售和设计导入，不是低摩擦 developer-first board-to-robot kit。

Luxonis / OAK

smart sensor 很强，可以减轻 perception 压力，但不替代主控、power、OTA、安全和 productization stack。

03 · Solution

BoardBringupKit 是 Qualcomm board-to-robot 的可执行验收层。

它把开发板、线束、相机、I/O、安全、运行时和证据导出接成同一套 workflow。目标不是让 demo 碰巧跑起来，而是让每次 board、image、sensor、runtime 变化都能重新验收。

Profile

锁定 SoC、board revision、OS、kernel、BSP、firmware、runtime、image digest 和 dependency lock。

Harness

受保护 12/24V 输入、E-stop cut path、camera mount、USB/CSI/GMSL、CAN/RS-485 和状态灯。

Gates

first boot、camera frame、USB speed、actuator ping、ros2_control lifecycle、watchdog、E-stop trigger。

Runtime

LeRobot episode、policy replay、AI Hub/QNN/ONNX profile hook、EdgeRuntimeBench route、SkillDock package。

Evidence

生成 board bring-up report、wiring map、safety ledger、runtime image lock 和 audit pack。

04 · Why Now

Qualcomm 机器人板卡窗口打开了，但开发者时间窗口很短。

复赛从 8 月到 11 月 1 日，底层 bring-up 的每一天都会挤压数据采集、训练、端侧部署和商业包装。现在需要把 Qualcomm board 的优势转成“第一天就能验收”的开发体验。

542K IFR 报告 2024 年全球工业机器人安装约 54.2 万台，机器人部署已进入规模市场。
RB3 RB3 Gen 2 / QCS6490 是 practical first target：Ubuntu 24.04、AI Hub、Edge Impulse、QIR/ROS path。
QCS8550 premium compute target，但 AI Hub 上 QCS8550 profile 常见为 proxy，必须标注。
48h 目标：48 小时内完成 board-to-robot acceptance，而不是两周 debug。
v0.6.0 LeRobot v0.6.0 和 Dataset v3 让 first episode 和 evidence 变得可标准化。

05 · Product

不是一套线，而是一个 bring-up operating system。

BoardBringupKit 把每块板分成明确状态：Ready、Sensor Risk、I/O Risk、Runtime Risk、Safety Blocked、Needs BSP Fix、Production Migration Required。每个状态都能追到命令、log、镜像和硬件照片。

  • Board Recipe Library：Q6A、Rhino X1、RB3 Gen 2、IQ-9075、VENTUNO Q 的 manifest、known-good image、接口图和版本锁。
  • Protected Harness BOM：12/24V keyed connector、fuse/eFuse、reverse polarity、over/undervoltage、inrush、TVS/surge。
  • Production Carrier Pattern：power sequencing、watchdog、RTC、debug/recovery、test pads、wide input、GbE、USB、CAN、M.2、fan、camera expansion。
  • Sensor / I/O / Safety Gates：MIPI/USB/GMSL camera、GStreamer、protected CAN/RS-485、24V GPIO、dual-channel E-stop、watchdog。
  • Runtime Gate：LeRobot episode、policy replay、AI Hub/QNN/ONNX profile hook、EdgeRuntimeBench route、SkillDock package hook。
  • Factory/EOL Gate：flash image、program serial/MAC/keys、verify rails、USB、Ethernet、CAN、camera lanes、fan/tach、thermal smoke test。

06 · Product API/Evidence

bring-up 证据必须能被评委、SI、客户和供应链同时读懂。

每项指标标注来源：`measured-on-board`、`AI-Hub-device-cloud`、`proxy`、`vendor-doc`、`simulated` 或 `blocked`。诚实标注比假装全链路稳定更可信。

board-bringup-manifest

board、SoC、serial、OS、kernel、BSP、image digest、runtime versions、container digest。

wiring-map

power rails、camera、I/O、E-stop、actuator、network、safety boundary 和 connector photo index。

power-thermal-log

input voltage、current estimate、temperature、throttling flags、fan state、load scenario。

camera-gate-report

device path、driver、V4L2/media topology、GStreamer pipeline、FPS、drop rate、calibration hash。

safety-event-ledger

E-stop、watchdog、command timeout、soft-boundary deny、manual takeover 和 rollback pointer。

release-evidence

SBOM/provenance、OTA signing status、secure-boot state、vulnerability triage 和 approvals。

CLI

一条命令出验收包

`bbk init --board rb3-gen2-qcs6490 --robot lab-transfer-arm` → `bbk gate boot,camera,io,safety,runtime` → `bbk export`。

Boundary

证据支持，不等于认证

支持 bring-up、release review、supplier audit 和 certification preparation；不写 certified、compliant 或 production approved。

Pack

board-bringup-audit-pack.zip

manifest、wiring、power/thermal、camera、I/O、safety、runtime lock、edge profile、acceptance report 和 demo index。

07 · Market & Business Model

Global Edition

startup + OEM prototype + certification preparation

围绕 RB3/QCS6490、QCS8550 SOM、Jetson 对照、ROS 2/HIL、fleet telemetry 和 pre-certification evidence。强调 audit pack 和 production migration。

08 · Competition & Moat

壁垒不是线束，而是 Qualcomm robot bring-up 的验证记忆。

Jetson 像“默认机器人开发者信心”。BoardBringupKit 的目标是给 Qualcomm robotics hardware 建立同等信心，但从第一天就带 production-oriented artifacts。

Compatibility Matrix

board、carrier、camera、LiDAR、motor controller、power stack、ROS distro、kernel/BSP 组合。

HIL Fixture

first boot、rail/current、USB/MIPI、CAN/Ethernet、thermal soak、camera timing、motor loop。

Manifests

machine-readable 配置生成 udev、ROS launch、diagnostics、factory tests、wiring report 和 deploy manifest。

Golden Images

可复现 build、OTA hooks、secure defaults、logs、health topics、rollback 和 image diff。

Failure Corpus

每个失败项、驱动版本、BSP patch、传感器组合和客户现场问题都会增加路径依赖。

Partner Loop

Qualcomm、Radxa、APLUX、Thundercomm、Advantech、SI、传感器厂和机器人团队共同更新验收语言。

09 · Why Qualcomm

Qualcomm 的机会不是再发一块开发板，而是让 Dragonwing board 更快进入机器人产品化标准。

BoardBringupKit 把 AI、camera/multimedia、low-power edge、Linux/BSP、AI Hub/QNN 和 ROS guidance 放进真实机器人控制闭环。比赛中它是可行性证据；商业上它是生态 adoption 工具。

  • RB3 Gen 2 / QCS6490：practical first target，官方 dev kit、Ubuntu 24.04、AI Hub、Edge Impulse、camera-first I/O、QIR/ROS path。
  • Dragon Q6A / QCS6490：比赛 immediate target，可作为 Core Lite 和 fallback。
  • Rhino Pi-X1 / QCS8550：premium compute target，适合多相机、高性能边缘 AI 和 RobotMac Core Pro；QCS8550 profile 常见为 proxy，必须标注。
  • IQ-9075：high-end industrial path，ECC memory、2.5GbE、CAN-FD、PCIe、MIPI CSI，按 partner/sampling path 谨慎推进。
  • IQ10 RRD：enterprise roadmap / early-access ask，不写成 2026 年 7 月可交付比赛硬件。
