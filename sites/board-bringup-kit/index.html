Pitch 21 · Board-to-Robot Commercialization Layer

BoardBringupKit

开发板不是机器人产品。BoardBringupKit 把 Rhino Pi-X1 / QCS8550、Dragon Q6A / QCS6490 等 Qualcomm 板卡接入电源、相机、I/O、安全、ROS 2、LeRobot 和 runtime 证据，变成可复现、可验收、可交付的 RobotMac Core 候选核心。

看开发板风险 看三分钟演示

01 · Problem

比赛团队和机器人创业公司真正缺的不是板卡，而是把板卡变成机器人核心的确定路径。

收到开发板后，时间通常花在电源、热、摄像头、USB、CAN/UART、ROS 2 lifecycle、E-stop、watchdog、image digest 和 runtime 版本上。一个松动的线束、错误的电平或静默降频，都可能让 demo 失败，更不用说企业试点和量产迁移。

Power

供电不是插电

峰值电流、分路供电、USB 电流、保险、状态灯和 undervoltage flags 决定机器人是不是会随机重启。

Camera

相机不是画面

MIPI lane、USB 2/3、V4L2、GStreamer、frame drop、标定和时间戳会直接影响 LeRobot episode。

I/O

接口不是针脚

GPIO、CAN、UART、PWM、I2C、SPI 的电平、transceiver、terminator 和 pinmux 必须被测试。

ROS

节点能跑不等于系统可靠

ros2_control lifecycle、QoS、topic rate、diagnostics 和启动顺序需要工程门禁。

Safety

安全不能赛后补

硬急停、software watchdog、command timeout、soft boundary 和 manual takeover 必须进入第一版线束。

Evidence

没有证据就不能复现

OS、kernel、BSP、firmware、QNN/QAIRT、LeRobot、dataset hash 和 image digest 必须锁定。

02 · Current Alternatives Fail

文档、教程、机器人套件和外包服务都重要，但没有把 first robot bring-up 产品化。

BoardBringupKit 不替代 Qualcomm AI Hub、vendor BSP、ROS 2、Jetson/Isaac、整机平台或 fleet tools。它把这些分散能力组织成一套可执行验收流：从裸板、线束、镜像、传感器、I/O、安全到 runtime evidence。

Vendor Docs

解释接口和 SDK，但很少覆盖机器人线束、E-stop、sensor timing、ROS graph 和客户验收包。

ROS Tutorials

能让节点通信，却不会验证电源、热、相机 lane、USB bandwidth、CAN transceiver 和生产镜像。

Jetson / Isaac

生态强，但生产 carrier、pinmux、device tree、BSP、thermal 和 validation 仍然是团队自己的责任。

Robot Kits

Clearpath、Husarion、AgileX、Unitree、Elephant 强在整机或教育，不解决客户自研 compute stack。

Fleet Tools

Foxglove、Formant、InOrbit 从机器人已经存在之后开始，不能替代 first boot 和硬件 fixture。

One-off SI

能救一个项目，但难沉淀成 board recipe、golden image、HIL fixture 和可复用 evidence corpus。

03 · Solution

BoardBringupKit 是 Qualcomm board-to-robot 的可执行验收层。

它把开发板、线束、相机、I/O、安全、运行时和证据导出做成同一套 workflow。目标不是让一个 demo 碰巧跑起来，而是让每次 board、image、sensor、runtime 变化都能重新验收。

Profile

锁定 SoC、board revision、OS、kernel、BSP、firmware、runtime、image digest 和 dependency lock。

Harness

电源、E-stop、camera mount、USB/CSI、CAN transceiver、IO terminal、level shifter 和状态灯。

Gates

first boot、camera frame、USB speed、actuator ping、ros2_control lifecycle、safety trigger。

Runtime

LeRobot episode、policy replay、AI Hub/QNN/ONNX profile hook、CloudTwin agent、SkillDock package。

Evidence

生成 board bring-up report、wiring map、safety ledger、runtime image lock 和 judge audit pack。

2026 年高通具身智能与机器人开发者大赛把 QCS6490、QCS8550 和 IQ 系列推到开发者面前。复赛从 8 月到 11 月 1 日，底层 bring-up 的每一天都会挤压数据采集、训练、演示和商业包装。

Q6A Radxa Dragon Q6A / QCS6490 是最现实 immediate target，可做 Core Lite 和 fallback。
X1 Rhino Pi-X1 / QCS8550 是标准赛道高性能目标，可承载 LabForgePilot 主 demo。
48h 产品目标是把开箱到 first camera、first actuator、first episode、first edge run 压到 48 小时内。
IQ10 IQ10 RRD 只作为 enterprise roadmap / early-access ask，不写成 2026 年 7 月承诺。
Evidence 企业试点要的是 wiring、image、power、thermal、safety、runtime 和 rollback 证据。

05 · Product

不是一套线，而是一个 bring-up operating system。

BoardBringupKit 把每块板分成明确状态：Ready、Sensor Risk、I/O Risk、Runtime Risk、Safety Blocked、Needs BSP Fix。每个状态都能追到测试命令、log 和硬件照片。

  • Board Recipe Library：Q6A、Rhino X1、RB3 Gen 2、VENTUNO Q 的 manifest、known-good image、接口图和版本锁。
  • Harness BOM：电源、E-stop、camera mount、USB/CSI cable、CAN transceiver、IO terminal、level shifter。
  • First Boot Runner：boot、network、storage、clock、thermal、fan、USB speed、serial log、kernel flags。
  • Sensor Gate：camera enumerate、V4L2/GStreamer capture、frame age、drop rate、calibration snapshot。
  • Actuator Gate：机械臂/夹爪/底盘 ping、single-axis move、command timeout、ros2_control lifecycle。
  • Evidence Export：judge/enterprise audit pack、factory bring-up report、SI handoff packet。

06 · Product API

bring-up 证据必须能被评委、SI、客户和供应链同时读懂。

每项指标标注来源：`measured-on-board`、`vendor-doc`、`proxy`、`simulated` 或 `blocked`。诚实标注比假装全链路稳定更可信。

board-bringup-manifest

board、SoC、serial、OS、kernel、BSP、image digest、runtime versions、container digest。

wiring-map

power rails、camera、I/O、E-stop、actuator、network、safety boundary 和照片索引。

power-thermal-log

input voltage、current estimate、temperature、throttling flags、fan state、load scenario。

camera-gate-report

device path、driver、resolution、FPS、drop rate、latency、calibration image hash。

robot-io-report

CAN/UART/GPIO/PWM/I2C/SPI test、motor ping、ros2_control lifecycle state。

safety-event-ledger

E-stop、watchdog、command timeout、soft-boundary deny、manual takeover 和 rollback pointer。

07 · Market & Business Model

硬件便宜，失败的 bring-up cycle 很贵。

BoardBringupKit 不靠卖板卡赚钱，而是在节省 2-8 周 BSP、driver、power、thermal、sensor、ROS 和 safety debug 的地方收费。它可以先服务比赛团队，再扩展到机器人创业公司、SI、板卡厂商和企业实验室。

中国版

赛事 + 高校实验室 + 机器人集成商

围绕 Q6A/Rhino X1、桌面机械臂、低成本相机、国产云训练和本地 SI 交付。强调 48 小时 first robot、中文文档、现场支持和可复用线束。

海外版

startup + OEM prototype + certification readiness

围绕 RB3/QCS6490、QCS8550 SOM、Jetson 对照、ROS 2/HIL、fleet telemetry 和 pre-certification evidence。强调 audit pack 和 production migration。

¥3,999-19.8k Builder Kit：镜像、BOM、诊断、模板和基础远程支持。
¥50k-180k Pilot Sprint：一块板、一台机器人、一套 sensor/IO 的 bring-up。
$75k-200k Production Pack：BSP/device-tree、HIL fixture、factory flash、cert-readiness。
$10k-25k/y Team Pro：CI/HIL、兼容矩阵、版本回归和远程支持。
NRE 与 SOM/dev-board 厂商共建 reference robot recipe 和 co-sell package。

08 · Competition & Moat

壁垒不是线束，而是跨板卡、传感器、驱动、现场和结果的验证语料库。

机器人套件、开发板、ROS、fleet tools 和外包服务会长期存在。BoardBringupKit 的机会是站在 productization gap 上，把每个 board recipe、BSP fix、sensor failure 和 safety gate 沉淀成可复用资产。

Compatibility Matrix

board、carrier、camera、LiDAR、motor controller、power stack、ROS distro、kernel/BSP 组合。

HIL Fixture

first boot、rail/current、USB/MIPI、CAN/Ethernet、thermal soak、camera timing、motor loop。

Manifests

machine-readable 配置生成 udev、ROS launch、diagnostics、factory tests 和 deploy manifest。

Golden Images

可复现 build、OTA hooks、secure defaults、logs、health topics、rollback 和 image diff。

Evidence Corpus

每个失败项、驱动版本、patch、传感器组合和现场问题都增加路径依赖。

Partner Loop

板卡厂、SI、传感器厂、机械臂厂、比赛团队和企业客户共同更新 acceptance language。

BoardBringupKit 把 Qualcomm 的 AI、camera/multimedia、low-power edge、Linux/BSP、AI Hub/QNN 和 ROS guidance 放进真实机器人控制闭环。比赛中它是可行性证据；商业上它是生态 adoption 工具。

10 · Demo & Ask

三分钟只证明一件事：开发板已经变成可复现机器人核心。

演示不能只拍基础设施，必须服务 LabForgePilot：样品识别、抓取、放置、复核、扰动、安全拒绝和证据导出。BoardBringupKit 解释为什么这件事可以复现。

0:15

裸板、线束、相机、机械臂、E-stop 同框，打出“开发板不是机器人产品”。

0:40

Bring-up runner 过门禁：boot、network、camera frame、actuator ping、E-stop event、image digest。

1:20

LabForgePilot 执行样品识别、抓取、放置、复核，明确 cloud not in control loop。

2:20

扰动：错样本、挪动目标或 unsafe command；系统 pause、deny、recover 并写入 ledger。

2:50

导出 `board-bringup-audit-pack.zip`：profile、wiring、camera、I/O、safety、runtime、rollback。

Fallback

板卡或 NPU 延迟不致命

如果主板晚到，用 Q6A/RB3 或 Linux proxy 跑 perception + one safe action，并把迁移清单写入 evidence。

Artifacts

评委可检查文件

`board-bringup-manifest.json`、`wiring-map.svg`、`camera-gate-report.json`、`safety-event-ledger.jsonl`。

Ask

向高通要资源

目标板、AI Hub quota、QNN/QAIRT office hours、camera/GStreamer/ROS guidance、IQ10 roadmap review。

BoardBringupKit 的目的，是让“收到开发板”不再是项目风险，而是 RobotMac Core 工程样机启动的第一天。
