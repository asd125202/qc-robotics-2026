Pitch 34 · Mixed Robot Fleet Orchestration

FleetConductor

让工厂、仓库、医院和实验室里的每台机器人，遵守同一张地图、同一套交通规则、同一个任务节拍。Qualcomm 边缘塔台本地运行，云端只做训练和优化增强。

查看机器人塔台 连接 OpsConnector

01 · Edge Tower Runtime

机器人越多，现场越需要一座本地塔台。

FleetConductor 不是又一个远程 dashboard，而是现场边缘运行时：任务队列、交通调度、地图服务、fleet registry、incident manager 和 event log 都在本地保持在线。

Order

WMS/MES/ERP/LIMS/HIS 任务进入统一队列。

Allocate

按能力、位置、电量、payload、SLA 选择机器人。

Schedule

走廊、门、电梯、充电桩和装卸位进入资源时刻表。

Operate

本地边缘执行，云端断开也能继续运行和记录。

Learn

成功、失败和人工接管进入 DataFlywheel。

02 · Adapter Hub

不替代厂商大脑，连接所有车队系统。

MassRobotics 适合共享状态，VDA 5050 适合 master-control 到 AGV/AMR 的订单和状态，Open-RMF 适合多车队与门梯充电等共享资源。FleetConductor 用 adapter 把这些接入企业流程。

VDA 5050

MQTT/JSON order、state、connection、factsheet、zone 和 map adapter。

Open-RMF

fleet adapter、traffic schedule、door/lift/charger adapter 和 task bidding。

MassRobotics

identity、location、speed、health、availability 和 shared awareness。

Facility API

REST、MQTT、OPC-UA、BACnet、WMS、MES、ERP、BMS 和厂商 SDK。

03 · Live Facility Map

地图不是背景图，是可执行的交通规则。

走廊、会车点、禁行区、门、电梯、装卸点、充电位和等待点都进入同一个资源模型。任务不只是派发，而是被安全编排、预约、冲突检测和可回放。

Lanes

单向、双向、限速、会车和临时封控。

Doors

通过授权设施系统请求开门和记录结果。

Elevators

楼层、等待队列、占用窗口和失败回退。

Chargers

低电任务插入、排队、兼容桩和充电窗口。

Zones

禁区、慢行区、人机混行区和权限边界。

Replay

事件日志、轨迹、视频快照和根因时间线。

04 · Incident Command

异常不是弹窗，是处置流程。

低电、堵塞、急停、传感器遮挡、门梯失败和任务超时自动进入事件时间线，触发重路由、隔离、人工接管、证据留存和复盘包。

  • 定位为运营安全和可追溯层，不替代机器人本体安全控制器。
  • 异常视频、状态、轨迹和人工接管片段默认留在现场边缘。
  • 云端同步摘要、精选 incident clips、LeRobot 数据集和优化指标。
  • 恢复后回灌 DataFlywheel，形成训练、仿真、评估和边缘部署闭环。

05 · Qualcomm Role

从单机 edge AI，升级到设施级机器人操作系统。

FleetConductor 把 Qualcomm 放在现场协同核心：多媒体、连接、边缘 AI、低云依赖、模型部署和本地事件闭环共同构成可交付的 factory edge tower appliance。

Edge 本地任务、交通、地图、事件和日志运行，不依赖云端往返。
Video 现场视频、远程接管、异常快照和隐私边界在边缘处理。
Network Wi-Fi、private 5G、VLAN、mTLS、DDS Security 和 OT 分段。
Flywheel 轨迹、失败、人工接管和复盘进入 LeRobot 数据飞轮。

FleetConductor 不承诺“一键兼容所有机器人”，它卖的是一座现场塔台：连接现有机器人、设施系统和企业流程，让多品牌车队少堵车、少停机、更可追溯。
