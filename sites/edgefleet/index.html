Pitch 06 · Fleet Evidence Control Plane

EdgeFleet

机器人商业化的瓶颈正在从“能不能训练”转向“能不能运营”。EdgeFleet 把 LeRobot 数据、AI Hub/QNN 产物、Dragonwing 设备、灰度发布、远程协助、事故回放和训练回流串成一条 Qualcomm-first 队列证据链。

从上线风险开始 连接 DragonWorks

01 · Problem

客户不买 demo 视频，客户买可运营、可回滚、可审计的机器人劳动。

一台机器人能跑一次已经不够。企业真正追问的是：哪台机器在跑哪个模型？这次失败来自传感器、模型、网络、现场还是操作员？更新后表现变差能否回滚？事故前后有没有证据？这些问题回答不了，机器人就停在试点。

风险 1

版本黑箱

技能、模型、QNN artifact、OS image、地图、相机和参数分散记录，现场没人能还原真实状态。

风险 2

更新失控

云端训练的新策略直接上线，缺少 shadow、canary、健康门、previous-known-good 和 rollback 证据。

风险 3

事故不可复盘

视频、ROS topic、MCAP、操作员动作、模型 hash 和设备状态没有被打包成可校验时间线。

风险 4

远程协助无边界

远程接管容易变成“人在公网开机器人”，没有 TTL、权限、速率限制、本体安全门和审计日志。

风险 5

现场经验不回流

失败片段、低置信度、人工接管和站点瓶颈没有进入下一轮 LeRobot 数据集与训练任务。

02 · Why Now

机器人从试点走向队列，运维证据会先商业化。

IFR 2025 报告显示，2024 年全球工业机器人新增安装约 54.2 万台，中国占全球新增部署约 54%；专业服务机器人销量接近 20 万台，RaaS fleet 增长 31%。LeRobot Dataset v3 把多模态 episode、传感器信号、多相机视频和元数据标准化；Qualcomm AI Hub Workbench 可以把模型 compile/profile 到目标设备。市场需要的下一层，是从 checkpoint 到客户现场的证据控制平面。

542K IFR：2024 年全球工业机器人新增安装约 54.2 万台。
RaaS +31% IFR：专业服务机器人租赁/订阅队列增长，买家正在接受运营化采购。
LeRobot v3 多模态时间序列、传感器动作、视频和元数据给训练血缘提供基础。
AI Hub/QNN 边缘延迟、内存、NPU 使用和 runtime profile 可以成为上线门禁。

03 · Solution

EdgeFleet 是 Qualcomm-powered robots 的 fleet evidence control plane。

它不替代 OEM fleet manager、ROS 2、Open-RMF、Viam、Formant 或 Foxglove。它站在生产发布链路上，记录“这个技能为什么可以上线、在哪些 Dragonwing 设备上运行、出了问题如何复盘、下一轮训练如何改进”。

Register

注册 robot profile、Dragonwing/QCS/IQ target、OS image、ROS graph、sensor、actuator 和证书。

Release

绑定 LeRobot dataset、training run、model digest、AI Hub profile、QNN/QAIRT artifact 和 SkillCertKit gate。

Rollout

按站点、机型、任务和风险级别做 shadow、canary、staged rollout、pause 和 rollback。

Operate

采集 autonomy rate、intervention rate、latency、thermal、battery、QoS、operator session 和 incident。

Learn

把失败片段、接管片段和现场标签回流到 TeleopStudio、TrainRouter 和下一版 LeRobot dataset。

04 · Product

最核心的产品不是看板，而是 incident-to-training evidence loop。

每台机器人运行 `edgefleet-agent` 作为 ROS 2 旁路代理：不替代本体 autonomy stack，不绕过安全控制器，只采集、门禁、协助、回放和回流。

  • `fleet-event.schema.json`：CloudEvents + OpenTelemetry 事件，包含 robot、skill、model、artifact、operator、safety state。
  • `skill-release.manifest.json`：dataset -> checkpoint -> AI Hub/QNN profile -> release cohort -> robot instance。
  • `incident-capsule.mcap`：触发前后 5-15 分钟 ROS/MCAP 缓冲、视频片段、控制命令、地图和参数快照。
  • `remote-assist.audit.jsonl`：WebRTC 会话、TTL、操作员身份、命令白名单、速率限制、本体 safety gate。
  • `rollback-record.json`：停止新技能、恢复 previous-known-good、运行 smoke test、发布 rollback completed 事件。

05 · Market & Business Model

先卖可用率、远程运维和证据，再扩成机器人劳动平台。

第一批买家不是单店和个人开发者，而是已经把机器人放进生产、试点或售后体系的人：AMR/物流、巡检、清洁/物业、能源、制造、人形试点、机器人 OEM、系统集成商和 RaaS operator。

中国版

OEM/SI 出海运维层

绑定中国机器人厂商、系统集成商和行业渠道，按设备/月、站点/年、私有化和白标收费，强调内网、国产云、企微/钉钉、权限审计和本地服务。

海外版

3PL/能源/制造 fleet ops

面向 5-50 台 commercial pilot 和 50-500 台 rollout，按 robot/month、site/month、integration、evidence retention 和 remote ops 收费。

平台化

从观测到生产链路

先做 fleet observability 和 incident capsule，再接 task orchestration、remote assist、SLA report、CMMS/WMS/ERP 和 outcome billing。

渠道

让集成商赚持续收入

集成商不只赚一次性交付，EdgeFleet 给他们售后订阅、客户门户、远程诊断、升级记录和 SLA 复盘包。

06 · China / Global Stack

同一套产品抽象，两套隔离部署。

中国版默认独立 control plane，账号、密钥、遥测、视频、日志、模型输入输出、对象存储和审计留在中国区；全球版使用 Cloudflare 做入口和安全边界，AWS/GCP/Azure 与 GPU cloud 做控制面、训练和企业集成。

中国 POC

阿里云 / 腾讯云

适合快速完成 ICP、K8s、对象存储、CDN/WAF、AI 服务、企微/钉钉集成和本地客户演示。

强合规

华为云 / 私有化

面向工业、政企、园区和等保协同，输出系统边界、资产清单、日志留存、访问控制和测评材料。

全球

Cloudflare + Hyperscaler

Cloudflare 负责 DNS/CDN/WAF/Zero Trust，控制面和企业集成走 AWS/GCP/Azure，GPU 训练走弹性 provider。

边界

跨境只同步低敏摘要

默认不同步原始视频和工业遥测，只同步匿名指标、版本号、artifact hash、模型性能和非敏感运营摘要。

07 · Competition & Moat

EdgeFleet 不做又一个 RobOps dashboard，而做发布证据和现场复盘的系统记录。

Formant、Viam、Foxglove、InOrbit、Boston Dynamics Orbit、NVIDIA Isaac、Open-RMF、MassRobotics、VDA 5050、中国 AMR/OEM 云平台都证明机器人运维有预算。EdgeFleet 的切入点是 Qualcomm-first edge evidence、LeRobot lineage、技能发布记录、远程协助审计和 incident-to-training loop。

短期壁垒

release evidence schema

把 model、skill、QNN artifact、robot profile、canary cohort、incident 和 rollback 放进同一证据图。

中期壁垒

connector compound

WMS/MES/CMMS/ERP、ROS 2、Open-RMF、VDA 5050、MassRobotics、OEM API 和 charger/door/elevator adapters。

长期壁垒

field incident memory

每次失败、接管、回滚、热降频、QoS deadline miss 和客户复盘都变成下一版门禁规则。

生态壁垒

Qualcomm hardware profile library

Dragonwing / RB / QCS / IQ 设备的 runtime、camera、NPU、thermal、network 和离线运行证据持续积累。

08 · Why Qualcomm

EdgeFleet 把 Qualcomm 从“能跑模型的边缘芯片”提升为“能运营机器人队列的默认底座”。

Qualcomm 的竞争位置不应只停在单机 benchmark。EdgeFleet 让每次客户上线都带着 Dragonwing target、AI Hub/QNN evidence、Device Cloud/Profile job、离线运行、远程诊断、回滚记录和现场失败回流。

Attach 每个 reference workflow 都绑定 Dragonwing / RB / QCS / IQ 设备和边缘 runtime profile。
Adopt AI Hub / QNN / QAIRT 从开发工具变成上线门禁和客户证据。
Prove 比赛 demo 不只展示推理，而展示灰度、回滚、断网、远程协助和事故回放。
Scale 中国数据驻留版、全球多云版和离线私有版形成企业级参考架构。
需要高通支持

Fleet validation sprint

6-8 周用 2-3 台 Qualcomm target 跑通 skill install、canary、incident replay、rollback 和 QNN profile。

需要高通支持

AI Hub / Device Cloud quota

把当前网页证据替换为真实 compile/profile/run logs，并标注 SDK、板卡、散热、电源和模型版本。

需要高通支持

09 · Demo & Evidence

三分钟演示要证明：机器人出了问题，我们知道发生了什么、能回滚、还能训练下一版。

初赛网页可以模拟完整证据；复赛拿到开发板后补真实 Qualcomm profile、端侧日志、视频、功耗、温度和回滚记录。

Enroll

注册三台机器人：一台 canary、一台 stable、一台离线缓存。

Release

上传 `dock-assist-v2`，展示 LeRobot dataset、QNN profile、SBOM、签名和兼容矩阵。

Canary

只有电量、任务空闲、硬件 profile 和 safety gate 通过，才允许激活。

Incident

触发低置信度或 QoS deadline miss，生成 MCAP、OTel、CloudEvents 和 operator audit。

Rollback

回退到 previous-known-good，并把失败片段送回 TrainRouter。

可讲

audit-ready

EdgeFleet 帮助生成上线、运维、远程协助和事故复盘证据。

谨慎

不是安全认证

它支持安全案例准备和审计，不替代 ISO/IEC/UL/TUV、现场风险评估或法律责任判断。

边界

不是万能 fleet controller

MVP 先支持明确 robot profile、ROS 2/MCAP、指定 Qualcomm target 和受限技能类别。

EdgeFleet 的一句话：把“机器人上线了”变成“每个模型、每次发布、每次失败、每次回滚都有 Qualcomm-first 证据”。
