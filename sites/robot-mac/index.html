Pitch 01 · Qualcomm-first Robot Computer

RobotMac Core

装进商业机器人身体里的“机器人电脑”：把 Qualcomm 边缘计算、电源、机器人 IO、安全边界、RobotCoreOS、LeRobot 云训练接口和 fleet runtime 打包成一体。

01 · Problem

机器人团队不缺开发板，缺的是可交付的产品底座。

开发板解决“能跑”，不解决“能卖”。每个团队仍要重复处理供电、散热、相机、CAN/UART/PWM、急停、ROS 2、模型部署、日志、OTA、回滚、设备身份和现场恢复。买方真正担心的是版本不可追溯、更新不可恢复、安全边界不清和 20 台机器没人维护。

电源先绊倒团队

电池、外部 DC、稳压轨、逻辑/电机隔离、保险丝、急停和状态灯常被当作样机小事，最后变成现场事故源。

IO 每次重做

相机、以太网、USB、CAN、UART、PWM、GPIO、IMU、LiDAR、ToF 和电机控制没有统一产品边界。

模型难落地

云上训练出的 policy 要落到本体，还要处理输入规格、QNN/QAIRT、fallback、温度、功耗和延迟。

安全边界分散

急停、watchdog、限位、门禁、云断连、热失控和低电压不应该散落在应用代码里。

现场不可运营

没有 OTA、rollback、device identity、日志、SBOM 和恢复镜像，企业客户很难把试点扩大成队列。

生态无法复用

没有统一 robot core，技能包、课程、集成商模板和行业套件都很难跨形态复用。

02 · Why Now

机器人正在从“能做动作”进入“能被采购、部署和维护”。

LeRobot 把真实机器人数据、模仿学习和数据集流程推向标准化；机器人队列和 RaaS 让 uptime、远程诊断、灰度更新和审计变成采购问题；Qualcomm Dragonwing、AI Hub、QNN/QAIRT、Qualcomm Linux、Device Cloud 和 IQ10 reference design 正在形成从 prototype 到 production 的边缘 AI 路径。

市场已经够大

IFR 报告 2024 年工业机器人安装量约 54.2 万台，全球运行存量约 466.4 万台，服务机器人也在持续增长。

中国是主战场

中国占 2024 年全球工业机器人安装量过半。一个中文、双云、Qualcomm-first 的机器人核心有本土落地价值。

开发者要省时间

高校、系统集成商和机器人创业公司愿意为 ready-to-develop 套件付费，因为它省掉机械、电气、OS、ROS 和远程支持时间。

Qualcomm 有窗口

NVIDIA 已经把 robot brain 讲清楚。Qualcomm 需要一个低功耗、camera-rich、connected edge 的机器人产品化样板。

03 · Insight

机器人应用的最小商业单元，不是模型，也不是底盘，而是可验证 runtime slot。

这个 slot 必须同时声明电源预算、IO 拓扑、传感器时序、模型 hash、动作权限、安全边界、运行日志、OTA、rollback 和现场恢复路径。PC 普及不是因为每个人会装主板；机器人普及也不会靠每个团队重新做电源、驱动和运行时。

04 · Solution

RobotMac Core 是 Qualcomm-first 的机器人电脑，不是另一块裸板。

它把 edge compute、protected power plane、robot IO plane、safety supervisor、RobotCoreOS、LeRobot / AI Hub deployment bridge 和 fleet runtime 放进一个产品边界。用户接上机器人身体，就能围绕任务、数据和技能迭代，而不是从底层硬件开始。

Product target

Compute

Qualcomm Dragonwing edge AI module，承载视觉、策略推理、日志、设备身份和本体运行时。

Product target

Power

电池/外部 DC、受保护稳压轨、逻辑/电机隔离、过流/低压监控、状态灯和现场恢复供电策略。

Competition target

Robot IO

USB/Ethernet/camera baseline，UART/I2C/SPI/PWM/GPIO 经 adapter；CAN/CAN-FD 作为工业或扩展 profile。

Product target

Safety IO

物理急停、watchdog、限位/门/bumper 输入、cloud-disconnect safe state 和 local safety service。

Competition target

Runtime

RobotCoreOS、ROS 2 bridge、QNN/QAIRT/ONNX path、signed skill package、telemetry、OTA 和 rollback。

Future profile

Industrial

IQ-8275 / IQ10 路线扩展多摄像头、实时 MCU、安全 island、GMSL2、工业温度和长生命周期。

05 · Product Workflow

从选机器人形态到导出运行证据，一条闭环。

RobotMac Core 的产品体验应该像电脑开箱，而不是工程板 bring-up。它先让机器可接、可测、可录、可训、可部署，再让客户把失败 episode 变成下一轮技能改进。

选择形态

桌面机械臂、移动底盘、巡检车、教学套件或行业夹具。

接入 Core

电源、相机、执行器、急停、网络和传感器一次接好，运行硬件自检。

采集数据

LeRobotDataset 记录视频、状态、动作、人类示教、接管和失败片段。

云端训练

CloudTwin/TrainRouter 调度中国或海外 GPU，生成评估报告和模型 artifact。

边缘部署

通过 AI Hub / QNN / QAIRT / ONNX 路径 profiling，签名成 skill package 回到本体。

本地运行

本体执行感知、策略、安全检查和控制，云端不进入 safety-critical loop。

记录证据

输出 FPS、p95 latency、placement、memory、power、temperature、success 和 safety events。

异常回滚

坏模型、坏配置或传感器异常触发 admission gate、safe mode 或 rollback。

回流训练

失败片段、低置信度和人工接管进入下一轮数据与策略训练。

导出 Manifest

SoC、OS、IO 拓扑、模型 hash、功耗、温度、rollback、SBOM 和审计记录。

06 · Market

第一批买家，是不想再花 3-6 个月做基础集成的人。

RobotMac Core 不该拿自己和一块低价 SBC 比。买方真正比较的是几个月系统集成、现场失败、定制 OTA、远程支持缺口和企业采购风险。

机器人 OEM

需要 reproducible hardware profile、BSP、ROS 2 image、drivers、OTA rollback 和 telemetry。

系统集成商

需要 known-good core、harness map、ROS packages、远程支持和可重复安装 checklist。

RaaS 运营方

需要 uptime、少出现场、远程诊断、staged OTA、incident replay、teleop hooks 和 SLA evidence。

企业 IT / 安全

需要 identity、patching、SBOM、CVE、访问控制、审计日志和 lifecycle support。

高校与开发套件

需要 day-one working、课堂重置、示例技能、安全 IO、课程和可复用实验环境。

行业试点客户

工厂、仓库、实验室、园区和服务机器人试点需要先把 5-50 台机器可靠运营起来。

07 · Business Model

硬件建立入口，runtime 和 fleet 才是长期收入。

RobotMac Core 的价格要覆盖它节省的工程拖累：硬件、供电、IO、安全、OS、ROS、QNN deployment、OTA、日志、SBOM 和现场恢复。业务上应从套件收入走向设备授权、fleet 订阅、云训练和企业集成。

Packaging

Packaging

Packaging

Recurring

按设备授权和按机器人/月订阅，覆盖 signed image、profile、rollback、telemetry、OTA stages、logs、diagnostics 和 audit。

08 · GTM

先做 Qualcomm robot core starter kit，再进入 SI 和行业试点。

比赛不是最终销售渠道，但可以制造可信样板：用 Core Pro 原型证明采集、训练、部署、运行、异常回滚和 runtime manifest；随后用课程、开发套件和系统集成商模板扩大复用。

比赛样板

教育入口

课程、示例技能、刷机恢复、课堂安全和 LeRobot-to-edge 教学闭环。

SI 模板

把 power/IO/safety/runtime 做成可重复交付包，减少系统集成项目脏活。

90 天试点

交付 3-5 台 Core、2 个技能、fleet dashboard、rollback demo 和验收报告。

双云销售

09 · Competition

不要打“谁的板子更便宜”，要打“谁能把机器人变成可交付产品”。

竞品已经证明每个层都有人愿意买：Jetson/Isaac 证明 robot brain 叙事，ROS 证明中间件，Viam/Formant/Foxglove 证明 fleet/data，Mender/Balena/Ubuntu Core 证明 OTA/OS。空位是把这些压进 Qualcomm-first 的 robot core 产品边界。

NVIDIA Jetson / Isaac

强 GPU 与 robotics 软件生态；RobotMac 避免 TOPS 战，强调低功耗 connected edge、power/IO/safety 和训练闭环。

ROS 2

事实标准中间件；RobotMac 不替代 ROS，而是把 ROS 放进可采购、可维护、可回滚的产品边界。

Viam / Formant

强 robot ops、fleet、telemetry 和 cloud control；RobotMac 从本体侧 compute、IO、power、runtime 开始。

Foxglove

强机器人数据可视化；RobotMac 默认生成同步 sensor/action episodes、runtime metrics、safety events 和 model versions。

Unitree / AgileX

卖具体机器人本体或 ROS 套件；RobotMac 卖跨形态核心，可插入机械臂、小车、巡检车和教育平台。

工业控制器 / PLC

可靠、安全、现场工程强；RobotMac 借鉴 trust vocabulary，但服务新一代 AI 学习型机器人。

10 · Moat

RobotMac Core 的复利不是外壳，而是每一次 bring-up、每一种 IO、每个模型部署、每次失败回滚和每个行业模板形成的可复用证据。

Profiles Qualcomm board、camera、IO、sensor timing、thermal、power 和 accelerator 路径验证矩阵。
Skills 技能包与硬件、ROS graph、runtime、动作权限、安全 envelope 和 rollback 的兼容关系。
Evidence SBOM、签名、OTA、rollback、latency、thermal、power、episode 和 incident report。
Channels 比赛、课程、开发套件、系统集成商和 Qualcomm 生态伙伴形成复用网络。

12 · Why Qualcomm

RobotMac Core 把 Dragonwing 从开发板变成机器人产品核心。

Qualcomm 的优势不只是 TOPS，而是低功耗异构 AI、多摄像头/多媒体、连接、端侧隐私、长期供货和 prototype-to-production 工具链。RobotMac Core 把这些能力组织成可购买、可演示、可迁移的机器人核心。

QCS8550 / Core Pro

适合作为比赛主线和性能型 service robot/desktop arm demo，承载 vision、policy、runtime 和数据记录。

RB3 Gen 2 / QCS6490

适合 Core Lite：robotics、AI vision、smart security、multi-OS、camera/USB/Ethernet/GPIO 和 AI Hub 入口。

IQ-8275 / VENTUNO Q

适合 Core Industrial：AI 负责感知与策略，MCU/real-time path 负责确定性动作和工业 IO。

AI Hub + QNN

把 LeRobot/ONNX policy artifact 变成可 profile、可部署、可记录 runtime evidence 的 Qualcomm edge package。

13 · Demo Story

三分钟证明：这不是开发板实验，而是产品底座。

比赛 demo 选择桌面机械臂抓取/分拣或移动底盘巡检。重点不是炫技动作，而是展示从硬件自检、LeRobot 数据、云训练、Qualcomm edge deployment 到坏模型回滚的商业闭环。

Open

展示 RobotMac Core appliance、board profile、image digest、device identity、power/IO/safety map。

Capture

记录 LeRobot episode：camera frames、robot state、actions、success/failure labels、human takeover。

Train

CloudTwin/TrainRouter 提交 GPU job，生成 model artifact、dataset lineage 和评估报告。

Deploy

AI Hub/QNN/QAIRT 或 ONNX path profile 后，签名成 skill package 推送到本体。

Recover

坏模型或传感器异常触发 admission denial、safe mode 或 rollback，失败 episode 回到训练队列。

14 · Ask

请求 Qualcomm 支持一个 6-8 周 Dragonwing Robot Core validation sprint。

我们不需要泛泛赞助，而需要让比赛原型变成可验证 reference project：硬件访问、AI Hub/QNN 工作流、Qualcomm Linux/BSP 指导、Profiler evidence、Device Cloud quota 和 Robotics Hub 发布支持。

  • 硬件：Rhino X1/QCS8550、RB3 Gen 2 Vision Kit、Radxa Q6A 或 VENTUNO Q 测试路径。
  • Runtime：LeRobot/ONNX policy artifact 到 QNN/QAIRT 的转换、profiling、fallback 和 manifest 格式。
  • Linux：camera、GStreamer、ROS 2、CAN/UART/GPIO、OTA 和 local safety services 的 BSP 指导。
  • 证据：CPU/GPU/DSP/NPU、memory、thermal、latency、power 和 long-run stability 的 Profiler 方法。
  • 生态：Dragonwing Robotics Hub reference runtime project、partner intro 和 developer tutorial 机会。

15 · Claim Boundaries

RobotMac Core 是机器人电脑产品架构，不是已认证的成品机器人。

我们可以说它是 Qualcomm-first compute/power/IO/runtime 原型，目标是降低从 demo 到部署的工程成本。不能说已获 Qualcomm 官方认证、官方合作、官方投资、安全认证、IP rating、零停机、开箱支持所有机器人，或性能优于 Jetson/x86。所有 TOPS、功耗、延迟、温度和加速路径必须按最终开发板、SDK、散热、电源和模型版本实测标注。

如果个人电脑的价值是让用户不再从主板开始，RobotMac Core 的价值就是让机器人团队不再从开发板、电源和运行时开始。
