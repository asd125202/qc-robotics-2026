Pitch 35 · Hospital Operations Robot Workflow

CareOps

院内智运平台：让样本、药品、耗材和设备在医院里自动流转，并留下可审计证据。它不做诊断、不替代医嘱，只优化医院后勤与实验室支持流程。

01 · Care Flow

把跑腿交给机器人，把护理时间还给医护。

CareOps 从非临床任务切入：样本取送、药房配送、耗材补给、PPE、小设备、夜间低人力覆盖和清洁流程 proof-of-work。机器人负责可验证的运营工作，不进入诊断和治疗边界。

Specimen

采血点、病区、检验科之间的 sealed sample 取送。

Pharmacy

药房到护士站的锁柜配送和授权签收。

Supply

耗材、PPE、小设备和中心库房补给任务。

Access

电梯、自动门、门禁、禁行区和夜间路线。

Audit

条码、温控、交接、异常和电子签收证据。

02 · SpecimenChain

从病区到检验科，一路可见。

CareOps 把 barcode / RFID、锁柜、电子签收、温控、交接人、取件点、目的地和异常事件串成 chain-of-custody。目标是支持追溯和院方验证，不声称保证临床检测结果。

Request

LIMS/HIS/FHIR-style 任务进入队列。

Scan

样本条码、工牌、锁柜和取件点绑定。

Transit

路线、电梯、门禁、温控和时间戳记录。

Handoff

目的地扫码、电子签收和状态回写。

Review

异常、超时、错码、开柜和人工接管复盘。

03 · Privacy First Edge

本地感知、最小化采集，让数据留在院方可控边界。

医院机器人不能默认把走廊视频和 PHI 送到云端。CareOps 用 Qualcomm edge 完成本地感知、避障、脱敏和任务执行，只上传任务状态、审计事件和经批准的训练片段。

  • BAA-ready / HIPAA-oriented controls：RBAC、SSO/MFA、审计日志、保留和删除策略。
  • 不默认把 PHI 或原始视频用于模型训练；训练数据需要明确边界和去标识化证据。
  • 边缘视觉识别人、障碍、区域和交接动作；云端负责分析、训练和版本管理。
  • 采购包包含数据流图、安全白皮书、SBOM、更新策略、消毒 SOP 和风险评估模板。

04 · Competition Demo

用 sealed mock specimen，展示商业闭环，不碰临床风险。

比赛 demo 使用模拟样本管和二维码：扫码放入锁柜、生成任务、跨区域移动、到达扫码签收、条码错误进入异常流程，再把取送 episode 导出为 LeRobot-compatible 数据。

Mock 只使用 sealed mock specimen 和合成标签，不处理真实临床材料。
Trace 条码、工牌、锁柜、温控、路线、交接和异常全程留痕。
Learn 非临床示教数据进入 LeRobot，用于取送和交接动作优化。
Bounded 不做诊断、给药、患者监护或医疗器械审批承诺。

05 · Qualcomm Value

医疗场景需要边缘智能，不是云端裸奔。

CareOps 让 Qualcomm 成为强合规场景里的可信边缘层：多摄像头、低功耗、本地推理、连接、隐私边界、断网降级和模型部署共同支撑医院、实验室、养老与服务机器人 OEM。

Edge 本地感知、避障、脱敏、任务缓存和断网降级。
Ops LIMS/HIS/药房/库存/电梯门禁进入运营工作流。
Trust 权限、审计、温控、签收、消毒 SOP 和采购证据包。
Train 非临床取送、避让、排队和交接数据进入训练闭环。

CareOps 的商业价值不是“机器人替代医护”，而是让医院后勤、样本、药房和耗材流转变成可追踪、可审计、可持续优化的运营系统。
