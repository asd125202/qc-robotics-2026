Pitch 45 · Hospital Logistics Control Plane

CareOps

护士、药师和检验人员每天被迫当院内快递员。CareOps 是院内智运控制面：用 Qualcomm 边缘节点把样本、药品、耗材、机器人、门梯、隐私和审计连成一个非临床物流闭环。

01 · Problem

医院最短缺的人力，被消耗在送标本、送药、取耗材和等电梯上。

医院不是缺一个会移动的机器人，而是缺一套能证明“谁下单、谁取件、谁交接、是否超时、是否越权、是否泄露数据”的院内物流系统。人工 runner 灵活，但不可追溯；机器人能跑，但没有工作流和合规证据就很难规模化采购。

人力

护士时间被非临床任务切碎

护理短缺让每一次跑药房、跑检验科、找耗材都变成真实成本，而不是小杂活。

样本

错误常发生在到达实验室之前

错码、延迟、容器问题、交接不清和运输异常会影响检验周转时间和责任追溯。

药品

安全配送需要锁柜和签收

药房到护士站的配送要有身份、权限、锁状态、时间戳和异常闭环。

设施

真正难的是门梯和路线

老院区、多楼宇、服务电梯、门禁、拥挤走廊和夜间路线决定机器人能不能稳定跑。

隐私

走廊视频也可能变成敏感数据

患者位置、工牌、屏幕、样本编号和异常视频不能默认送到云端训练。

02 · Current Alternatives Fail

现有方案各解决一块，但没有把“可采购的院内智运”拼成闭环。

CareOps 不做“比 Moxi 更会送东西”的机器人。它做医院物流控制面：让已有机器人、气动物流、人工 runner、LIS/LIMS、药房系统、门梯和审计要求形成同一个任务、证据和隐私边界。

人工 Runner

灵活但不可复制，交接、温控、路线、延迟、错误和责任边界常靠人工记录。

气动物流

适合小件高速传输，但覆盖不了大件、锁柜、多点配送、机器人路线和异常复盘。

单一机器人厂商

Moxi、TUG、Relay、Pudu、Keenon、MiR 都能跑任务，但客户还要一层跨系统 workflow。

实验室自动化

提升 lab 内部效率，但病区到检验科的 last mile、身份签收和电梯门禁仍在外面。

通用 RobOps

InOrbit、Formant 擅长 fleet ops，但不天然包含样本链路、BAA-ready、消毒 SOP 和医院招采包。

03 · Solution

CareOps 是院内部署的非临床物流控制面。

它运行在 Qualcomm Dragonwing / RB / IQ edge appliance 上，连接机器人和医院系统，但不做诊断、不分诊、不给药、不替代医嘱。它把样本、药品、耗材和设备取送变成可调度、可审计、可脱敏、可复盘的运营流程。

Request

LIS/LIMS、HIS、药房、SPD、护士站或人工台生成非临床物流任务。

Verify

条码/RFID、工牌、锁柜、温控、目的地和权限进入 SpecimenChain。

Dispatch

按优先级、路线、门梯、电量、区域规则和机器人能力派发任务。

Protect

视频、患者位置、工牌和屏幕默认本地处理，导出前完成最小化和脱敏。

Replay

交接失败、错码、超时、堵路和人工接管变成审计包和 LeRobot 训练片段。

04 · Why Now

护理短缺、医院机器人成熟和数据合规压力正在同时逼近。

医院已经开始买机器人，但采购决策越来越看重 ROI、网络安全、隐私、消毒、系统集成和可验收证据。2026 年的机会不是做一个新底盘，而是把机器人变成医院可以采购、可以审计、可以持续优化的运营系统。

56% AHA 报告显示人工和薪酬约占美国医院成本的 56%。
17.6% NSI 2025 报告中 RN turnover 为 17.6%，每名 bedside RN turnover 成本约 $60k。
15% McKinsey / ANA 估计技术和任务重分配可释放最高约 15% 护士时间。
77 中国 2025 年前 8 个月已有多起医院物流机器人招采项目，三甲是核心买家。
Edge 医院走廊视频、PHI、工牌和样本事件需要本地处理，不适合默认上云。

05 · Product

先做 sealed specimen transport，再扩展到药房、耗材和设备补给。

第一阶段的 wedge 是病区/急诊/手术室到检验科的 sealed mock specimen workflow：扫码、锁柜、路线、电梯、交接、异常和回写。它足够高频、足够可衡量，也能避开诊断和治疗边界。

  • CareOps Edge：本地任务引擎、机器人 adapter、隐私视觉、事件账本和 sync agent。
  • SpecimenChain：条码/RFID、锁柜、工牌/PIN、温控、时间戳、签收和 hash-linked custody events。
  • AccessBridge：按医院授权接口请求电梯、自动门、门禁和禁区通行，并记录 ack。
  • PrivacyVision：本地人脸、工牌、腕带、屏幕和非任务标签脱敏，原始视频短期留院内。
  • Incident Replay：MCAP/JSON 复盘包，包含任务状态、机器人轨迹、交接、设施事件和模型版本。

06 · Product API

CareOps 交付的是院内物流事件账本，而不是又一个机器人 app。

API 模型对齐 FHIR / HL7 v2 / vendor API / GS1 EPCIS 思路：任务、样本、地点、交接、访问请求和审计事件都有结构化记录。安全关键和临床判断仍由医院流程和合规系统负责。

CareOpsTask

kind、priority、source refs、objects、pickup、dropoff、custody_required、due_by。

CustodyEvent

object_id、actor hash、read point、biz location、temp、lock state、signature。

AccessLease

elevator、door、restricted zone、valid_from、valid_to、request ack、fallback。

PrivacyExport

redaction status、retention class、approved clip、dataset lineage、PHI boundary。

IncidentReplay

barcode mismatch、elevator timeout、path blocked、handoff failed、temp excursion。

LeRobotEpisode

redacted frames、robot pose/action、human intervention、outcome label、no raw PHI。

07 · Market & Business Model

卖给医院的不是机器人，是更少跑腿、更快交接和更完整的招采证据。

第一批买家不是“所有医院”，而是已有院内物流痛点的三类组织：多楼宇三甲/大型医院、检验量高的实验室/医学中心、以及有 SPD/药房/机器人伙伴的系统集成商。

海外版

BAA-ready Edge + RaaS

90 天 pilot：2-3 台机器人 + CareOps edge + LIMS/pharmacy mock integration，$30k-$75k。站点集成：$25k-$100k。运营费：$3k-$8k / secure delivery robot / month。核心 KPI：护士时间、药房配送时长、specimen TAT、异常关闭时间。

中国版

数据不出院 + 招采包

三甲/大型院区采用试点到招采：轻量样本机器人人民币 198k-298k，安全药品/温控机器人人民币 350k-500k，试点包人民币 0.6M-1.5M，年维保 8%-12%。强调院内私有化、等保、SPD/本地机器人伙伴和国产云训练通道。

08 · Competition & Moat

Moxi、TUG、Relay、Pudu 和 MiR 负责跑；CareOps 负责证明这件事可采购、可审计、可改进。

竞争很多，所以定位必须清楚：CareOps 不靠底盘赢，而靠医院工作流、证据链、隐私边界、门梯设施、LIS/LIMS/药房/SPD 集成和 Qualcomm edge profile 复利。

Workflow Memory

每条病区到检验科、药房到护士站、SPD 到科室的路线、权限和异常都会变成站点资产。

Evidence Packet

BAA-ready、PIPL/等保、消毒 SOP、SBOM、patch SLA、数据流图和验收 KPI 直接服务招采。

Hospital Adapters

FHIR / HL7 v2、LIS/LIMS、药房、SPD、BMS、电梯门禁和 robot adapter 逐步复用。

Privacy Data Flywheel

只把批准的脱敏事件、模拟数据和非临床片段进入训练，形成中国和海外两条数据边界。

09 · Why Qualcomm

Qualcomm 的机会，是成为医院机器人可被信任的边缘底座。

医疗运营场景天然需要本地推理、低功耗、多摄像头、稳定连接、断网降级、隐私边界和模型部署证据。CareOps 把 Dragonwing / RB / IQ 从单机 demo 变成医院 edge workflow reference。

Local AI

本地运行 barcode/OCR、脸部/工牌/屏幕脱敏、人群和障碍识别、交接异常检测。

Connectivity

Wi-Fi / private 5G、VLAN、mTLS、设备身份和医院网络分段进入采购证据。

AI Hub

PyTorch/ONNX 到 Qualcomm runtime 的 compile、quantize、profile 变成上线门禁。

Offline

断云时仍能派单、记录、脱敏、签收和审计；云端只做训练、分析和包更新。

Ecosystem

机器人 OEM、医院 SI、SPD、LIS/LIMS、药房系统和电梯门禁围绕 Qualcomm edge 连接。

10 · Demo & Ask

八分钟演示：一个 STAT mock specimen，从护士站到检验科，途中电梯超时。

比赛 demo 使用 sealed mock tube、二维码、工牌 mock、锁柜和模拟电梯。重点不是机器人能走，而是任务、身份、隐私、异常、回写和 LeRobot 数据闭环都能被看见。

1. Scan

护士站扫码 mock specimen + 工牌，锁柜关闭，生成 CareOpsTask 和第一条 CustodyEvent。

2. Dispatch

SiteGraph 预约走廊、电梯和检验科门；机器人 adapter 接收任务，本地画面显示脱敏 overlay。

3. Incident

注入 elevator timeout 或 barcode mismatch，系统暂停、告警、生成 replay timeline。

4. Export

人工接管完成交接，FHIR-style Task 变为 completed，导出 redacted LeRobot episode。

我们的申请：用 Qualcomm Dragonwing / RB / IQ edge hardware 做一套 hospital logistics reference workflow，让评委看到机器人、隐私、合规、招采和训练闭环如何在同一个现场产品里成立。

CareOps 不声称 HIPAA/PIPL compliant robot，不做诊断、分诊、给药、患者监护或临床决策。它卖的是非临床院内物流控制面：让样本、药品、耗材、电梯门禁、隐私和审计形成闭环。
