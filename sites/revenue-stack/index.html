Pitch 17 · Robot Revenue Operating System

RevenueStack

把机器人从一次性硬件销售，变成可定价、可授权、可计量、可续费、可融资的劳动力账户。RevenueStack 补上 Qualcomm prototype-to-production 之后的 production-to-revenue 层。

看核心问题 看 7 分钟演示

01 · Problem

机器人 demo 越来越容易，机器人公司赚钱越来越难。

买方不想为“机器”本身付高价，只愿为产能、节省人工、稳定 uptime 和责任边界付费。可是大多数机器人公司仍然把硬件、软件技能、训练积分、运维、质保、渠道和数据服务拆成一堆表格。

Demo Trap

试点赢掌声，规模亏毛利

从 1 台 demo 到 10-500 台部署时，报价、租赁、RaaS、技能包、SLA、渠道和收入确认同时爆炸。

Robot Work

工作量没有进入账单

自主小时、任务数、配送、巡检点、抓取次数、远程接管和训练 GPU minutes 还停在运维日志里。

Finance

CFO 看不到真收入

ARR、usage revenue、deferred revenue、warranty liability、partner payout 和 renewal expansion 分散在不同系统。

China

中国交付更复杂

项目采购、经销商/SI、e-fapiao、押金短租、现场运维和本地数据边界必须一起设计。

Global

海外也不是纯 SaaS

RaaS、ERP/e-invoice、Data Act/GDPR、保险、融资材料和多币种结算同样要求机器人语义。

02 · Current Alternatives Fail

现有工具各管一段，但没有谁理解“机器人收入对象”。

机器人商业化不是普通 SaaS seat 计费。它同时包含 BOM、序列号、技能授权、QNN profile、现场验收、替换机、保修、SLA、渠道佣金、租赁资产和任务证明。

ERP / NetSuite / Oracle

管 BOM、订单和会计，但不理解 robot serial、技能授权、现场 SLA、替换机和任务证明。

Stripe / Metronome

能做 usage billing 和 credit wallet，但默认对象是 SaaS feature，不是传感器、BOM revision 和保修责任。

Fleet Dashboard

能看在线率、任务状态和故障，却不会自动生成 quote、invoice、dealer payout、SLA credit 和续费预测。

Hardfin / HaaS Finance

证明 RaaS finance ops 是真痛点，但缺少技能商店、训练积分、Qualcomm edge evidence 和多方分账。

Robot App Stores

UR+、MiR Go、Viam、Unitree、OpenMind 证明生态价值，但 OEM 仍需要授权、兼容性和结算账本。

Spreadsheet

1-5 台试点能撑住；多站点、多 SKU、多技能、多渠道后，漏收费和错承诺会变成毛利黑洞。

03 · Solution

RevenueStack 是从 robot event 到 recurring revenue 的 system of record。

它把“机器人干了什么”变成“客户该付什么、伙伴该分什么、公司该确认什么收入、哪条证据支持这笔收入”。

Product Ledger

hardware SKU、BOM revision、robot serial、payload、firmware、skill、credit、SLA、warranty、price book。

Entitlement Ledger

customer、site、fleet、robot、skill、quota、effective dates 和 install policy 决定“现在能用什么”。

Usage Ledger

autonomy hours、tasks、picks、deliveries、inspection points、teleop、inference、training、data、SLA incidents。

Revenue Ledger

contract、fapiao/invoice、usage、prepaid credits、warranty obligations、partner payout 和 revenue schedule。

Expansion Ledger

从 starter kit 到 paid pilot，再到 fleet subscription、skill marketplace、RaaS 和行业包。

04 · Why Now

RaaS、机器人技能商店和 edge AI 量产路径在 2025-2026 同时成熟。

这不是提前十年的平台幻想。RaaS fleet、per-robot/month 软件、机器人 app store、AI Hub/QNN profile 和 China/global 双市场商业化都已经出现，只是还没有一个统一的机器人收入层。

200K IFR 2025 报告：2024 年专业服务机器人销量接近 20 万台。
31% IFR：专业服务机器人 RaaS fleet 2024 年增长 31%。
295K 中国 2024 年新增工业机器人安装约 29.5 万台，占全球 54%。
160+ MiR Go 生态已有 160+ off-the-shelf applications / top modules。
$5+ Viam 等平台已把机器人存储、计算、上传、通知和能力做成 usage-based pricing。
$8B Qualcomm 将 industrial / networking / robotics 列为 FY2029 非手机增长重点之一。

05 · Product

从 paid pilot 到规模部署，把每一台机器人变成可扩张账户。

第一批 ICP：正在走向 10-500 台部署，并通过经销商、SI、RaaS、租赁或海外渠道销售机器人的 OEM。产品不是财务后台，而是 quote-to-cash + usage-to-bill + profile-to-marketplace 的机器人业务控制面。

  • Robotics CPQ：硬件、BOM、payload、安装、训练积分、技能包、SLA、租赁、押金、买断、渠道折扣。
  • Robot Account：customer、site、fleet、serial、owner、operator、lease status、subscription、warranty、data lane。
  • RaaS / Lease Billing：按月、按小时、按任务、按配送、按巡检点、按亩、按 tote、按节省金额、按事件短租。
  • Skill Store：技能包、兼容 BOM、Qualcomm AI Hub/QNN profile、license、trial、canary、usage meter、开发者分成。
  • SLA / Warranty Ops：保修、延保、备件、远程支持、现场派单、替换机、RMA、service credit。
  • Partner & Finance：经销商/SI、e-fapiao、EU e-invoice、US ERP adapters、ARR、usage revenue、warranty liability。

06 · Product API/Evidence

它不是“财务概念”，而是一组能演示的机器人商业 API。

每个商业动作都被结构化：报价、激活、授权、部署、edge profile、计量、训练、保修、渠道分成、发票和收入确认。

RobotAccount

`serial`、`customer`、`site`、`fleet`、`lease_status`、`warranty`、`data_region`。

EdgeProfile

`device`、`model`、`QAIRT/QNN version`、`latency`、`memory`、`power`、`accuracy`、`support_boundary`。

UsageMeter

`autonomy_hours`、`tasks`、`picks`、`deliveries`、`inspection_points`、`teleop_minutes`、`training_credits`。

BillRun

订阅、用量超额、训练积分消耗、App Store 分成、dealer commission、fapiao/invoice、rev-rec schedule。

Skill Listing Gate

没有 robot capability、BOM compatibility、ODD、安全门禁和 Qualcomm edge profile，就不能上架。

Finance Evidence

每台机器人产生 payback、utilization、SLA risk、warranty reserve、partner payout 和 renewal forecast。

核心闭环：`POST /robots/{serial}/activate` -> `POST /entitlements/grant` -> `POST /edge-profiles` -> `POST /meter-events` -> `POST /bill-runs` -> `POST /revenue/fulfillment-events`。

07 · Market & Business Model

卖给 robot OEM 的不是财务软件，而是从试点到规模部署的收入基础设施。

RevenueStack 的收入和客户的 fleet growth 绑定：active robot、meter volume、GMV、渠道分成、skill marketplace、training credits 和高级质保/财务模块。

中国版

fapiao-native、partner-native、project-procurement-native

Starter：RMB 3k-20k / month，10-100 台机器人，基础 CPQ、订阅、用量、e-fapiao、partner portal。Growth：RMB 100k-500k / year，渠道、RaaS、质保、私有化、本地云/模型计费、招投标和验收材料。支持活动短租、押金、现场操作员、维修派单和本地数据优先。

海外版

RaaS-native、ERP/e-invoice-native、finance-ready

Startup：$999-$5k / month + active robot fee。Scale：$25k-$150k / year + usage metering volume。Enterprise：$150k-$500k / year，audit、ASC 606/842 handoff、dealer/channel、private marketplace、warranty automation。GMV take rate 0.5%-2%；skill / channel rev share 10%-25%。

08 · Competition & Moat

竞争不是“有没有计费工具”，而是谁能成为机器人收入事实表。

Stripe、ERP、Hardfin、fleet platforms、UR+/MiR/Viam/Intrinsic/Unitree/OpenMind 都能覆盖一部分。RevenueStack 的壁垒是把机器人语义、任务证据和 Qualcomm edge profile 变成商业记录。

Robot Revenue Schema

BOM、serial、skill、training credit、warranty、lease、usage、partner payout 在同一模型。

Telemetry-to-Invoice

真实 robot task 直接生成 overage、SLA credit、renewal expansion 和 gross-margin evidence。

Profile-to-Marketplace

AI Hub/QNN/QAIRT profile 成为 skill listing、采购、分账和支持责任的商业门槛。

Vertical Templates

仓储、餐饮、医疗、安防、农业、教育、人形的 RaaS / lease / task pack / SLA 模板沉淀。

Partner Network Effects

技能开发者、SI、dealer、leasing partner、insurer 围绕 entitlement 和 payout 增长。

System Of Record

一旦成为 quote-to-cash、usage-to-bill、renewal 和 finance diligence 的记录源，迁移成本高。

09 · Why Qualcomm

Qualcomm 的机会不是只卖开发板，而是占据开发板周围的收入层。

Dragonwing、RB3、IQ 系列、AI Hub、QNN/QAIRT 和 Partner Network 已经在铺 prototype-to-production 路径。RevenueStack 把 production profile 转成可售 SKU、技能 listing、渠道分账、SLA 和续费。

Starter Kit

Dragonwing / RB3 starter kit 不只是硬件销售，而是进入可续费技能、训练和 fleet ops 的入口。

Training Credit

LeRobot 数据、云训练、仿真和模型评估变成 credit burn-down 与 edge export 证据。

AI Hub / QNN

每个模型都有 device、runtime、latency、memory、power、accuracy 和 support boundary。

Marketplace

No Qualcomm edge evidence, no marketplace-ready skill：profile 变成采购和分账门槛。

Production SKU

Prototype on RB3，productize on IQ / Dragonwing，把 demo 迁移到量产硬件和持续收入。

NVIDIA sells the robot brain; RevenueStack helps Qualcomm sell the efficient edge robot business system.

10 · Demo & Ask

7 分钟演示：机器人跑一次任务，收入系统自动闭环。

演示目标不是展示发票 UI，而是证明每一次机器人商业动作都能被定价、授权、计量、结算和确认收入。

0-2m Dealer 配置 20 台巡检机器人报价：硬件、Inspection Pro、10k 训练积分、3 年 SLA、渠道折扣。
2-3m 扫描序列号激活 fleet；绑定 BOM、firmware、warranty、site、subscription、data region。
3-5m 校验 AI Hub/QNN profile 后安装技能；记录 3 小时 uptime、182 tasks、12 skill calls、40 training-credit minutes。
5-7m Bill run 输出订阅、用量、训练积分、App Store 分成、dealer commission、fapiao/invoice、递延收入。

Ask：6-8 周 Dragonwing Revenue Attach Sprint；RB3 / QCS6490 demo target；IQ / Dragonwing production-forward profile；AI Hub + QNN quota for 2-3 reference skills；共同定义 Dragonwing Skill Profile v0.1；引入 module partners、SI、教育渠道和 RaaS/leasing partners。

如果没有收入结构，机器人平台只是 demo；有了 RevenueStack，它才像一个可以销售、续费、分账、融资和扩张的产品公司。
