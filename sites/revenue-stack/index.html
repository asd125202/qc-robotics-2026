Pitch 17 · Robot Revenue Operating System

RevenueStack

机器人不是一次性卖货，而是持续产生现金流的劳动力。RevenueStack 把硬件、租赁、技能、云训练、SLA、渠道和用量计费变成一个收入操作系统，让机器人公司从 demo 走向可续费生意。

看收入漏洞 看计费演示

01 · Problem

机器人越来越便宜，但机器人公司还在用卖硬件的方式赚钱。

一个 robot OEM 可以做出很好的 demo，却在商业化时卡住：合同只写硬件价格，试点没有升级路径，软件技能无法收费，远程运维没有订阅，售后和质保吞掉毛利，渠道佣金和租赁账本全靠表格。

Lumpy

收入一次性

硬件交付当天看起来有收入，但客户成功、技能升级、训练数据、运维和续费没有被产品化。

RaaS

租售混乱

买断、租赁、试用、押金、保内/保外、买断尾款、资产归属和替换机没有统一账本。

Usage

机器人工作无法计量

自主运行小时、任务数、配送单、巡检点、抓取次数、远程接管和训练积分没有变成可计费事件。

Channel

渠道越多越难结算

经销商、SI、租赁公司、现场运维、技能开发者和保险服务商都需要佣金、分成和责任边界。

Finance

财务看不到真实 ARR

硬件、订阅、用量、服务质保、预付训练积分、App Store 分成和递延收入分散在不同表里。

02 · Current Alternatives Fail

ERP、Stripe 和普通 SaaS 计费工具不理解机器人。

机器人商业化同时包含硬件 SKU、BOM、序列号、租赁资产、现场部署、运维 SLA、技能授权、云训练积分、用量证明、保修责任和渠道分账。把它们拼在 CRM、ERP、Stripe、飞书表格和工单系统里，早晚会漏钱。

ERP / BOM

会管物料和订单，但不理解技能订阅、用量计费、训练积分、机器人任务证明和 App Store 分成。

Stripe / SaaS Billing

适合 seat 和 API 计费；机器人还需要序列号、BOM 修订、替换机、保修、SLA 和部署里程碑。

Fleet Dashboard

能看设备在线和任务状态，但通常不会生成 quote、invoice、revenue recognition、dealer payout 和续费预测。

Marketplace

UR+、MiR Go、Viam Registry 证明生态有价值，但 robot OEM 仍需要自己的授权、结算和兼容性账本。

Leasing Tools

能处理设备租赁，却不理解每台机器人今天跑了多少任务、用了哪个技能、触发了哪种 SLA。

Spreadsheet

试点阶段够用；一旦进入 10-500 台部署，报价、续费、渠道、保修和用量都会失控。

03 · Solution

RevenueStack 是机器人公司的收入操作系统。

它把 robot、component、skill、fleet workflow、training run、support event、dealer motion 和 app install 变成可定价、可授权、可计量、可开票、可分账、可确认收入的对象。

Product

SKU、BOM、序列号、payload、firmware、skill package、training credit、service plan 和 price book。

Entitlement

客户、站点、fleet、robot serial、skill、meter、quota 和有效期决定“现在能用什么”。

Usage

自主小时、任务数、配送、抓取、巡检点、teleop、inference、GPU-minute、数据 GB 和 SLA 事件。

Revenue

合同、发票、用量、质保、预付积分、渠道分成和交付事件转成收入排期。

Expansion

从 starter kit 到 pilot，再到 fleet subscription、skill marketplace、RaaS 和行业包。

04 · Why Now

硬件价格下探、RaaS 扩张和技能生态正在创造机器人 App Store 时刻。

IFR 显示 2024 年专业服务机器人销量接近 20 万台，RaaS fleet 增长 31% 至超过 24,500 台。中国工业机器人仍是全球最大市场，服务机器人、AMR、餐饮、清洁、医疗、安防、农业和人形机器人都在寻找“硬件之外”的收入结构。

295K 中国 2024 年新增工业机器人安装约 29.5 万台，占全球安装 54%。
200K IFR 报告 2024 年专业服务机器人销量接近 20 万台。
31% 专业服务机器人 RaaS fleet 2024 年增长 31%，超过 24,500 台。
$4.9K 低价人形机器人和开发套件把硬件入口拉低，收入重心转向软件和服务。
500+ UR+ 的生态规模证明：机器人买家愿意为可验证组件、软件和集成包降低部署风险。

05 · Product

从报价到续费，把每一台机器人变成可扩张账户。

RevenueStack 的第一批 ICP 是正在从 pilot 走向 10-500 台部署的机器人公司，尤其是通过经销商、SI、RaaS、租赁和海外渠道销售的 OEM。

  • Robotics CPQ：硬件、BOM、payload、安装、训练、技能、SLA、租赁、买断和渠道折扣。
  • RaaS Billing：按月、按小时、按任务、按配送、按巡检点、按亩、按 tote、按节省金额计费。
  • Asset & Lease Ledger：序列号、部署状态、租期、折旧、押金、替换机、退货、买断和保险。
  • Skill Store：技能包、QNN profile、兼容 BOM、license、trial、canary、usage meter、开发者分成。
  • SLA / Warranty：保修、延保、预防性维护、备件、远程支持、现场派单、service credit 和收入确认。
  • Finance Dashboard：ARR、usage revenue、deferred revenue、gross margin、warranty liability、partner payout 和 expansion。

06 · Product API

机器人收入系统的基本对象不是 seat，而是 SKU、序列号、技能、任务和 SLA。

每个商业动作都被结构化：报价、激活、授权、部署、计量、训练、保修、渠道分成和收入确认。这样机器人公司才能从“卖设备”升级为“经营机器人账户”。

SKU / BOM

hardware、component、skill、cloud_credit、fleet_plan、warranty、SLA、service、marketplace_app 和 BOM revision。

Entitlement

customer、site、fleet、robot_serial、package、quota、effective_dates 和 install policy。

UsageMeter

autonomy_hours、tasks、picks、deliveries、inspection_points、teleop_minutes、training_credits、data_gb。

Subscription

fleet ops plan、included robots、included usage、support tier、add-ons、overage price 和 renewal date。

SLAWarranty

covered components、response time、uptime commitment、RMA、service credit、assurance vs service warranty。

PartnerPayout

dealer、SI、publisher、territory、deal registration、rev share、commission、fapiao/tax 和 settlement。

07 · Market & Business Model

卖给 robot OEM 的不是财务软件，而是从试点到规模部署的收入基础设施。

RevenueStack 自己的收入也要跟客户增长绑定：active robot、用量计费量、GMV、渠道分成、skill marketplace 和高级财务/质保模块。

中国版

租售一体 + 渠道友好 + 发票/补贴材料

Starter：人民币 3k-20k / month，用于 10-100 台机器人和基础 CPQ/订阅/用量。Growth：人民币 100k-500k / year，加入渠道、RaaS、质保、私有化和国产云/模型计费。GMV take rate：0.5%-2%，技能商店/渠道分成 10%-25%。支持 fapiao-ready billing、设备更新材料和本地 SI。

海外版

Revenue OS For Commercial Robotics

Startup：$999-$5k / month + active robot fee。Scale：$25k-$150k / year + usage metering volume。Enterprise：$150k-$500k / year for SOC2-style audit, ASC 606/842 handoff, dealer/channel management, private marketplace and warranty automation。Best ICP: robotics companies moving from pilots to 10-500 deployed robots.

08 · Competition & Moat

真正的壁垒不是计费规则，而是机器人收入事实表。

Hardfin、Stripe、Metronome、ERP、CRM、marketplace 和 fleet platforms 都能覆盖一部分。RevenueStack 的护城河是把它们缺少的 robot context 变成系统记录：BOM、序列号、技能、任务、SLA、渠道、训练积分和 Qualcomm edge profile。

Telemetry + Billing

机器人真实任务和 fleet 状态直接生成 invoice、overage、SLA credit 和 renewal expansion。

Contract Templates

仓储、餐饮、医疗、安防、农业、人形和教育的 RaaS / lease / task pack 合同模板沉淀。

Marketplace Network

技能开发者、SI、经销商、租赁商和保险服务商围绕同一套 entitlement 和 payout 增长。

Finance Graph

硬件交付、订阅履约、用量消耗、预付积分、质保义务和渠道分成形成融资需要的数据。

Qualcomm Profile

技能是否可售，取决于目标硬件、AI Hub/QNN profile、runtime constraints 和支持责任。

System Of Record

一旦 RevenueStack 成为 quote-to-cash、usage-to-bill 和 renewal 的记录源，迁移成本很高。

10 · Demo & Ask

7 分钟演示：机器人跑一次任务，收入系统自动闭环。

演示的目标不是展示发票 UI，而是证明每一次机器人商业动作都能被定价、授权、计量、结算和确认收入。

0-2m Dealer 配置 20 台机器人报价：硬件、Inspection Pro 技能、10k 训练积分、3 年 SLA、渠道折扣。
2-3m 扫描序列号激活 fleet；每台机器人绑定 BOM、firmware、warranty、site、subscription 和 entitlement。
3-5m 安装付费技能并跑任务；系统记录 3 小时 uptime、182 tasks、12 skill calls、40 training-credit minutes。
5-7m Bill run 输出订阅、用量、训练积分消耗、App Store 分成、dealer commission、SLA 状态和递延收入。

对 Qualcomm 的请求：给我们 6-8 周 Dragonwing Revenue Attach Sprint：RB3/IQ10 demo target、AI Hub/QNN quota、2-3 个 reference skills、OEM/SI 介绍，以及谨慎使用 “Qualcomm-edge-ready candidate” 这类 profile 语言的许可。

如果没有收入结构，机器人平台只是 demo；有了 RevenueStack，它才像一个可以销售、续费、分账、融资和扩张的产品公司。
