Pitch 51 · Warehouse Exception Closure

把仓库异常从“群里喊人”变成自动闭环。

WarehouseLoop 面向电商仓、3PL、多客户履约仓、跨境海外仓和制造备件仓，把错拣、缺货、卡单、滞留、破损、设备卡停和退货待判变成可派单、可追踪、可验证、可回写、可复盘的异常闭环。

看异常闭环 看商业模型

01 · Problem

仓库最贵的不是作业，而是异常没人闭环。

仓库主管每天被异常打断：WMS 只记录状态，群聊只传递情绪，现场人员靠经验补救。结果是订单延迟、重复盘点、跨班扯皮、客户投诉、加班成本和管理层救火。

库存不可信

WMS 说 SKU 在 A-03，现场却在 B-02；拣货员、AMR、客服和客户都被错误库存拖慢。

异常无人认领

错拣、缺货、卡单、补货断点和设备卡停在多个系统之间漂移，没有 owner、SLA 和升级链。

退货拖毛利

破损、错货、疑似欺诈、可售/不可售判定缺标准证据，返架慢，索赔和折价空间流失。

自动化断层

WMS 记录流程，机器人执行动作，相机看到现场，但异常没有统一任务语言和证据标准。

02 · Why Now

仓库正在从“人海调度”进入“利润率保卫战”。

订单碎片化、即时履约、退货增长、用工波动、物流降本和机器人普及，让异常频率和处置成本同时上升。企业不再只买自动化设备，而是需要一套能把异常压下去的运营系统。

$849.9B NRF 预计 2025 年美国零售退货规模；线上退货率 19.3%。
13.9% 2025 年中国社会物流总费用/GDP 首次低于 14%，政策目标到 2027 年约 13.5%。
1989.5 亿件 2025 年中国快递业务量，仓内波峰、临时工和错件压力持续存在。
2.75 万亿 2025 年中国跨境电商进出口额，海外仓退货、换标、质检和库存纠偏成为刚需。

03 · Core Insight

异常不是单点事件，而是一条未完成的业务链。

真正的问题不在“有没有报警”，而在报警之后谁负责、多久响应、是否升级、证据是否足够、有没有根因、下次是否避免。闭环系统比单个机器人或看板更接近仓库经理的预算语言。

责任

每个异常必须有 owner、SLA、影响订单/客户、升级规则和目标状态。

证据

扫码、照片、视频、RFID、重量变化、机器人状态和人工确认一起证明“已经纠正”。

复盘

失败、低置信度、人工接管、超时和复发进入异常库，变成下一版流程和 edge skill。

04 · Solution

WarehouseLoop 是 existing warehouse 之上的异常闭环操作层。

它不替代 WMS/WES/AMR，也不从“全自动无人仓”开始。它连接 WMS、TMS、PDA、摄像头、RFID、称重、AMR/叉车/机械臂和人工复核，把每个异常从现场发现推进到派单、纠偏、验证、回写和复盘。

  • Capture：订单、库存、路径、设备、人员状态、固定相机、扫码、RFID、称重和 WMS 对账发现异常。
  • Classify：判断影响订单、客户、时效、SLA、退货价值、dock dwell 和 AMR 任务风险。
  • Dispatch：异常变成有负责人、SLA、优先级、目标状态和证据要求的任务。
  • Close：扫码、照片、视频、RFID、重量变化和机器人状态组成证据包，回写 WMS/WES/TMS。
  • Learn：失败、低置信度、人工接管和复发导出为 LeRobot episode。

05 · Product Workflow

发现、判断、派单、追踪、复盘，一条链跑完。

第一版不做全仓替换，只做一个高频异常流程：库存差异、收货/月台异常、退货质检、putwall 复核或 AMR 卡停。客户要看到异常响应时间、关闭率、回写延迟、证据完整率和主管追踪工时。

发现

Edge camera、PDA、RFID、WMS 对账或 AMR 日志发现错位、漏扫、破损、卡停或退货待判。

判断

系统评估影响订单、客户、时效、库存准确率、退货价值和机器人任务。

派单

分配到仓员、班组、主管、AMR、机械臂或远程协助，并设置 SLA 与升级链。

验证

扫码、拍照、视频、RFID、重量变化或机器人状态验证完成。

复盘

WMS/WES/TMS 回写已修正；根因、复发、接管和失败片段进入训练队列。

06 · Market Wedge

先切入“异常密度最高”的仓库现场。

最好的首单不是全仓智能化，而是 60-90 天把一个高频异常流程闭环：SKU 多、退货高、10+ 月台、临时工多、有 WMS 但自动化碎片化的仓库。

中大型 3PL

多客户、多 SLA、多系统、多临时工，需要中立的异常闭环层。

DTC 品牌仓

服饰、美妆、消费电子 SKU 多、退货多，返架周期直接影响收入和折价。

跨境海外仓

退货分级、换标、破损、疑似欺诈、单证异常和库存纠偏是高价值场景。

物流园 / 分拨

dock dwell、ASN 不符、格口爆仓、包材/面单异常和设备卡停需要分钟级升级。

制造仓储

MES order、kitting、QR/RFID 验证、线边配送失败和备件错发可以复用。

自动化集成商

AMR/ACR/WMS/WES 项目里，异常闭环是可复制的插件和售后增购项。

07 · Business Model

按仓库规模和异常闭环量收费，而不是按硬件 BOM 收费。

收入模型是试点费 + 仓年费 + 模块费 + 系统集成费 + 按订单行/闭环异常的用量费。商业语言聚焦减少异常处理成本、提升履约稳定性和缩短退货返架周期。

60-90 天试点

中国 ¥8万-30万/仓；海外 US$25k-100k/仓，先跑 baseline 再验证改善。

仓年费

中国 ¥20万-80万/仓/年，大仓多模块 ¥80万-200万；海外 US$60k-250k/仓/年。

用量 / 模块

按订单行、closed exception、dock door、视觉点位、AMR 连接和退货模块收费。

集成 / 成果

WMS/WES 集成、码头视觉和客户验证收益可加 10%-20% success fee。

08 · Go-To-Market

从一个痛点班组，扩到整个仓网。

先卖给仓储运营负责人，不从 CIO 长周期切入。以 4-8 周试点证明：异常响应时间下降、超时单减少、WMS 回写更快、主管人工追踪减少。成功后扩到同仓多流程、多仓复制、集团级看板。

Design Partners

2-3 家 3PL、跨境退货中心、服饰/美妆品牌仓或制造备件仓。

First Workflow

退货质检、收货/月台差异、错库位纠偏、putwall 复核或 AMR 卡停。

Trial Metrics

异常 dwell、错拣、dock-to-stock、退货 receive-to-disposition、AMR intervention 和证据完整率。

Channel

WMS/WES/RMS、MHE 集成商、AMR/ACR/机械臂厂商、叉车/货架服务商和 Qualcomm edge 生态。

09 · Competition

我们不是 WMS，也不是机器人调度系统。

WMS 记录流程，BI 展示结果，机器人系统调度设备，群聊处理临时问题。WarehouseLoop 位于它们之间，专注异常处理闭环，把状态、责任、动作、证据、回写和复盘串起来。

Locus / Geek+

AMR pick assist 和货到人强；WarehouseLoop 管异常、proof、WMS 回写和 HIL 数据。

AutoStore / Symbotic

AS/RS 和 mega-DC 自动化强；WarehouseLoop 切 brownfield 3PL、退货和上下游异常。

Corvus / Dexory

库存扫描和数字孪生强；WarehouseLoop 进一步派单、纠偏、验证和回写。

Hai / Quicktron

中国 AMR/ACR 生态成熟；WarehouseLoop 做多系统异常和训练插件。

WMS / WES

system of record 强；WarehouseLoop 是 system of action, evidence and learning。

群聊 / 人工

短期灵活但无法沉淀 SLA、根因、证据、跨班责任和复发预防。

10 · Moat

闭环数据越多，异常处理越准。

护城河来自真实异常库、仓库流程图谱、客户现场规则、设备和系统连接器、跨仓复盘模型，以及每个客户现场持续沉淀的运营知识。

异常图谱

错拣、缺货、卡单、库存差异、dock dwell、AMR 卡停和退货争议的根因链。

证据标准

SKU、库位、tote/bin、条码、RFID、照片、重量、视频、模型版本和 WMS 状态。

HIL 数据

人工接管、低置信度、错误识别、AMR 卡停、复核驳回和恢复策略。

连接器

WMS/WES/ERP/RMS/RCS/TMS、AMR、RFID、扫码、称重和工位系统连接器。

SOP Benchmarks

客户 SOP、SLA、责任分派、审计证据、退货规则和 KPI benchmark。

Edge Profiles

Qualcomm edge model profiles、deployment recipes、fallback layer 和 rollback records。

11 · Architecture

轻集成、边缘感知、云端闭环。

边缘侧接入摄像头、PDA、RFID、称重、传感器、机器人和本地系统；云端做异常规则、AI 判断、任务编排、SLA 管理、证据包和复盘分析。架构重点是低侵入部署，不替换现有 WMS。

LoopCore

WMS/ERP/WES/RMS/RCS/TMS 连接器、ExceptionCase、WarehouseTask、SLA、审计和回滚。

LoopEdge

Qualcomm edge device 连接固定相机、扫码、RFID、称重、AMR、叉车、机械臂和本地安全门控。

LoopRMF

ROS 2、Nav2、MoveIt 2、ros2_control、Open-RMF 和混合厂商 fleet adapter；安全由本地控制器优先。

Learning Cloud

LeRobotDataset、HIL loop、中国/海外训练云、AI Hub / QNN / ONNX Runtime QNN profile 和灰度回滚。

12 · Competition Demo

3 分钟看见一个异常如何被消灭。

桌面 demo 用 2-3 层小货架、6-9 个库位、3 个 mini pallet/tote、SKU 盒子、QR/AprilTag、固定 edge camera、可选扫码枪/RFID/称重台和低速 rover。目标不是全自动仓，而是证明现场异常如何闭环。

Mismatch

Mock WMS 显示 SKU-A 应在 A-03，现场故意放在 B-02。

Edge Detect

Qualcomm edge camera 识别库位 AprilTag + SKU QR，生成红色异常卡。

Correct

系统派单给仓员、低速 AMR 或小机械臂，完成纠偏并扫码/拍照。

Writeback

WMS 从 exception_open 变成 closed_verified，证据包展开时间线、图片、扫码和模型版本。

Train

退货破损/条码不匹配低置信度触发人工复核，接管片段导出为 LeRobot episode。

13 · Why Qualcomm

Qualcomm 是仓库边缘智能的放大器。

WarehouseLoop 需要现场端低延迟感知、设备连接、边缘 AI、稳定网络和低功耗终端。客户的 SKU、订单、视频和退货数据不能都上传到公共云，安全控制也不能依赖公网。

Edge Vision

库位、条码、包装破损、缺件、多件、错放和动作失败在现场判断。

Gateway

条码、RFID、相机、AMR、机械臂、安全门控和 WMS mock 需要稳定本地 gateway。

Runtime

AI Hub、QNN、QAIRT、ONNX Runtime QNN EP 把模型变成可验证 edge artifact。

Hardware Path

RB3 Gen 2/QCS6490 做比赛原型，QCS8550/RB6/Dragonwing IQ10 做更高阶迁移路径。

LeRobot HIL

人工接管转成下一版技能，Qualcomm edge profile 让技能在现场可评估、可回滚。
