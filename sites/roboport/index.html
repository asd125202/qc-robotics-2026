RoboPort

让机器人像电脑一样扩展：传感器、夹爪、伺服、安全 IO 和相机不再从零适配，而是带着模组护照、ROS 2 接口、LeRobot 元数据和兼容证据进入 Qualcomm edge 生态。

01 · Integration Problem

每一次集成，不该从零开始。

线束、电源、驱动、ROS 节点、URDF、标定文件、LeRobot schema 和安全文档分散在供应商手册里。RoboPort 把这些隐性工作前置成一套可测试、可采购、可维护的机器人模组标准。

Power

电压、电流、热插拔、e-fuse、接地、屏蔽和热设计边界。

Driver

ROS 2 driver、controller、QoS、启动顺序和版本矩阵。

Geometry

URDF/xacro、frame、payload、collision、标定和工具中心点。

Data

LeRobot observation/action schema、timestamp、FPS、unit、dtype。

Safety

急停、STO、安全 IO、风险 delta、故障注入和证据包。

02 · Four Port Classes

不是一个万能插头，而是四类清晰边界。

机器人扩展不能把 USB-C 当成工业安全总线。RoboPort 把实时运动、智能工具、工业视觉和内部高速扩展分开定义，再用模块护照把它们接回统一的软件与训练工作流。

Motion / I/O

EtherCAT、CANopen、24V industrial I/O、STO、安全控制器桥接和周期要求。

Smart Tool

夹爪、吸盘、力控、IO-Link、RS-485/Modbus、工具供电和诊断。

Perception

MIPI CSI-2、GMSL/FPD-Link、GigE Vision、PoE、trigger、PTP timestamp。

Expansion

M.2、PCIe、USB、radio、storage、accelerator、capture card 和热边界。

03 · Module Passport

插上模块，系统自动识别；开始采集，数据天然可训。

每个模块都带签名 Manifest，声明硬件身份、驱动胶囊、URDF 坐标、ros2_control 接口、LeRobot features、功耗、固件 ABI、标定状态和安全边界。

Identify

serial、EEPROM、USB/CAN/Ethernet match、firmware hash。

Model

URDF/xacro、frame、payload、collision、MoveIt snippet。

Control

ros2_control interfaces、QoS、lifecycle、diagnostics。

Record

LeRobot action/state/image schema、timestamp、unit、FPS。

Gate

安全边界、功耗、固件版本和兼容测试通过后激活。

04 · CertLab + Marketplace

兼容不是口号，是可测试、可追溯、可审计。

RoboPort Certified 定位为兼容性与交付证据认证：它帮助企业进入系统级风险评估，但不替代 ISO、UL、CE 或功能安全认证。

  • 测试电气、热、延迟、timestamp drift、QoS、驱动启动、掉线恢复和 24-72h soak。
  • 输出 SBOM、签名驱动、固件回滚、兼容矩阵、功耗曲线和风险 delta。
  • 市场按认证等级、总线、功耗、ROS 2、LeRobot schema 和交付周期筛选。
  • 集成商可以采购 inspection kit、force-gripper kit、AMR safety kit 和 servo bridge。

05 · Qualcomm Ecosystem

从开发板到生态，缺的是可信模组港。

Qualcomm 已经有边缘 AI、连接、多媒体和机器人 reference design。RoboPort 把每个相机、夹爪、驱动器、安全 IO 和应用套件都变成 Qualcomm edge 的可认证入口。

Core Qualcomm edge 负责模块发现、推理、诊断、数据采集和部署门禁。
Driver ROS 2 driver capsule 让外设版本、QoS、资源和回滚可管理。
Data LeRobot metadata 保证新模块采集的数据可以训练和复现。
Market 认证费、市场分成、支持订阅和联合营销构成生态收入。

RoboPort 的目标不是发明更多接口，而是让每一个机器人外设都有清晰的兼容规则、软件说明书、训练数据边界和交付证据。
