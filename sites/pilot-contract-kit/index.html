Pitch 22 · Commercial OS For Paid Robot Pilots

PilotContractKit

企业不是不想试机器人，是不知道怎样低风险购买一次可验收试点。PilotContractKit 把 robot demo 变成采购、运营、安全、IT、法务和工程都能审查的 paid pilot，再把 paid pilot 变成 rollout contract。

看采购断点 看试点生成器

01 · Problem

机器人公司能做 demo，但企业不能采购“一个看起来不错的 demo”。

Demo 后真正卡住的是跨部门审批：采购要价格和退出，运营要 baseline 和 KPI，IT/OT 要数据和网络边界，安全要急停和监督，法务/保险要责任分配，工程要训练版本、edge profile 和可复现证据。

Procurement

担心被锁死

没有取消权、扩张价、CapEx/OpEx 选择、付款里程碑和供应商责任。

Operations

担心看不出 ROI

没有人工 baseline、throughput、intervention、uptime、punch list 和班次影响。

IT / OT

担心数据和远程访问

视频、遥测、云区域、保留期限、网络分段、MFA/RBAC 和日志没写清。

Safety

担心现场责任

急停、防护、人工监督、stop-work authority、rollback 和 operating envelope 混在一起。

Legal

担心谁为失败买单

数据权利、工伤/财损、保险、隐私、出口/制裁和责任限制没有结构化。

Engineering

担心不可复现

dataset、训练版本、runtime、latency、温度、功耗和回滚记录没有统一证据包。

02 · Current Alternatives Fail

RaaS、marketplace、OEM 生态、fleet tools 和集成商都重要，但没有中立的 pilot acceptance layer。

PilotContractKit 不替代机器人厂商、SI、Formant、Viam、Foxglove、InOrbit、Formic、Locus、UR+、FANUC、ABB 或 KUKA。它把这些工具产生的证据翻译成买方能签字的试点语言。

RaaS Vendors

让自家或合作 stack 更可买，但通常绑定特定机器人、场景或商业模式，不是跨厂商标准。

Marketplaces

帮助找供应商、报价和组件，但 discovery 不是 bankable pilot，也不定义 success arbitration。

OEM Ecosystems

降低某个 OEM orbit 内的集成风险，但不统一 KPI、付款、数据权利和扩张条款。

RobOps Tools

强在运行、遥测、调试和 fleet orchestration，但不是采购、融资、合同或保险入口。

System Integrators

能现场交付，但每个项目都重写 SOW、验收、风险边界和扩张报价，复用弱。

03 · Solution

PilotContractKit 是机器人付费试点的 neutral commercial operating system。

它把一次 4-8 周轻量试点或 60-90 天复杂现场试点，组织成可采购工作包：SOW、baseline、KPI、FAT/SAT、风险边界、Qualcomm edge evidence、支持责任和 scale decision memo。

Scope

site、zone、task、robot count、operating hours、exclusions、RACI、deliverables、timeline、price。

Baseline

人工流程、cycle time、labor hours、错误率、停机、SKU/order mix、site constraints。

Accept

throughput、quality、autonomy、uptime、integration、safety、edge evidence、ROI。

Boundaries

safety、operating envelope、data/privacy、cyber/OT、insurance、change control、stop conditions。

Scale

go/no-go、rollout price、RaaS/monthly option、support SLA、RevenueStack handoff。

04 · Why Now

机器人 adoption 已经不是科幻，采购却更理性。

工业机器人、专业服务机器人、RaaS、cobots 和中国具身智能供给都在增长。新的瓶颈不是“能不能动”，而是企业能不能用 KPI、责任、数据、安全、SLA 和 rollout path 批准一次 paid pilot。

542K IFR：2024 年全球新增工业机器人约 54.2 万台。
295K IFR：中国 2024 年新增约 29.5 万台工业机器人，约全球 54%。
31% IFR：2024 年专业服务机器人 RaaS fleet 增长约 31%。
$2.25B A3：2025 年北美机器人订单价值约 22.5 亿美元。
83% MHI/Deloitte：五年内供应链 robotics/automation 预期采用率达 83%。

05 · Product

不是模板文档，而是 pilot generator + evidence binder + scale router。

PilotContractKit 把试点状态分成 Draft、Baseline Ready、FAT Ready、SAT Ready、Risk Blocked、Pilot Passed、Scale Candidate、No-Scale。每个状态都有证据和责任人。

  • Pilot SOW Builder：scope、assumptions、exclusions、site readiness、interfaces、deliverables、RACI。
  • Baseline Data Pack：2-4 周代表性数据、measurement calendar、variance controls、manual process map。
  • KPI Scorecard：operational KPIs 与 robot KPIs 分开，定义 target bands、pass/fail、exceptions。
  • ROI / Payback Model：CapEx vs RaaS、labor、throughput、quality、maintenance、ramp-up、sensitivity。
  • FAT/SAT Protocol：requirements -> test scripts -> evidence -> punch list -> retest -> signoff。
  • Support/Safety/Data/Cyber Addenda：SLA、site risk prompts、training、data map、MFA/RBAC、OT segmentation。

06 · Product API

把技术证据转成付款、验收和扩张证据。

所有指标标注来源：`measured-on-site`、`measured-on-board`、`AI-Hub-device-cloud`、`vendor-reported`、`proxy` 或 `simulated`。试点可以有模拟项，但不能把模拟项当验收结论。

pilot-sow

customer、site、task、timeline、deliverables、assumptions、exclusions、price、RACI。

baseline-data-request

process map、task frequency、cycle time、labor hours、quality errors、downtime。

acceptance-scorecard

metric、baseline、target、measurement method、test window、owner、pass/fail。

risk-boundary

operating envelope、safety assumptions、supervision、change control、stop conditions。

qualcomm-edge-evidence

board manifest、AI Hub/QNN profile、runtime report、SafetyOps gate、rollback record。

scale-decision-memo

ROI、payback、risk residuals、rollout quote、RaaS option、next-stage contract path。

07 · Market & Business Model

先卖可控试点，再转 RaaS、订阅、运维和交易抽成。

买方不是“想看机器人”的人，而是要把试点批下来的人。中国版帮助机器人公司和 SI 把硬件速度转成可采购材料；海外版帮助自动化买方把 labor pressure 和预算转成可控 paid pilot。

中国版

让机器人厂商和 SI 更容易卖到海外和大客户

AMR、cobot、仓储机器人、实验室自动化、工业巡检、清洁、服务机器人。痛点是 demo 多，但 SLA、数据边界、保险语言、英文 SOW 和海外采购材料弱。

海外版

让 warehouse、factory、lab 买方低风险批准机器人试点

面向 food/CPG、life sciences、semiconductor/electronics 和 3PL。强调 baseline、ROI、safety、cyber、insurance、liability、support 和 rollout option。

¥3万-15万 中国 scoping / pilot readiness sprint。
¥30万-100万 中国 paid pilot package，按场景复杂度校准。
$10k-50k 海外 pilot readiness sprint。
$50k-250k 海外 paid pilot package + edge evidence + support planning。
2%-5% 未来对执行试点、RaaS 合同或 milestone payment rails 收交易费。

08 · Competition & Moat

护城河不是合同文本，而是 telemetry-backed pilot knowledge base。

PilotContractKit 不做 RaaS vendor、lead marketplace、fleet dashboard 或法律事务所。它的机会是中立地回答一句商业问题：这台机器人是否满足了合同里的试点 outcome？

Pilot Knowledge

场景、任务、KPI、失败类型、site readiness、punch list、rollout patterns。

Telemetry Acceptance

把 Formant/Viam/Foxglove/InOrbit/vendor APIs 数据变成 milestone evidence。

Cross-Vendor Benchmarks

cycle time、intervention、deployment duration、ROI、support burden、contract outcome。

Risk Clause Library

safety、data、cyber、insurance、support、change control、stop conditions。

Buyer Trust

不卖某台机器人，先站在试点是否能被验收的中立位置。

Edge Evidence Default

每个试点都留下 Qualcomm runtime profile、board/runtime evidence 和 rollback path。

LabForgePilot 造任务，BoardBringupKit 证明板卡 ready，EdgeRuntimeBench 证明 runtime，DualCloudOps 证明本地优先和云边界，SafetyOps 管门禁，RevenueStack 管报价。PilotContractKit 把这些放进买方能签字的试点包。

10 · Demo & Ask

三分钟证明：一个 robot task 可以从 demo 进入可采购 paid pilot。

演示不是下载合同模板，而是现场生成一个试点包：客户痛点、Qualcomm target、SOW、baseline、KPI、risk boundary、edge evidence、pilot price、rollout/RaaS/subscription path。

0:20

选择 LabForgePilot 或 FactoryPilot，展示客户痛点、Qualcomm target、timeline、price anchor。

0:50

生成 Pilot SOW：scope、baseline、KPI、site boundary、RACI、deliverables、exclusions。

1:30

导入 BoardBringupKit 与 EdgeRuntimeBench：board、camera、NPU/runtime、p95 latency、rollback。

2:25

SafetyOps + DualCloudOps + RevenueStack：安全门禁、数据边界、本地优先、SLA、报价。

3:00

导出 `pilot-contract-audit-pack.zip`，包含 SOW、scorecard、risk boundary、edge evidence、scale memo。
