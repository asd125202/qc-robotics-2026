01 · Pilot Package

商业化不是说服客户买机器人，而是先让客户买一个可控试点。

每个试点只选一个主任务，配套硬件、数据、训练、部署、运行证据和复盘视频。试点成功后再进入多机运维和行业包。

Scope

LabForge、FactoryPilot、LogisticsMove 或 EduForge 中选择一个主任务。

Baseline

记录人工流程、任务频率、失败类型、安全边界和数据可用性。

Deliverables

硬件、dataset、训练报告、edge profile、安全日志和复盘视频。

Acceptance

成功率、延迟、接管次数、稳定性和客户可复现程度。

Expansion

试点成功后进入 Fleet Kit、Industry Pack 和持续训练订阅。

02 · Acceptance Metrics

试点能不能成交，取决于验收指标是否清楚。

评委和企业客户都需要看到可量化证据。PilotContractKit 把技术指标和商业指标放在同一张验收表里。

Success

任务成功率、失败类型和稳定运行时长。

Time

单次任务耗时、接管时间和训练迭代周期。

Data

示教数量、失败样本、数据质量和版本记录。

Runtime

Qualcomm edge 延迟、资源占用和部署包证据。

Safety

急停、软边界、回滚和审计日志。

Expansion

是否值得升级 Fleet Kit、行业模板和年度订阅。

03 · 4-8 Week Contract

把销售周期切成四个工程阶段。

企业不需要承诺一次性大部署。先用 4-8 周完成场景发现、硬件 bring-up、训练部署和试点复盘，再决定是否进入多机和订阅。

  • Phase 0：1 周 discovery，确认场景、任务、数据边界和验收标准。
  • Phase 1：1-2 周 bring-up，完成板卡、相机、IO、安全和遥操作采集。
  • Phase 2：1-2 周 training，完成 CloudTwin 训练、评估和 edge profile。
  • Phase 3：1-2 周 pilot run，运行真实任务、失败回流、复盘和扩展报价。

04 · Qualcomm Business Value

每个试点都应该留下 Qualcomm edge deployment evidence。

PilotContractKit 让 Qualcomm 的价值不只停留在开发板，而是进入客户验收、技能上架、队列运维和行业模板。试点成功后，客户继续购买的是围绕 Qualcomm edge target 的持续训练和运维。

Pilot

客户购买低风险试点合同。

Data

真实任务数据进入 DataFlywheel。

Edge

策略部署到 Qualcomm runtime。

Fleet

试点成功升级多机运维。

Industry

多客户复用形成行业模板。

PilotContractKit 把“我们有商业潜力”变成“客户可以先买一个低风险试点，然后用数据决定是否扩张”。
