Pitch 22 · Procurement-Ready Robotics Pilot

企业不是不想试机器人，是不知道怎样低风险购买一次试点。

PilotContractKit 把机器人 Demo 变成可采购、可验收、可扩张的企业试点包：任务范围、KPI、数据权益、安全边界、云训练额度、Qualcomm edge 运行证据、支持责任和扩张报价一次写清。

01 · Problem

机器人 Demo 很多，能被采购、法务、运营、IT 和安全一起批准的试点很少。

机器人团队通常能让本体动起来，却很难回答企业内部审批问题：到底试哪个任务、怎么算成功、谁拥有数据、失败谁处理、云端能不能看视频、现场安全边界在哪里、试点后买什么。结果是演示很热闹，合同推进很慢，试点后规模化更难。

采购担心锁死

一次 Demo 之后就被推向大额 CapEx，没有取消权、扩张价和退出路径。

运营担心无效

没有 baseline、测试周期、产能指标、接管规则和 punch list。

IT 担心数据

视频、遥测、生产数据、云区域、保留期限和模型训练权没有写清。

安全担心边界

云 dashboard、急停、人工监督、rollback 和现场风险责任容易混在一起。

工程担心证据

没有 latency、uptime、intervention、训练版本和 edge profile 的可验收记录。

02 · Why Now

机器人购买正在从“看演示”转向“要可量化的试点 ROI”。

工业机器人、专业服务机器人和 RaaS 都在增长，但买方越来越关心 ROI/payback、劳动力节省、吞吐、集成、准确率和 uptime。机器人企业如果只卖愿景，很难通过企业审批；如果能先卖一个有基线、有验收、有退出、有扩张价的试点，成交阻力会小得多。

542K IFR：2024 年全球新增工业机器人约 54.2 万台，说明自动化预算真实存在。
199K+ IFR：2024 年专业服务机器人销量接近 20 万台，物流/运输是最大类别。
31% IFR：2024 年专业服务机器人 RaaS fleet 增长约 31%。
55.6% A3：2026 Q1 北美 cobot 订单按台数同比增长 55.6%。

03 · Insight

让机器人可规模化之前，先让机器人试点可采购。

Formic、Locus、Brightpick 证明 RaaS/托管部署能降低 CapEx 恐惧；Formant、Viam、Foxglove、InOrbit 证明 fleet software 和运行证据有价值；UR+、FANUC、ABB、KUKA 证明认证伙伴和预测试包能降低风险。缺口是跨厂商、跨场景的试点采购层：把范围、KPI、风险、数据、训练、edge evidence 和扩张路径绑定成一个可审批包。

Make the robot pilot purchasable before it is scalable.

04 · Solution

PilotContractKit 是机器人企业试点产品包，不是法律合同生成器。

它帮助机器人厂商、系统集成商和客户把一次 4-8 周轻量试点，或复杂现场 60-90 天付费试点，变成采购、运营、安全、IT 和工程都能审查的工作包。法律、安全、保险和监管仍需要专业团队审核；PilotContractKit 负责把证据和边界组织好。

  • Scope：机器人类型、场地范围、任务、排除项、速度/载荷/区域和 change control。
  • Baseline：人工流程、任务频率、当前成本、失败类型、风险和数据可用性。
  • Evidence：LeRobot 数据集、训练 job、edge profile、日志、运行视频和验收报告。
  • Risk Boundary：安全边界、人工监督、rollback、数据权利、网络安全和退出条款。
  • Expansion：Fleet Kit、Industry Pack、RaaS/月费、私有技能库和持续训练订阅。

05 · Product Workflow

不是先卖大系统，而是先帮客户内部立项。

销售动作从“请客户看技术”变成“帮客户写清审批材料”：一页 ROI、一个任务、一个验收表、一个数据边界、一个安全边界、一个试点价格、一个扩张报价。

选任务

LabForge 样品转移、FactoryPilot 巡检、LogisticsMove 搬运或 EduForge 训练。

定基线

记录人工流程、单位产出、失败类型、干预次数和现场约束。

生成包

scope、KPI、数据权利、安全边界、支持 SLA、验收表和报价。

运行试点

Teleop/HIL 数据采集、LeRobot 训练、Qualcomm edge 部署和日志。

转扩张

用验收报告决定是否进入 fleet、industry pack 或 RaaS/月费。

06 · Acceptance Metrics

验收指标必须是业务指标，不只是机器人成功率。

企业买方关心 ROI/payback、劳动力节省、吞吐、准确率、集成和 uptime。PilotContractKit 把技术指标和业务指标放在同一张 scorecard 里，并写清 baseline、测试周期、测量方法、排除项、pass/fail 阈值和 punch list 处理方式。

Throughput

units/hour、picks/hour、cases/hour、单次任务耗时和 changeover time。

Quality

订单准确率、错拣/漏检、缺陷率、返工率和客户可复现程度。

Autonomy

任务成功率、intervention rate、人工接管时间和失败类型分布。

Uptime

最小运行小时、停机原因、支持响应、备件和恢复时间。

Integration

WMS/MES/ERP/API 接入完成度、网络权限和操作员培训完成。

Edge Evidence

p50/p95 latency、action rate、资源占用、温度、功耗和 rollback 记录。

07 · Market Wedge

先服务已经为机器人试点卡住的买方。

第一批客户不是“所有企业”。最现实的 wedge 是已经有自动化需求、已经看过 Demo、但缺采购路径的人：中型食品/CPG 工厂、3PL/电商/生命科学仓库、中国电子/汽车供应链/食品/纺织工厂、系统集成商和机器人厂商。

食品/CPG

装箱、码垛、质检、上下料和物料搬运，买方关心产出、劳动力和 uptime。

仓储/3PL

AMR、拣选、库存可视、tote movement 和 WMS 集成。

中国工厂

硬件供给强、价格竞争强，需要本地采购模板、数据边界和集成证据。

系统集成商

把定制交付变成可复制试点包、FAT/SAT 式验收和支持责任矩阵。

机器人厂商

Demo 做得出，但需要帮客户内部推进预算、法务、IT、安全和运维审批。

08 · Business Model

先卖可控试点，再转 RaaS、运维和行业模板。

商业模式不从“大规模替代人工”开始，而从一个可衡量试点开始。试点结束后，客户用数据决定是否扩张，我们用同一套证据包转向 fleet、行业模板、持续训练和服务合同。

诊断/Scoping

付费 discovery：现场、数据、风险、KPI、硬件和集成估算。

固定价试点

30万-100万元人民币区间作为 pitch 锚点，按场景复杂度校准。

云训练/Edge

训练额度、数据处理、AI Hub/profile、部署包和运行报告。

RaaS / 订阅

试点成功后转月费、运维、备件、SLA、OTA 和持续训练。

SI 分成

让系统集成商用标准包卖更多可验收项目，而不是每次重谈。

09 · Go To Market

从“帮供应商成交试点”切入，而不是等企业直接买平台。

机器人厂商、系统集成商和 Qualcomm 生态伙伴最痛的是 Demo 之后合同推不动。PilotContractKit 可以作为他们的销售工具、交付模板和验收 evidence binder，先帮他们拿下 paid pilot，再把成功项目转成模板。

Partner

找机器人厂商、SI、教育/工厂试点方做 2-3 个样板包。

Template

把 LabForge、FactoryPilot、LogisticsMove 做成标准试点生成器。

Evidence

每个试点留下数据、训练、edge profile、KPI 和风险边界。

Channel

给 SI/OEM 一套可复用 SOW、验收表、报价和扩张路径。

Scale

试点成功转 Fleet Kit、Industry Pack、SafetyOps 和 DataFlywheel。

10 · Competition

PilotContractKit 坐在 fleet tools、RaaS、集成商和采购平台之间。

我们不替代 Formant、Viam、Foxglove、InOrbit、Locus、Formic、UR+、FANUC、ABB、KUKA 或系统集成商。它们证明市场需要 fleet ops、RaaS、认证伙伴和预测试包；PilotContractKit 的差异是把试点变成采购、法务、运营、安全、IT 和工程都能批准的跨厂商工作包。

Fleet SaaS

强在运行和观测；弱在试点前的采购范围、验收、数据权利和安全边界。

RaaS Vendor

强在托管部署和月费；通常绑定自家机器人或特定仓储/工厂场景。

Integrator

强在现场交付；弱在 LeRobot 数据、edge evidence 和跨客户模板化。

11 · Risk Boundary

把风险写清，比承诺“全都安全合规”更可信。

PilotContractKit 是试点风险和证据组织工具，不是法律意见、保险意见、安全认证或出口合规结论。它把专业团队需要审查的材料整理出来，并把云 dashboard 的角色限制在运营可视、报告、维护支持和配置流程，不把它说成 safety-rated control system。

安全边界

本地急停、防护、保护停止、风险评估和现场监督仍由机器人/集成流程验证。

数据权利

区分客户数据、遥测、视频/音频、个人信息、生产数据和聚合学习。

人工监督

指定 site owner、vendor operator、stop-work authority、培训记录和重启权限。

Rollback / Exit

安全事件、near miss、网络事件、严重 bug 或 KPI 未达标时的退出和复盘。

Cyber / Export

网络分段、MFA/RBAC、日志、SBOM、区域选择、ECCN/实体清单审查入口。

任务模板

LabForge、FactoryPilot、LogisticsMove、EduForge 和更多行业场景。

KPI Ledger

Evidence Binder

Channel Memory
