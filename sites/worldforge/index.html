Pitch 28 · Simulation CI & Synthetic Data Factory

WorldForge

仿真不是目的，规模化可验证机器人数据工厂才是产品。WorldForge 把真实示教、合成变体、失败挖掘、LeRobot 数据集、云训练和 Qualcomm edge validation 串成一条技能生产线。

00 · Five-Thread Research

具身智能的瓶颈，正在从模型参数转向数据、评测和端侧上线。

五条并行研究线给出同一结论：NVIDIA、Google、Physical Intelligence、Skild、Covariant 和 Figure 都在建设数据飞轮；Isaac、MuJoCo、Gazebo、Genesis 已经足够强；真正的产品机会是把它们编排成可交付的 simulation CI。

Foundation Models

GR00T、Gemini Robotics、OpenPI、Helix、RFM 都需要 post-training 和回归评测。

Sim Backends

Isaac 做高保真，MuJoCo 做快速动力学，Gazebo 做 ROS 兼容，Genesis 做实验加速。

Data Flywheel

LeRobotDataset、HIL correction、failure replay 和 benchmark 变成产品 rails。

Business

仓库、工厂、实验室为更快切换 SKU / protocol / station 和更低失败率付费。

Qualcomm

云端训练后必须经过 static export、AI Hub profile、QNN/ONNX 和真实板端验证。

01 · Skill Production Line

从一条真实示教，到一套可以上线的机器人技能证据。

WorldForge 不卖“一个仿真界面”，而卖从真实数据到合成数据、从评测到训练、从云端策略到 Qualcomm 本体运行的完整证据链。

Capture

Teleop demo、ROS bag、camera、LiDAR、robot state、HIL takeover 和失败片段。

Forge

real-to-sim 校准，生成位置、光照、遮挡、摩擦、相机和对象变体。

Validate

标准 benchmark、行业 stress test、失败聚类和上线门禁。

Train

导出 LeRobot 数据集，走中国 / 海外云 GPU 训练，保留 provenance。

Edge Gate

AI Hub profile、QNN / ONNX / TFLite artifact、真实 Qualcomm 板端验证。

02 · Backend Adapters

不要重写物理引擎，要把现有工具变成客户能用的生产系统。

WorldForge 的护城河是兼容性、数据契约、评测门禁和端侧部署，而不是声称自己比所有仿真器都强。

Isaac

USD 场景、Replicator 标注、视觉传感器、trajectory expansion 和高保真评测。

MuJoCo

快速 dynamics、system identification、RL rollout 和 contact stress test。

Gazebo

ROS 2、SDF、Nav2、客户已有工程和低成本教育 / 比赛兼容。

Genesis

新兴多物理后端，用于 deformable、tactile、contact-rich 和并行 rollout 探索。

Robo Packs

robosuite、RoboCasa、RoboVerse 任务包，转成工业、实验室、物流变体。

LeRobot

统一导出视频、状态、动作、语言、场景参数、失败标签和数据血缘。

03 · Failure Factory

失败不是事故，是下一轮训练最值钱的数据。

企业机器人试点失败，通常不是因为 demo 不会跑，而是新 SKU、新光照、新摩擦、新容器、新遮挡把策略击穿。WorldForge 把这些失败变成可复现、可标注、可训练、可回归的任务。

  • Failure mining：grasp miss、slip、collision、occlusion、wrong object、timeout。
  • Synthetic variants：从少量真实轨迹扩展出可控场景组合。
  • Simulation CI：每次策略更新自动跑行业 benchmark 和 stress test。
  • Promotion gate：只放行干预率、复发率、cycle time、edge latency 达标的版本。

04 · Qualcomm Edge Gate

把 Qualcomm 从“硬件选择”变成“机器人技能上线门禁”。

WorldForge 的最后一关不是云端 benchmark，而是真实 Qualcomm edge target。策略必须通过静态导出、量化校准、AI Hub profile、真实板端延迟 / 内存 / 功耗 / ROS 2 loop timing，才能进入 SafetyOps 和 EdgeFleet。

RB3 Now 用 RB3 Gen 2 / QCS6490 / QCS8550 做当前原型和复赛实测。
IQ10 Ready 长期路线对齐 Dragonwing IQ10 robotics reference design。
AI Hub 模型转换、profile、numerical check、runtime mapping 进入正式门禁。
Canary 部署包带 hash、calibration report、rollback 和现场失败回流。

WorldForge 的目标不是让仿真取代真实世界，而是让每一次真实失败都能变成下一版技能的可验证训练资产。
