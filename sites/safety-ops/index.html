Pitch 47 · Robot Skill Release Gate

SafetyOps

机器人不是训练完就上线。SafetyOps 是机器人模型和技能上线前的安全门禁：谁批准、能控制什么、在哪些场景运行、失败后如何停用、证据如何交给客户、认证和保险。

看上线阻力 看门禁演示

01 · Problem

企业不是怕机器人不聪明，而是怕策略不可控。

Demo 阶段只要机器人能动；生产阶段必须回答：这个模型是谁批准的？能发哪些 ROS/DDS 指令？在哪个 ODD 生效？如果撞到人、越界、延迟飙升、传感器失效，如何停用、回滚、追责和给保险/客户看证据？

Change

事故常发生在变更时刻

编程、维护、测试、调试、路线变更、模型升级和人工 override，是企业最需要证据的环节。

Permission

技能边界不清

一个技能到底能读哪些传感器、写哪些 topic、控制哪些执行器、在哪些区域运行，经常写在文档里。

Release

模型上线缺门禁

数据集、评测、HIL、replay、QNN profile、机器人 profile 和 rollback 目标没有绑定成一个 release record。

Liability

保险和客户要事件链

事故后需要证明版本、权限、传感器状态、命令、停机、人工接管和恢复动作。

Export

出海要安全证据

ISO/ANSI/EU/OSHA/NIST/GB 语境下，客户要的不是口头承诺，而是可导出的 safety-case artifacts。

02 · Current Alternatives Fail

今天的工具各管一层，但没人把“模型变更到现场事故”串成证据链。

SafetyOps 不替代 Safety PLC，不替代 ISO 认证，也不替代 Formant、InOrbit 或 Foxglove。它把训练、权限、发布、运行时边界、incident replay 和审计包连接起来，服务 robot OEM、SI、企业客户、保险和评委。

PDF / Spreadsheet

风险评估和验收材料很难跟上模型版本、skill permissions、OTA、site config 和现场日志。

Safety PLC

认证安全控制器保护机器和产线，但不记录 AI model lineage、dataset、QNN profile 和 skill approval。

Fleet Ops

Formant、InOrbit 管 fleet 和 incident，但不是以 release gate、safety-case evidence 和 certification support 为核心。

Robot Logs

Foxglove、MCAP、ROS bag 很适合调试，但还需要权限、审批、回滚和客户可读的 audit pack。

AI Governance

Credo、IBM、Holistic 等管数字 AI 风险，但通常不理解 ROS topics、执行器、E-stop、ODD 和边缘延迟。

03 · Solution

SafetyOps 是机器人技能上线前的安全门禁和事故证据层。

第一阶段只做一个强 wedge：Skill Release Safety Gate。每个技能上线前，必须声明权限、ODD、安全包络、模型证据、QNN profile、回放评测、审批人、灰度策略和回滚包；运行时再用 command admission 和 incident ledger 记录边界触发。

Declare

技能声明 sensors、actuators、ROS/DDS allow/deny、ODD、risk level 和 rollback target。

Evaluate

固定 eval split、replay tests、HIL clips、QNN/AI Hub profile 和 robot compatibility。

Approve

safety owner、fleet admin、site owner 对 canary、scope 和 rollout gate 签名。

Enforce

边缘 safety-agent 阻止 raw command、越界速度、禁区、传感器失效和延迟异常。

Replay

事件进入 MCAP ring buffer、incident ledger、rollback record 和 LeRobot/HIL 数据队列。

04 · Why Now

机器人规模、AI 策略更新和监管压力同时把 release gate 变成刚需。

全球工业机器人在役量已达数百万级，服务机器人和 RaaS fleet 也在扩张。与此同时，ISO 10218:2025、EU Machinery Regulation、Cyber Resilience Act、EU AI Act、NIST AI RMF 和中国 GB/AI 安全治理都在把“软件更新、网络安全和 AI 风险”推入机器人安全文件。

542K IFR 报告 2024 年全球工业机器人新增安装约 542,000 台。
4.66M 全球工业机器人在役量约 4.66M，变更治理不再是小众问题。
2025 ISO 10218 于 2025 更新，安全、软件和网络攻击语境更突出。
2027 EU Machinery Regulation 将于 2027-01-20 适用，自动移动机械进入更清晰监管周期。
$165K 2026 年 OSHA willful/repeat 最高罚款超过 $165k，事故证据影响真实成本。

05 · Product

从 release gate 切入，再扩展到 fleet safety passport。

SafetyOps 不做认证机构。它交付的是 safety-case workflow：让 robot OEM、SI 和企业客户把每次 robot/model/skill change 变成可追踪、可审批、可回滚、可导出的证据包。

  • Skill Registry：signed manifest、SBOM、model lineage、robot profile compatibility 和 rollback target。
  • Release Gate：eval、replay、HIL、AI Hub/QNN profile、approvals、canary rules 和 rollout stop 条件。
  • Safety Agent：在 ROS 2 stack 旁运行，观察 ODD、sensor health、thermal、network、E-stop 和 QNN runtime。
  • Command Admission：阻止 raw cmd_vel、arm trajectory、gripper force、map change、OTA 等高风险命令。
  • Incident Ledger：append-only timeline、MCAP/video ring buffer、replay pointer、hashes、custody 和 rollback proof。

06 · Product API

安全不是一个勾选框，而是一组版本化 artifact。

所有 release 都绑定 skill manifest、dataset hash、evaluation、edge profile、approvals、runtime policy、incident ledger 和 audit pack。LeRobot/AI policy 永远不绕过认证安全控制器；ROS policy 是软件 guardrail，不是 Safety PLC。

SkillManifest

skill_id、risk_level、sensors、actuators、ROS allow/deny、ODD、rollback target。

ReleaseGate

dataset_hash、success_rate、intervention_rate、replay regressions、approvals、decision。

EdgeProfile

AI Hub job、target device、QNN runtime、latency p95、memory、compute units、numeric delta。

RuntimePolicy

DDS enclave、command TTL、speed/force/geofence、confidence/OOD、sensor/thermal/network state。

IncidentEvent

policy violation、near miss、E-stop、manual takeover、QoS miss、rollback action、MCAP pointer。

AuditPack

buyer pack、judge pack、insurer pack、standards map、claim boundaries、data lane summary。

07 · Market & Business Model

卖给机器人公司的是更快进 pilot，卖给企业的是更少部署风险。

第一批买家是正在从 demo 走向付费 pilot 的机器人 OEM、系统集成商和企业自动化团队。SafetyOps 帮他们减少 release review 时间、降低 warranty/SLA 争议、生成 insurer-ready evidence，并让客户验收更快。

08 · Competition & Moat

竞争证明需求真实，但 SafetyOps 的位置在“物理 AI 发布证据”。

FORT、3Laws、Veo、NVIDIA Halos、Formant、InOrbit、Foxglove、Viam、Safety PLC vendors、AI governance platforms 都解决一层。SafetyOps 的 wedge 是 Qualcomm-edge robot skills 的 release gate、runtime command evidence 和 cross-OEM safety event schema。

Safety Event Schema

跨 OEM 记录 skill、ODD、command、policy denial、QNN profile、incident、rollback 和 warranty evidence。

Hazard / Skill Library

dock assist、pick、carry、clean、patrol、elevator、charging 等技能逐步沉淀权限和安全包络模板。

Insurer Graph

版本、地点、触发、停机、人工接管和恢复形成 underwriting / claims 可读的风险图谱。

Edge-Signed Evidence

Qualcomm target 上的 latency、memory、thermal、QNN runtime、signed logs 和 replay evidence 形成上线门槛。

09 · Why Qualcomm

Qualcomm 不只需要机器人跑 AI，还需要 AI 策略能被批准上线。

SafetyOps 把 Qualcomm edge 从推理目标升级成物理 AI release target：每个模型都有 AI Hub/QNN profile、runtime artifact、latency/memory、numeric delta、target device、rollback 和 incident evidence。

Profile

AI Hub / QNN profile 进入 release gate，不再只说 TOPS 或 demo FPS。

Enforce

本体侧 safety-agent 检查 ODD、sensor health、network、thermal 和 command policy。

Rollback

release manifest 绑定 previous-known-good，事故时一键停用和回滚。

Evidence

signed logs、MCAP replay、QNN runtime、incident timeline 进入客户验收包。

Ecosystem

OEM、SI、企业客户、认证实验室、保险和 RaaS financier 围绕 Qualcomm-ready evidence 协作。

10 · Demo & Ask

八分钟演示：dock-assist-v2 技能被门禁拦截、限权上线、触发 incident、再回滚。

比赛 demo 可以用桌面 AMR/机械臂模拟器：上传新技能，缺少 human-proximity eval 被挡住；补上 HIL/replay/QNN profile 后只允许 canary；运行时 raw /cmd_vel 被拒绝；人进入 keepout zone 触发停机、replay、回滚和 audit pack。

1. Gate

技能卡显示 sensors、actuators、ROS allow/deny、ODD、risk、rollback target 和 missing evidence。

2. Profile

发布门禁读取 dataset hash、replay pass、HIL clips、AI Hub/QNN latency、memory 和 approvals。

3. Incident

canary 部署后，unsafe raw command 被拒绝；keepout 事件触发 slow/stop 和 incident ledger。

4. Rollback

打开 MCAP replay，失败片段进入 LeRobot/HIL 队列，一键回滚并导出 judge-audit-pack.zip。

我们的申请：用 Qualcomm edge target 打造 robot skill release gate reference workflow，让评委看到机器人从训练到上线、事故到回滚、证据到保险/客户验收，如何成为商业产品的一部分。

SafetyOps 不声称通过 ISO/CE/OSHA/EU AI Act，不替代 safety PLC 或认证实验室，也不保证零事故。它卖的是物理 AI 发布证据层：让每个机器人技能在上线前后都有权限、边界、评测、运行时证据、回滚和审计包。
