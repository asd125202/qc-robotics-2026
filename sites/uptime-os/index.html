00 · Five-Thread Research

机器人从 demo 走向规模化，瓶颈会从部署转向持续在线。

五条研究线显示：机器人存量和 RaaS 队列已经足够大；远程诊断、备件、现场技师、SLA 和一次修复率决定 ROI；中国客户把售后能力当采购门槛；领先厂商都在把 uptime 做成服务商品。

Downtime

非计划停机是制造与物流的真实成本锚点，机器人服务必须围绕 MTTR、MTBF 和一次修复率设计。

Installed Base

IFR 口径 2024 年全球工业机器人运行存量约 4.664M，专业服务机器人销售接近 200,000 台。

RaaS Risk

月付降低客户 CapEx，但把折旧、维护、备件、质保和 SLA 风险转移给运营商。

China Service

中国市场需要 400/企微入口、本地备件库、授权服务商、现场响应和 SI 交付生态。

Aftermarket OS

机会不是再做一个 dashboard，而是服务记录、健康分、备件、工单和证据的统一操作系统。

01 · Service Data Plane

每台机器人都应该有服务记录、健康分和可用率合同。

UptimeOS 位于 RobotCoreOS 之上、客户业务系统之下。边缘侧采集和预处理诊断信号，云端编排工单、备件、SLA、证据和训练闭环；安全关键动作仍由 SafetyOps gate 和现场授权控制。

Sense

ROS、CAN、BMS、热区、NPU/GPU、定位、充电、校准和安全状态本体采集。

Packet

自动生成日志包、MCAP 片段、健康信号、错误码、证据 hash 和脱敏状态。

Predict

Qualcomm edge 先做异常检测和健康分，云端汇总 RUL、原因码和服务计划。

Dispatch

备件兼容性、库存、授权服务商、技师 checklist 和客户签收进入一个工单。

Learn

RiskLedger、CertForge、RobotLeaseOps 和 LeRobot failure mining 更新同一资产记录。

02 · Product Modules

售后不是维修单，而是机器人全生命周期的资产系统。

UptimeOS 把服务工作拆成六个可卖模块：健康护照、边缘诊断、事件证据包、FRU/备件图谱、技师移动工作流和 RaaS 可用率账本。

Health Passport

序列号、运行小时、电池循环、固件、模型、校准、维修、质保和 SLA。

Edge Agent

低带宽诊断、离线缓存、本地脱敏、异常优先上报和安全审计。

Incident Packet

任务上下文、地图位置、日志、视频/MCAP、错误码和建议处理步骤。

FRU Graph

BOM 到可更换件、兼容性、库存、替代件、交期和质保状态。

Tech Mobile

QR 扫码、离线 checklist、备件消耗、照片、签收和回访评价。

Uptime Ledger

可用率、MTTR、一次修复率、服务成本、残值和续约风险。

03 · China Lane / Overseas Lane

同一个售后内核，需要两种商业语言。

中国客户看重本地服务网络、快速响应、备件前置和 SI 项目交付；海外客户更强调支持门户、RMA、warranty reserve、IT 安全、OT 分区和服务证据。

中国版

400 / 企微 / 微信小程序入口、区域备件中心、授权服务商、24-48 小时到场、中文故障码、WMS/MES/PLC/电梯/门禁/消防接口模板。

海外版

OEM / RaaS support portal、ServiceNow / Jira / Zendesk adapter、RBAC、audit log、JIT remote access、depot repair、residual value 和 insurability data。

04 · Demo Story

从电机异常，到工单、备件、签收和训练回流。

比赛 demo 不需要真实搭一个售后网络。浏览器里展示一条闭环就够：电机电流异常，边缘 agent 生成 incident packet，系统推荐 FRU，技师扫码完成维修，证据进入 RiskLedger，失败片段进入 LeRobot。

  • Edge detection：本体侧识别电机温升和电流趋势。
  • Service twin：显示健康护照、质保、SLA、备件兼容性和库存。
  • Technician flow：QR 扫码、离线 checklist、照片、备件消耗和客户签收。
  • Closed loop：RobotLeaseOps 计算可用率，CertForge 记录维修变更，LeRobot 标记 failure episode。
