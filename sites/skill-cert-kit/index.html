Pitch 23 · Pre-Install Trust Layer

SkillCertKit

企业不会把陌生模型随手装进真实机器人。SkillCertKit 把一个策略模型变成 profile-scoped evidence package：数据血缘、硬件兼容、Qualcomm edge profile、安全边界、SBOM、签名、安装门禁、回滚和版本记录，先过信任门，再进 SkillDock。

从安装风险开始 连接 SkillDock

01 · Problem

机器人技能如果没有安装前证据，就只能停留在一次性集成。

Demo 能跑，不代表企业敢安装。买家不知道技能是否适配自己的相机、机械臂、底盘、runtime、网络、安全 IO、数据权限和现场边界；开发者卖不了第二次，集成商重复验证，IT/安全无法审计。

买家风险

陌生模型接触真实设备

普通 App 崩溃是软件问题；机器人技能失败会影响设备、人员、现场和责任边界。

开发者风险

技能无法产品化

没有兼容矩阵、证据等级、安装手册和支持边界，同一个技能很难卖给第二个客户。

集成商风险

每个客户重新验收

相机、执行器、速度、工作空间、网络、回滚和安全接管都要一遍遍重做。

IT 风险

无法审计依赖与权限

SBOM、许可证、漏洞、云依赖、数据流、签名、日志和升级策略没有统一格式。

生态风险

Qualcomm 价值不沉淀

如果技能不声明并验证 edge target，Dragonwing 只是一块板，不是技能分发标准。

02 · Why Now

机器人技能正在软件化，但物理世界还缺信任层。

UR+ 和 MiR Go 证明企业买家接受认证生态来降低部署风险；LeRobot、DROID、Open X-Embodiment 让数据和策略可复用；Qualcomm AI Hub、QNN/QAIRT、Device Cloud、Profiler、Dragonwing Robotics Hub 让端侧 profile 和部署证据可以产品化。

542K IFR 2025：2024 年全球工业机器人新增安装约 54.2 万台。
160+ MiR Go 已有 160+ AMR 应用/模块，说明买家重视已测试生态。
SBOM 企业采购、CRA readiness、SSDF 和安全审查正在要求更清晰软件证据。
QNN Qualcomm edge artifact 可以从开发工具变成技能上架证据。

03 · Solution

SkillCertKit 不是法律认证机构，而是 marketplace-readiness evidence engine。

它为命名 robot profile 和测试条件生成结构化证据：这个技能来自哪些数据、在哪些硬件上验证过、能否在 Qualcomm edge 上运行、有什么安全假设、哪些安装会被阻止、失败后如何回滚。

Submit

提交 policy artifact、`skill.yaml`、task boundary、依赖、目标 robot profile 和证据等级。

Trace

绑定 LeRobot dataset、episode hashes、训练版本、model card、dataset card 和 license。

Profile

生成 AI Hub / QNN / QAIRT / ONNX Runtime QNN evidence、latency、memory、fallback。

Gate

运行 SBOM、VEX、签名、runtime constraints、safety boundary、OPA policy 和 rollback gate。

Publish

生成 SkillDock 技能卡：兼容范围、证据等级、价格、支持方和 claim boundaries。

04 · Evidence Bundle

技能包应该是 digest-addressed evidence graph，不是一个模型文件。

根对象是 `skill_id:version@sha256`。模型、容器、`skill.yaml`、SBOM、VEX、签名、SLSA/in-toto provenance、edge profile、safety evidence、policy decision 和 rollback metadata 都挂在同一个 digest 上。

  • 证据等级：`simulated`、`hardware_in_loop`、`bench_real`、`field_real`。
  • 兼容标签：`tested_on`、`expected_compatible`、`not_validated`。
  • 安全语言：记录 assumptions，支持 safety-case preparation，不声称法定安全认证。
  • 区域路径：中国版、海外版分别维护 GB/GB/T、EU CRA/Machinery、NIST/IEC 等证据映射。

05 · Trust Gates

安装前要能说“不”，否则 SkillDock 只是高风险模型仓库。

SkillCertKit 的价值不是生成漂亮报告，而是把证据变成可执行门禁。每一次允许、阻止、警告、回滚都成为后续 compatibility graph 的训练数据。

Gate 1

Signature

验证 root artifact、SBOM、edge profile 和 policy decision 的签名。

Gate 2

Lineage

检查 dataset、training run、model export、container build 和 QNN compile 的 provenance。

Gate 3

Profile

确认 robot profile 满足 camera、actuator、firmware、calibration、安全 IO 和 edge target。

Gate 4

Security

SBOM、license、VEX、CVE、container privilege、network egress 和 cloud dependency。

Gate 5

Safety

工作空间、速度/力假设、急停、人工接管、fail-closed 和禁用动作。

Gate 6

Rollback

previous-known-good、健康阈值、状态清理、操作员确认和审计日志。

06 · Market & Business Model

先卖 evidence pack，再开技能市场。

第一批客户不是消费者，而是为部署信任付费的人：skill developers、robot OEM、系统集成商、RaaS/fleet operator、企业 IT/安全、教育/研究实验室。SkillCertKit 让技能可卖，SkillDock 让技能可买、可装、可升级、可回滚。

Enterprise Trust

07 · Competition & Moat

SkillCertKit 不替代 ROS、LeRobot、Viam、Isaac 或认证实验室。

ROS 解决包和依赖；LeRobot 解决数据/训练；Viam 解决 registry 和 fleet rollout；NVIDIA Isaac/GR00T 解决 GPU-first physical AI 工具链；UL/TUV/Intertek 做正式安全/法规认证。SkillCertKit 的位置是安装前 marketplace-readiness evidence，帮助技能进入这些流程，而不是冒充它们。

短期壁垒

evidence schema

`skill.yaml`、Dragonwing Skill Profile、safety boundary、SBOM/VEX、policy decision 和 rollback 模板。

中期壁垒

compatibility graph

技能、数据集、模型、robot profile、sensor、runtime、Qualcomm target、现场失败的关系图。

长期壁垒

recertification network

硬件变更、runtime 更新、现场失败、漏洞和新数据触发再验证，形成持续收入与信任数据。

09 · Demo & Evidence

3 分钟演示：从模型文件，到可阻止、可发布、可回滚的技能证据包。
