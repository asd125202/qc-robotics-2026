Pitch 23 · Pre-Install Trust Gate For Robot Skills

SkillCertKit

机器人技能市场不会先靠 SKU 数量成功，而会先靠信任成功。SkillCertKit 把策略模型、ROS 包或机器人 app 变成 signed Skill Passport，证明它能否在某个 robot profile、Qualcomm edge target 和现场边界下被安装、灰度发布和回滚。

从安装风险开始 连接 SkillDock

01 · Problem

企业不会把陌生模型随手装进真实机器人。

Demo 能跑，不代表它适合买家的相机、机械臂、夹爪、AMR、runtime、网络、安全 IO、数据权限和现场边界。普通 App 崩溃是软件问题；机器人技能失败会影响设备、人员、现场和责任。

Developer

技能卖不了第二次

没有兼容矩阵、证据等级、安装门禁和支持边界，同一个技能很难复用给下一个客户。

Integrator

每个项目重复验证

相机、payload、速度、工作空间、网络、回滚和人工接管都要一遍遍重做。

Enterprise

安全和 IT 无法审计

SBOM、漏洞、许可证、云依赖、数据流、签名、日志和升级策略没有统一格式。

Fleet

升级影响 uptime

每次 skill update 都可能影响 SLA、责任边界、support load、incident replay 和 rollback。

Qualcomm

edge 价值不沉淀

如果技能不声明并验证 target profile，Dragonwing 只是一块板，不是技能分发标准。

没有安装前信任门，机器人技能就无法像软件一样分发。

02 · Current Alternatives Fail

市场、仿真、开源包、OEM 生态和正式认证都重要，但中间缺一层。

真正缺口不是“skill exists”，而是“skill may run here”：这个技能是否能在这个 robot profile、这个站点、这个数据规则和这个 Qualcomm edge target 上被允许安装。

App Store

下载按钮不等于物理信任

机器人 app store 证明需求存在，但工业客户还需要 safety evidence、site approval、rollback 和采购包。

OEM Ecosystem

信任被锁在单一品牌

UR+、MiR Go、KUKA、FANUC、ABB 都证明认证生态有价值，但很难成为跨机器人 skill passport。

Open Stack

ROS/MoveIt/LeRobot 不是安装批准

包、模型和数据集不会自动回答 SBOM、回滚、相机、安全 IO、权限和现场边界问题。

Fleet Ops

观测发生在部署后

Viam、Formant、InOrbit、Foxglove 能看到发生了什么；SkillCertKit 决定下一版是否可以发生。

Physical AI

仿真通过不等于生产安全

Isaac、GR00T、Cosmos、Flowstate 帮助创建技能，但还需要安装前证据和客户 policy gate。

Formal Labs

正式认证不是日常 release gate

UL、TUV、Intertek、A3 很重要，但每个 skill version 仍需要版本、签名、SBOM、profile 和 rollback。

03 · Solution

SkillCertKit 是机器人技能的 CI/CD trust gate。

它生成一个 `Skill Passport`：digest-addressed、signed、profile-scoped evidence bundle。输出不是永久安全证书，而是可执行的 `allow / warn / block` 安装决策。

Submit

提交 policy/model/container/ROS package、`skill.yaml`、任务边界、目标 robot profile、权限和支持方。

Trace

绑定 LeRobot/TrainRouter lineage、dataset card、model card、eval report、license 和 train run hash。

Profile

生成 Qualcomm AI Hub / QNN / QAIRT evidence、latency、memory、thermal、fallback 和 unsupported-op 状态。

Gate

运行 signature、SBOM/VEX、runtime constraints、safety boundary、customer policy 和 rollback gate。

Publish

生成 SkillDock 技能卡：兼容范围、证据等级、价格、支持方、claim boundaries 和 install gate。

04 · Why Now

机器人技能正在软件化，采购和法规正在要求软件证据。

IFR 装机基数、LeRobot/Physical AI 数据循环、SBOM/VEX/SSDF/EU CRA/EU Machinery 风险文档，以及 Qualcomm Dragonwing reference platform 同时成熟，让安装前 trust gate 变成商业机会。

542K IFR 2025：2024 年全球工业机器人新增安装约 54.2 万台。
~200K 专业服务机器人销量接近 20 万台，物流 RaaS 继续增长。
SBOM 企业采购和法规正在把软件依赖、漏洞、签名、更新和责任前置。
QNN Qualcomm edge artifact 可以从工程产物变成技能上架证据字段。

05 · Product

第一版只做一件事：让一个机器人技能在一个明确 profile 上被批准或阻止。

不要一开始做大而全 app store。先手工帮 10 个技能、2 个 robot profiles、3 个 launch partners 做 evidence pack，把重复 review checklist 变成 API。

  • Skill Submit：模型、ROS 包、`skill.yaml`、任务边界、权限、target profile、support owner。
  • Evidence Builder：LeRobot lineage、SBOM/VEX、SLSA/in-toto、签名、TUF/Uptane metadata、Qualcomm edge profile。
  • Gate Engine：按 robot profile、site、region、customer policy 返回 allow/warn/block。
  • Skill Passport：面向 SkillDock、采购、安全、集成商、OEM 和保险的证据室。
  • Runtime Gate：preflight、staged rollout、health thresholds、rollback、recertification trigger。

06 · Product API/Evidence

机器人技能包应该是 signed release bundle，不是一个模型文件。

根对象是 `skill_id:version@sha256`。模型、容器、ROS 包、`skill.yaml`、SBOM、VEX、provenance、edge profile、safety boundary、policy decision 和 rollback metadata 都挂在同一个 digest 上。

API

Gate as a service

`POST /v1/gates:evaluate` 按 skill digest、robot profile、site、region 和 customer policy 返回 pass/block/warn。

Passport

机器可读证据室

`GET /v1/skills/{digest}/evidence` 导出采购、审计、SkillDock 和 runtime gate 共用的证据包。

Runtime

合法回滚，不绕过安全

`POST /v1/rollbacks` 发布新的签名 rollback metadata，回到 previous-known-good digest。

Evidence Purpose Claim boundary
SBOM / VEX 依赖、漏洞、许可证、可达性和修复状态。 SBOM 是 inventory，不等于 secure。
SLSA / in-toto source、builder、build recipe、artifact digest 和发布流程。 签名不等于 SLSA 合规。
Qualcomm edge profile AI Hub/QNN/QAIRT latency、memory、runtime、fallback、unsupported-op。 模型 benchmark 不等于完整机器人延迟。
Safety boundary ODD/SOE、速度/力、workspace、takeover、fail-closed、rollback trigger。 支持 safety-case preparation，不替代正式认证。

07 · Market & Business Model

先卖 evidence pack，再开技能市场。

第一批客户不是消费者，而是为部署信任付费的人：skill developers、Robot OEM、system integrators、RaaS/fleet operators、enterprise IT/EHS/security、insurers 和教育/研究实验室。

Beachhead

Lab sample transfer / light QA

工作空间受控、文档价值高、安全边界清楚，能串起 TeleopStudio、TrainRouter、QNN profile 和 SkillDock。

Overseas

Security/EHS-led adoption

映射 NIST SSDF、SBOM/VEX、EU CRA、EU Machinery technical docs、AI Act 和 Data Act。

China

技能上架证据包

面向 OEM、SI、园区、实验室、教育套件和数据工厂，支持本地/BYOC 数据平面和 GB/GB/T 映射。

Revenue

Evidence + runtime gate

Marketplace Ready Pack、profile test、HIL validation、private library、per-robot runtime gate 和 marketplace take。

YC wedge：帮 SI/OEM 把一个技能从“能跑”变成“企业安全/采购可以接受”，不要一开始泛泛做机器人 app store。

08 · Competition & Moat

SkillCertKit 不替代认证实验室，也不替代机器人开发平台。

ROS、LeRobot、Isaac、Flowstate、Viam、Formant、UR+、MiR Go、KUKA、FANUC、ABB、UL/TUV/Intertek 都是生态的一部分。SkillCertKit 的位置是安装前 marketplace-readiness evidence，帮助技能进入这些流程。

短期壁垒

Open spec + validator

`skill.yaml`、Skill Passport、Dragonwing Skill Profile、safety boundary、SBOM/VEX/provenance adapters。

中期壁垒

Compatibility graph

技能、数据集、模型、robot profile、sensor、runtime、edge target、现场失败、buyer acceptance 的关系图。

长期壁垒

Continuous recertification

代码、模型、相机标定、夹爪、site layout、漏洞、runtime 更新和 incident 都触发再验证。

把 Qualcomm 工具链变成机器人技能信任标准。

SkillCertKit 把 AI Hub compile/profile/inference、QNN/QAIRT artifact、ONNX Runtime QNN fallback、Device run、Profiler trace、Dragonwing profile 和 Robotics Hub reference skill 变成每个技能上架前的证据字段。

Claims

10 · Demo & Ask

3 分钟演示：从模型文件，到可阻止、可发布、可回滚的 Skill Passport。

评委要看到 SkillCertKit 真正改变安装决策：不兼容 profile 被阻止，兼容 Dragonwing profile 被允许 staged rollout，health/safety event 后能回滚并触发再验证。

0:00-0:55

提交技能

打开 `labforge.sample-transfer.v1`，展示没有证据的模型文件，再提交 `skill.yaml`、任务边界和 target profile。

0:55-1:45

生成证据

绑定 LeRobot lineage、model hash、SBOM、VEX、SLSA/in-toto、signature、safety boundary 和 Qualcomm edge profile。

1:45-2:25

门禁决策

不兼容 robot profile 被 block；兼容 Dragonwing profile 允许 one-robot canary，并明确证据等级。

2:25-3:00

发布与回滚

生成 SkillDock 技能卡；模拟 safety/health event，回滚到 previous-known-good digest 并记录 audit log。

SkillCertKit 的一句话：不是给机器人技能盖法律章，而是让每个技能在安装前说清“证据、边界和回滚”。
