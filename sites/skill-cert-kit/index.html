Pitch 23 · Robot Skill Certification

SkillCertKit

机器人技能不能像模型文件一样随便安装。SkillCertKit 把 manifest、数据来源、硬件兼容、边缘 benchmark、安全门禁、签名和回滚做成技能上架标准。

01 · Certification Pipeline

技能市场的关键不是“能上传”，而是“买家敢安装”。

SkillCertKit 把机器人技能的提交、审查、边缘评测、安全门禁、签名上架和运行回滚做成一条产品化流水线。开发者负责技能创新，平台负责证据和可信交付。

Submit

提交技能包、策略 artifact、运行依赖和目标任务说明。

Manifest

检查输入输出、传感器、执行器、权限和目标硬件 profile。

Lineage

记录数据集来源、episode 数量、失败样本和版本权限。

Bench

生成 Qualcomm edge compile、profile、deploy 和运行指标。

Publish

通过 SafetyOps 门禁后签名上架，并绑定 rollback package。

02 · Evidence Pack

每个机器人技能都应该带着证据，而不是只带着权重。

认证包让企业客户、系统集成商和评委都能看到技能是否适配、是否安全、是否可回滚、是否真的运行在 Qualcomm edge target 上。

Manifest

技能 ID、输入输出、权限、依赖和目标机器人 profile。

Dataset

LeRobot 数据来源、版本、episode、失败样本和授权边界。

Hardware

RobotMac profile、相机、执行器、IO、runtime 和固件版本。

Runtime

边缘延迟、资源占用、稳定性、部署包和 profile 报告。

Safety

工作空间、速度、急停、人工接管、审计日志和发布门禁。

Rollback

上一稳定版本、回滚触发条件、恢复流程和现场支持策略。

03 · Marketplace Rules

SkillDock 不是模型仓库，而是机器人能力商店。

普通 App 可以先安装再卸载，机器人技能必须先通过认证再触碰真实设备。SkillCertKit 定义 SkillDock 的上架规则，让技能成为可采购、可验收、可续费的商业产品。

  • 未认证策略不能进入企业技能市场。
  • 每个技能必须声明硬件 profile、传感器、执行器和权限。
  • 每个技能必须附带 EdgeRuntimeBench 和 SafetyOps gate 结果。
  • 每个技能必须有 rollback package 和版本签名。

04 · Qualcomm Ecosystem Value

让 Qualcomm 从硬件选择，变成机器人技能发布标准。

如果每个技能包都需要声明 Qualcomm edge target、运行 benchmark 和兼容性证据，Qualcomm 就不只是比赛开发板，而是技能市场、企业部署和开发者生态的认证锚点。

Edge Target 技能上架前必须声明 QCS8550 / QCS6490 / IQ 系列兼容 profile。
AI Hub 把模型优化、验证和部署证据接入 Qualcomm-first 开发路径。
SkillDock 开发者围绕 Qualcomm runtime 制作可售卖、可认证技能包。
Enterprise 企业购买的是带证据的机器人能力，而不是不可审计的 demo。

SkillCertKit 把“机器人会越来越多技能”变成“每个技能都能被审查、购买、安装、运行和回滚”。
