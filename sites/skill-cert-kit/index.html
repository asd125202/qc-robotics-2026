Pitch 23 · Robot Skill Trust Layer

机器人技能不能只上传模型文件，必须带着可安装、可验收、可回滚的证据。

SkillCertKit 是机器人技能的上架信任层：把一个策略模型变成带 manifest、LeRobot 数据血缘、硬件兼容矩阵、Qualcomm edge profile、安全边界、SBOM/许可证、签名、回滚和版本记录的 marketplace-ready skill package。

01 · Problem

技能市场如果只像模型仓库，企业买家不敢安装到真实机器人上。

今天的机器人技能常常还是一次性项目交付：Demo 能跑，但买家不知道它是否适配自己的相机、机械臂、底盘、runtime、网络、安全 IO 和现场边界。开发者也很难把同一个技能卖给第二个客户，因为缺少可信版本、数据来源、硬件矩阵、性能报告、安装/回滚步骤和支持边界。

买家不敢装

普通 App 崩溃是软件问题；机器人技能失败会碰到设备、人员和客户现场。

开发者卖不动

没有认证证据、兼容矩阵和安装手册，技能很难进入企业采购。

集成商反复重做

每个客户都重新验证相机、执行器、速度、工作空间、网络和回滚。

IT/安全无法审计

SBOM、依赖、漏洞、日志、权限、数据权利和云依赖没有统一证据。

Qualcomm 价值不沉淀

如果技能不声明 edge target，Dragonwing 就只停留在硬件选型。

02 · Why Now

机器人技能开始像软件资产，但信任层还没有成型。

UR+、MiR Go、KUKA Creator Portal 等生态证明认证和 marketplace 能降低采用风险；LeRobot 和 Open X-Embodiment 让数据、策略和跨本体学习进入开发者视野；Qualcomm Dragonwing/AI Hub 正在把边缘部署变成开发者生态。现在的问题从“有没有模型”转向“这个技能能不能可信地上架、安装、验收、升级和回滚”。

500+ UR+ 已形成 500+ approved products 的 cobot 生态，证明认证式 marketplace 有需求。
160+ MiR Go 提供 160+ off-the-shelf applications/top modules，AMR 也需要应用生态。
542K IFR：2024 年全球新增工业机器人约 54.2 万台，技能部署有真实硬件基数。
AI Hub Qualcomm AI Hub/Workbench 支持模型 compile/profile/deploy，适合成为 edge evidence 来源。

03 · Insight

Skill package 不是 ROS package，model card 也不是机器人部署批准。

ROS 解决软件包发现和依赖，LeRobot 解决数据和训练，Foxglove/Formant 解决运行后的数据和运维，UR+ 证明认证能促进购买。但可安装 robot skill 还需要一层更接近实体世界的证据：数据血缘、硬件 profile、edge benchmark、安全边界、人工接管、回滚和现场限制。

SkillCertKit is the trust layer for installable robot skills.

04 · Solution

把策略模型变成可购买、可审查、可回滚的技能证据包。

SkillCertKit 不声称给机器人做法律认证，也不声称跨所有机器人通用。它为每个 skill package 生成结构化证据：在哪些 robot profile 下测试过、训练数据来自哪里、依赖是什么、边缘性能如何、安全边界是什么、失败后怎么回滚、买家如何审计。

  • Skill Manifest：输入/输出、传感器、执行器、权限、ROS/SDK 依赖、runtime 和目标任务。
  • Dataset Lineage：LeRobot-compatible dataset、episode、失败样本、授权、版本和覆盖缺口。
  • Hardware Profile：processor、camera、arm/gripper、payload、firmware、安全 IO 和 calibration。
  • Qualcomm Edge Profile：compile/profile/deploy report、latency、memory、thermal、power 和 fallback。
  • Safety & Rollback：工作空间、速度/力限制、人工接管、审计日志、上一稳定版本和触发条件。

05 · Product Workflow

从开发者提交，到企业买家敢安装，中间必须有硬门禁。

SkillCertKit 不替代开发者创新。它把创新变成可审查、可发布、可支持的产品包，让 SkillDock 不是模型仓库，而是机器人能力商店。

Submit

提交 policy artifact、skill.yaml、依赖、任务边界和目标机器人。

Lineage

检查 LeRobot dataset、episode、失败样本、授权和版本。

Profile

生成 Qualcomm AI Hub/QNN/QAIRT 路线的 edge profile 和部署证据。

Gate

运行安全边界、SBOM/漏洞、license、rollback 和验收测试。

Publish

进入 SkillDock，显示兼容硬件、证据、版本、支持和风险边界。

06 · Evidence Pack

每个技能要带证据，而不是只带权重。

证据包把技术信息变成买家、集成商、IT/安全和 Qualcomm 生态能理解的安装前门禁。它必须区分 tested_on、expected_compatible、not_validated，也必须区分 simulated、hardware_in_loop、bench_real、field_real。

Model Card

模型名、hash、格式、训练代码、输入输出、限制、失败模式和 intended task。

Dataset Card

数据 ID、版本、采集设备、episode、成功/失败标签、license 和覆盖缺口。

Runtime Graph

ROS package、container digest、firmware、QNN/ONNX/runtime 和 calibration。

SBOM / License

SPDX/CycloneDX、依赖、license、漏洞扫描、release-blocking CVE 处理。

Edge Metrics

p50/p95/p99 latency、cold start、memory、temperature、power 和 benchmark duration。

Rollback

上一稳定版本、触发阈值、状态清理、操作员确认和审计日志。

07 · Market Wedge

第一批客户是为“部署信任”付费的人，不是下载免费脚本的人。

工业买方不会像下载手机 App 一样下载机器人技能。真正愿意付费的是需要缩短 commissioning、降低集成风险、通过 IT/安全审查、维护 fleet、复用技能模板的人。

技能开发者

需要 SDK、测试计划、marketplace 可见度、lead flow 和付费部署路径。

系统集成商

需要 repeatable deployment package、文档、兼容矩阵和更快 commissioning。

企业 IT/安全

需要 SBOM、SSO/audit、漏洞流程、网络图和无意外云依赖。

Fleet/RaaS

需要 staged rollout、version tracking、remote intervention、日志和 per-robot economics。

教育/研究

需要低成本 open tier、课程、重复实验和能转到工业平台的技能证据。

08 · Business Model

不要按消费者 App Store 定价，要按工业信任和部署证据定价。

SkillCertKit 的收入来自 certification evidence，而不是下载按钮。免费层服务教育和开源实验；付费层服务 marketplace readiness、enterprise trust pack、私有技能库和 Qualcomm edge 优化。

Open Dev Tier

免费/低价生成 manifest、本地测试和示例 skill，不声称企业认证。

Marketplace Ready

面向开发者/OEM/SI：产品页材料、测试矩阵、edge profile 和发布证据。

Enterprise Trust Pack

SBOM、漏洞流程、签名包、回滚计划、fleet hooks 和采购问卷。

Profile Fee

按技能包、硬件 profile、真实设备 benchmark、私有测试环境收费。

Marketplace Take

SkillDock 交易抽成、企业私有技能库、支持订阅和 SI 分成。

09 · Go To Market

先用一个样板技能证明：从 Demo 到可安装产品的路径。

比赛阶段用 LabForge sample-transfer 或 pick-place 技能做第一个认证样板。先服务 10-20 个机器人开发者和集成商，把“技能上架清单”做成事实标准，再把成功技能接入 SkillDock、EdgeFleet 和企业私有技能库。

Reference Skill

桌面机械臂 sample transfer：数据、模型、edge profile、safety gate。

Developer Beta

10-20 个开发者/SI 提交技能，补全 manifest 和证据 schema。

SkillDock

可搜索技能卡：兼容机器人、证据等级、版本和支持方。

Enterprise Pack

IT/安全、fleet、采购和现场工程可审查的安装材料。

Recertification

现场失败、硬件变更、runtime 更新触发重新验证或 rollback。

10 · Competition

我们不替代 ROS、LeRobot、Isaac、Viam、Foxglove 或 Formant，而是补上安装前信任层。

UR+ 和 MiR Go 证明认证 marketplace 有价值，但偏品牌生态；ROS 和 ROS-Industrial 证明软件包和工业适配重要，但不是企业技能认证；LeRobot 证明数据/策略工作流正在成型，但不是部署批准；NVIDIA Isaac 证明 physical AI 工具链强，但不是 Qualcomm-first 技能证据层；Viam/Foxglove/Formant 证明 registry、数据和 fleet ops 有价值，但不是安装前 certification gate。

Marketplace Listing 不够

上架页面不能证明数据权利、硬件边界、latency、回滚和现场安全假设。

Model Card 不够

模型说明不能替代机器人 profile、传感器/执行器、edge metrics 和接管策略。

Fleet Ops 不够

运行后的日志很重要，但企业更需要安装前 gate 和失败后的重新认证触发。

11 · Moat

护城河是越来越难复制的技能证据网络。

每个通过 SkillCertKit 的技能都会沉淀 manifest、robot profile、dataset lineage、benchmark、失败样本、安全规则、version history、buyer acceptance 和 edge evidence。越多技能通过，标准越难被绕过；越多 Qualcomm profile 沉淀，Dragonwing 生态越有默认入口。

Evidence Graph

skill、dataset、model、hardware、runtime、safety policy 和 buyer acceptance 的关系图。

Profile Library

QCS6490/QCS8550/IQ 路线、相机、机械臂、底盘和现场约束的矩阵。

Failure Memory

失败 episode、field incident、recertification trigger 和 rollback 证据。

Distribution

开发者、SI、OEM、教育和企业私有技能库围绕同一套上架标准。

12 · Why Qualcomm

让 Qualcomm 从 BOM 选择，变成机器人技能发布标准。

Qualcomm 的价值不只是“板子能跑模型”，而是让每个机器人技能必须声明和验证 Dragonwing target、AI Hub profile、QNN/QAIRT package、edge metrics 和 rollback。这样 Qualcomm 参与的不只是一次 Demo，而是技能如何被开发、验证、上架、采购和复用的生态标准。

Dragonwing Target

技能卡必须声明 tested_on / expected_compatible / pending validation 的 Qualcomm profile。

AI Hub Profile

模型 artifact 进入 compile/profile/deploy 路线，生成买家可读的 edge report。

QNN / QAIRT

部署包记录 runtime version、target profile、fallback 和 rollback package。

Reference Skills

视觉抓取、桌面样品转移、AMR 异常闭环可做 Robotics Hub 风格样例。

13 · Demo

三分钟演示一个技能如何从模型变成可上架产品。

Demo 选择 `labforge.sample-transfer.v1`：桌面机械臂样品转移。重点不是机械臂动一下，而是展示企业为什么敢安装这个技能。

  • 展示 skill.yaml：相机、机械臂、gripper、workspace、权限和 target profile。
  • 展示 LeRobot dataset：episode、失败样本、授权边界和训练版本。
  • 触发认证：生成 AI Hub / QNN profile、edge metrics 和 SafetyOps gate。
  • 发布 SkillDock 卡片：兼容硬件、证据等级、版本历史和 rollback。
  • 模拟安装到另一台 robot profile：不兼容时阻止安装，兼容时允许 staged rollout。

我们请求的不是官方认证背书，而是把 SkillCertKit 做成面向 Qualcomm 机器人生态的验证工作流原型：有 target、有 profile、有 runtime、有证据、有开发者样例。

Validation Template

Partner Beta

连接教育套件、SI、机器人 OEM 和自动化买家，测试第一批可认证技能。

15 · Claims & Sources

SkillCertKit 是证据工作流原型，不是官方认证机构。

页面不声称 Qualcomm 官方认证、官方背书、跨所有机器人通用、零风险部署或自动满足安全法规。它主张的是 marketplace readiness：在列出的 robot profiles 和 test conditions 下组织结构化证据，并明确 simulated / hardware_in_loop / bench_real / field_real 的证据等级。

每个技能都应该回答：谁训练的、用什么数据、跑在哪个机器人上、边缘指标如何、失败如何回滚。
