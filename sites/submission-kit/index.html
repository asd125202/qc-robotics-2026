01 · Official Constraints

最重要的不是做更多材料，而是避免错过一次性上传。

官方信息显示，初赛项目书提交截止为 2026-07-20 11:59:59，项目书格式为 .docx 或 .pdf。上传页面提示作品只能提交一次且不能修改，多文件应先打包。

07/20 初赛项目书截止日期。
11:59:59 官方模板中记录的截止时间。
docx/pdf 主项目书建议格式。
一次 上传后不可修改，提交前必须复核。

02 · Package Files

主文件讲清项目，支撑包证明执行力。

建议主项目书从 `docs/preliminary-project-book-cn.md` 导出为 DOCX/PDF；支撑包包含研究、架构、评分对齐、网站截图和链接说明。

主项目书

`RobotMac Core + LeRobot CloudTwin-[团队名].docx` 或 `.pdf`，包含背景、方案、技术路线、商业价值、Qualcomm 支持价值和计划。

支撑材料包

可选 zip：产品主张、评分对齐、云训练计划、硬件产品线、商业模式、研究笔记、网站截图。

在线证据

Cloudflare Pages、GitHub repo、Prototype dashboard、JudgeDeck、BusinessCase、HardwareMap 等页面链接。

03 · Evidence Pack

初赛重点是可行性和完整性，证据要围绕评分点组织。

官方初赛评分里可行性占 50 分，完整性占 30 分。提交包应该突出：已经有可访问网站、清晰技术架构、云训练路径、硬件路线、商业飞轮和可交互 prototype。

50 可行性：硬件路线、fallback dashboard、云训练 adapter。
30 完整性：产品、技术、商业、进度和支撑材料齐备。
20 团队契合：需要在报名表中补齐成员与分工。
URL 线上页面证明项目不是单纯文字概念。

04 · Final Review

上传前最后检查，不要把可修正问题变成不可修改问题。

提交前必须检查团队字段、联系方式、首选设备、文件格式、文件名、链接可访问性和敏感信息。Cloudflare/API/SSH/VPN 相关凭证不得进入任何提交文件。

填写团队

联系人、邮箱、电话、微信、团队名、成员分工。

导出项目书

把 Markdown 草稿整理成 DOCX/PDF。

检查链接

确认 GitHub、Pages、Prototype 页面可打开。

打包支撑

多文件先 zip，避免上传页只收一个文件。

最终上传

使用报名手机号和验证码，一次性提交。

初赛提交的目标不是把所有代码做完，而是证明：方向重要、方案完整、技术可行、商业清楚、Qualcomm 支持有战略价值。
