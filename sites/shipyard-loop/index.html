Pitch 59 · Shipyard Production Evidence Loop

船厂不缺计划，缺现场真相。

ShipyardLoop 是面向船厂、船修厂和海工制造基地的生产证据闭环：把分段、焊缝、打磨、涂装、质检、返工和进度风险从对讲机与纸单，变成可验证、可追溯、可回写的 work package。

01 · Problem

一条船晚，不是因为没人排计划，而是现场进度、质量和返工不够可信。

船厂有 CAD、PLM、ERP、MES、P6 和报表，但分段到了没有、哪条焊缝做完、哪块涂层要返工、哪个材料卡住、谁的下一道工序被挡住，答案仍然散落在对讲机、Excel、照片、纸单和老师傅脑子里。

进度不真

计划显示完成，现场可能还在等材料、等检验、等返工或等前一道工序释放。

质量不连

焊接参数、 seam 图像、打磨覆盖、涂装环境、DFT、NCR 和 inspector disposition 不在同一个证据包。

机器人孤岛

焊接、打磨、喷涂和扫描机器人能做单点自动化，但结果没有自然回到 work package 和排程。

返工太晚

缺陷在交付、试航、船级社或客户验收时才暴露，代价远高于班组当天纠偏。

02 · Why Now

订单爆满、焊工短缺、绿色船舶复杂化和机器人采购正在同时发生。

船厂现在不是缺愿景，而是缺可落地的现场闭环：既能接入 robot cells，又能让工长、质检、计划和项目经理每天看到真实 work package 状态。

322.3M DWT 中国 2026 Q1 造船手持订单量，同比增长 43.6%，交付压力真实存在。
85% GAO 报告中，45 艘在建美国海军战斗舰中 37 艘截至 2024-09 存在延期。
25-28% NDIA/ETI 报告指出焊接约占造船工时的 25-28%，是自动化和证据化的核心成本池。
$448M 美国海军宣布对 Ship OS / AI shipbuilding 投资，说明船厂 AI 已经进入预算层。

03 · Insight

数字船厂过去解决设计数据，下一步要闭环计划和现场之间的最后 100 米。

ShipyardLoop 的判断是：船厂先不会为“全自动船厂”买单，但会为更快释放工序、更少返工、更完整 QA 证据和更可信的每日进度付费。

Work Package

每个分段、焊缝、涂层、管段、电缆和检验点都有位置、计划、负责人和证据要求。

Evidence

机器人、传感器、工长确认和 inspector review 共同生成一份 production evidence packet。

Schedule Signal

现场证据不只是归档，而是直接影响排程、物料、班组、返工和交付风险。

04 · Solution

ShipyardLoop 是船厂 work package 的现场证据层。

它不替代 Siemens、Dassault、AVEVA、ERP 或 MES。它把边缘视觉、机器人、手持扫码、工长确认、质检记录和云训练连接起来，让每个工序从“计划中”变成“已验证、待返工、待检验或已释放”。

  • Work Evidence：焊缝、打磨、表面处理、涂装、DFT、尺寸扫描和 inspector disposition 绑定到 work package。
  • Exception Routing：NCR、返工、等待检验、材料缺失、前序阻塞和安全/许可异常进入任务队列。
  • Robot/Human Handoff：机器人做采集、模拟打磨/冷焊路径和覆盖验证，人负责审批、工艺和安全边界。
  • Schedule Writeback：已完成、待复核、返工中、已释放和风险原因回写 MES/ERP/Ship OS。

05 · Product Workflow

一个焊接/打磨/涂装工包，从计划到已释放。

第一版 demo 不做真实焊接和喷涂。用 mock hull panel、冷焊 marker、低风险打磨垫、相机/热图样例和 mock MES，就能展示商业闭环。

Import

导入工包、分段、焊缝、WPS/工艺、检验点、材料状态和排程。

Capture

边缘相机、LiDAR/深度、热图、工具电流/力、手机扫码和工长确认采集现场证据。

Detect

模型标出 seam、打磨覆盖、表面缺陷、涂层覆盖、等待检验和安全区域。

Approve

工长/质检确认异常、分派返工、补拍证据或释放下一道工序。

Learn

人工接管、低置信度、坏视角和返工结果进入 LeRobot HIL，训练后回到 Qualcomm edge。

06 · Market Wedge

先切高变化、高返工、高协同的工序，不从全船厂替换开始。

最好的试点不是 CIO 愿景项目，而是一个生产经理愿意用 90 天 KPI 验收的工包闭环。

船修 / 改装

现场变化多、返工贵、计划频繁变化，比全新造船更容易证明进度真实价值。

分段装配

block 到位、焊缝完成、尺寸偏差、等待检验和下一道工序释放都能闭环。

焊接质量

in-process weld monitoring、WPS 证据、参数、图像和 inspector review 是高价值切口。

打磨 / 表面处理

HII/GrayMatter 等信号说明 sanding、grinding、blasting、coating 是现实自动化 wedge。

海工模块

FPSO、FLNG、平台导管架、TKY 节点和厚板结构需要复杂工艺和可追溯证据。

中国私有云

CSSC、COOEC、杨子江、新时代、广船、CIMC 等场景需要本地化、私有化和工业数据分级。

07 · Business Model

按 yard、program、active work package、edge node 和验证节省收费。

ShipyardLoop 的价格锚定 rework hours、NCR cycle time、first-pass yield、inspection release、coating window compliance 和 schedule risk。

90 天付费试点

海外 75k-150k 美元；中国 50-150 万元，一条工序/一个分段/一个项目包。

年度站点

250k-600k 美元 / yard / year 起，企业级多项目、多系统集成可到 750k-1.5M+。

中国私有部署

按私有云、边缘节点、工包数量、系统接口、数据安全包和上线服务收费。

成功指标

返工工时、NCR cycle time、首检通过率、工包证据完整率、涂装返工、检验释放时间。

先卖给生产经理和质量负责人，不先卖“数字船厂愿景”。

落地路径必须足够窄：一个项目、一个工序、一个证据链、一组 KPI，然后从工包扩到车间、项目、全厂。

First Buyer

First Workflow

Channels

11 · Product Architecture

现场安全和证据采集在边缘，训练在区域云，工包状态回写企业系统。

ShipyardLoop 是 supervised-first。模型建议路径、识别覆盖、整理证据和发现异常；人类负责工艺审批、质检结论和安全边界。

Sensors

RGB/HDR、weld camera、thermal、LiDAR/depth、laser line、force/current、DFT/env probes。

ROS 2 Edge

/tf、/tool_pose、/rgb、/weld_cam、/thermal、/points、/tool_telemetry 和 rosbag2/MCAP。

Supervisor

seam detect、coverage、safety zone、action proposal、human approval、QA packet。

Dual Cloud

中国数据留在阿里/华为/腾讯/AutoDL/私有云；海外数据走 Runpod/Lambda/Modal/AWS/Azure。

Edge Export

PyTorch/ONNX -> AI Hub compile/profile -> QNN/QAIRT package -> signed model registry -> next shift。

12 · Competition Demo

3 分钟展示一个工包如何从计划变成已验证证据。

演示必须安全：不用真实电弧焊、真实喷涂或高风险打磨，用冷焊 marker、热图样例、低风险覆盖动作和 dashboard 证明闭环。

Scan

机器人/相机扫描 mock hull panel，识别 seam、rust mark、grind zone 和 coating target。

Act

cobot 走冷焊 marker 或低风险打磨覆盖路径，记录 tool pose、camera、thermal 和 force/current。

Review

模型标出风险区域，操作员 gamepad 修正，LeRobot 记录接管片段。

Packet

系统生成 QA packet：before/after、热图、3D map、路径、模型版本、inspector review。

Writeback

mock MES/Ship OS 状态从待处理变成待复核/已释放，并展示 QNN/QAIRT profile。
