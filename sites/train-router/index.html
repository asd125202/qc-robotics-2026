Pitch 25 · Robot Training Delivery Layer

TrainRouter

机器人训练不应该停留在“找一块 GPU 跑一次脚本”。TrainRouter 把 LeRobot 数据集、预算、数据地域、评估门槛和 Qualcomm 边缘目标写成同一份训练合约，让中国云和海外云都能交付同一种可验证的机器人技能版本。

从问题开始 看 8 分钟演示

01 · Problem

机器人团队能录数据，也能训练一次模型，但很难把训练变成交付。

真正的商业客户不只问“模型有没有训练出来”。他们会问预算是否失控、数据是否跨境、不同云训练结果是否可比较、模型能否部署回真实机器人，以及失败后能否回滚。

痛点 1

云 GPU 成本不可控

训练任务被拆散在多个账号、控制台和镜像里，团队很难在启动前承诺费用、时长和失败重试边界。

痛点 2

中国版和海外版割裂

中国客户、海外客户、公开 demo 数据和企业私有数据面对不同区域、支付、网络和合规约束。

痛点 3

训练结果不可比较

同一份 LeRobot 数据在不同 provider 上跑完，如果没有固定 split、seed、容器和评估脚本，就无法判断模型真的变好。

痛点 4

云端结果断在边缘部署前

商业价值发生在机器人本体。模型文件如果不能进入 Qualcomm edge runtime、profile、回滚和技能发布链路，就不是产品交付。

02 · Why Now

机器人学习正在从研究样例，走向可规模化的训练供应链。

时机来自三件事同时发生：LeRobot 把数据和策略工作流标准化，云 GPU 市场碎片化且价格/库存快速变化，Qualcomm 的边缘 AI 工具链让“训练后部署到真实设备”成为可验证目标。

LeRobot

机器人数据格式开始统一

LeRobotDataset v3 将多模态时间序列、视频和元数据组织成可训练、可索引、可流式使用的数据层，适合变成商业训练合约。

市场规模

机器人部署正在扩大

IFR 2025 报告显示 2024 年工业机器人安装量约 54.2 万台，亚洲占 74%，中国占全球部署 54%。中国版和海外版不是选择题，而是产品现实。

边缘目标

部署证据可以产品化

Qualcomm AI Hub 支持模型转换、on-device profile、数值正确性验证和 QAIRT/QNN 运行时路径，这给机器人训练结果提供了清晰的落地门槛。

03 · Solution

一份 job contract，路由到任意云，输出同一种机器人技能版本。

TrainRouter 的核心洞察是：机器人训练不是购买 GPU，而是发布一个可审计的 skill release。每次 release 都必须绑定 dataset hash、训练容器、预算、评估结果、Qualcomm target profile 和 rollback package。

Contract

写入 LeRobot dataset、policy、robot profile、budget、region 和 eval gate。

Route

按 China / Overseas / Auto lane 选择 AutoDL、PAI、Runpod、Lambda 或 Modal。

Train

锁定容器、split、seed、checkpoint 和重试策略，记录 cost estimate 与运行日志。

Evaluate

统一评估成功率、失败类型、动作平滑度和 replay 指标，不让 provider 差异污染结论。

Export

生成 Qualcomm edge policy package、runtime evidence、回滚包和 SkillDock 发布凭证。

04 · Product

用户看到的是训练订单，不是云厂商控制台。

TrainRouter 把 provider 差异封装成 adapter，把客户关心的边界放到任务合约里：最多花多少钱、数据留在哪里、训练失败怎么办、什么指标算通过、模型要部署到哪块 Qualcomm 板子。

  • 预算守卫：70% 预警、85% 自动 checkpoint、95% 停机或人工确认。
  • 数据边界：中国客户默认 China lane，海外公开 demo 可进入 Auto lane。
  • 训练复现：固定 dataset hash、split、seed、container digest 和 eval harness。
  • 交付闭环：输出 edge artifact manifest，而不是只输出一个权重文件。

05 · Market & Business Model

从一次训练脚本，升级成机器人技能生产线。

目标客户不是“所有 AI 团队”，而是已经开始采集真实机器人数据、需要把训练结果交付给硬件和业务场景的团队：高校实验室、机器人初创公司、OEM、系统集成商和企业创新部门。

Developer Starter

开发者起步包

面向比赛队伍和高校实验室，提供固定预算的小 GPU 训练额度、公开数据 lane 和基础 edge export。

Pro Lab

实验室订阅

按月收取平台费与计算转售服务费，支持多次 train/eval loop、artifact storage 和评估报告。

SI Pilot

集成商试点包

4-8 周固定试点，绑定数据采集、训练迭代、Qualcomm edge evidence 和现场验收指标。

Enterprise Lane

企业私有训练通道

私有存储、区域策略、SSO/RBAC、审计日志和指定 provider，对接企业数据与采购流程。

06 · Competition & Moat

我们不是又一个 GPU 云，而是机器人训练 release system。

Runpod、Lambda、PAI、ModelArts、Modal 解决算力；Ray、SkyPilot、Kubeflow 解决通用调度；Viam、Formant、Foxglove 解决部分机器人运维或数据可视化。缺口在于：没有一层把 LeRobot 数据、双云训练、预算边界、统一评估和 Qualcomm edge export 绑定成商业交付物。

短期壁垒

机器人训练合约库

积累不同机器人、传感器、动作空间、policy、GPU class 和 Qualcomm target 的可复用 job templates。

中期壁垒

评估与失败数据

把每次失败、人工 takeover、replay eval 和边缘 profile 变成下次训练的选择依据，形成数据飞轮。

长期壁垒

Qualcomm-first skill release graph

每个技能版本都有硬件兼容、runtime evidence、rollback 和市场上架记录，切换云供应商不会切断产品资产。

07 · Why Qualcomm

云可以换，机器人本体上的边缘标准不能换。

TrainRouter 把所有云端训练结果收敛到 Qualcomm 边缘 AI 证据包：模型格式、AI Hub compile/profile、QAIRT/QNN runtime、设备验证、功耗/延迟标签和回滚策略。这样 Qualcomm 不只是提供开发板，而是成为机器人技能交付的默认目标平台。

Route 中国云和海外云只是训练资源池，客户接口保持一致。
Compile 训练结果进入 Qualcomm AI Hub / QAIRT / QNN 导出门槛。
Profile 每个 release 附带延迟、内存、算力单元和数值正确性标签。
Publish 通过 SkillCertKit 后进入 SkillDock，成为可售机器人技能。
需要高通支持

AI Hub 额度与 target profile

用于比赛期间把 demo policy 真实跑过 compile/profile 流程，形成可信证据截图和报告。

需要高通支持

Dragonwing 开发板与 Device Cloud

让复赛作品从网页原型升级为端侧验证，不只停留在云端训练。

需要高通支持

Profiler 指标模板

把机器人技能发布指标标准化，帮助生态伙伴判断一个技能是否可以上架和交付。

比赛演示从一个 LeRobot job contract 开始，到一个 Qualcomm edge evidence packet 结束。中间每一步都要让评委看到商业客户会付钱的控制点。

Demo 3-4 min

Demo 5-6 min

TrainRouter 的一句话：把“租 GPU 训练模型”变成“按预算发布一个能在 Qualcomm 机器人上验证的技能版本”。
